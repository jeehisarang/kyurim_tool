## 배경
- 7/16 Commit 04bebf8에서 HRV AI 코멘트의 핵심 앵커를 "자율신경균형도 유형"(과로형만성스트레스/
  질병형만성스트레스/급성스트레스/초기부정맥/심한부정맥)과 "맥박다양성" 수치로 확정했음.
- 오늘(7/17) 이순영 환자, 상담모드(증상 입력)로 실제 코멘트 재생성 결과 확인한 결과 2가지 문제 발견:
  1) "이번 결과와 추이", "한의학적 해석", "치미병 양생 안내" 세 섹션 모두에서 
     동일하거나 거의 동일한 문장이 문단 내에서 반복 등장 (예: "한의학적 해석" 섹션은 
     "급성스트레스 구역에 위치해, 아직 뚜렷한 병증은 아니지만 자율신경 불균형이 시작되는 
     신호로 볼 수 있습니다"가 완전히 동일하게 두 번 등장)
  2) 04bebf8에서 핵심 앵커로 삼기로 한 "자율신경균형도 유형"이 명확한 인용 형태가 아니라 
     애매하게 스쳐 지나가듯("~가까운 것으로 보이며") 언급되는 수준이고, "맥박다양성" 수치는 
     이번 생성 결과에 아예 등장하지 않음
- 증상(가슴 두근거림/불면증/두통/현훈) 등 상담모드 입력 자체는 정상 반영되는 것으로 확인됨 
  (증상 기반 로직 자체는 문제 없어 보임)

## 사전확인체크리스트 (설계 승인 포함)
- [ ] 이 작업은 기존 확정된 설계(04bebf8의 앵커 방식)를 원래 의도대로 복구/수정하는 작업이라 
      별도 설계 변경 승인 불요. 단, 조사 결과 앵커 로직 자체를 바꿔야 할 필요가 발견되면 
      원장 승인 후 진행할 것.
- [ ] 반복 문장 버그의 원인 후보를 코드 레벨에서 먼저 조사할 것 (아래 3가지 중 하나 이상일 가능성):
      (a) 자체검토(self-check)/재시도 로직이 재생성 결과를 기존 텍스트에 append하고 있는지
      (b) 프롬프트 구조상 모델에게 "요약/정리"를 별도로 요청해 앞 내용을 다시 요약하게 되어있는지
      (c) 두 번의 API 호출(이미지 판독 + 텍스트 생성) 결과가 잘못 concat되고 있는지
- [ ] 상담모드(증상 입력 경로)와 일반모드(증상 미입력 경로)가 서로 다른 프롬프트/코드 경로를 
      타고 있는지 확인 — 04bebf8에서 확정한 자율신경균형도/맥박다양성 앵커가 상담모드 경로에서는 
      누락되고 있을 가능성이 있음 (일반모드에서는 정상 반영되는지도 같이 확인)

## 동작 규칙
1. 반복 문장 버그 원인 확인 후 근본 수정 (append→replace 수정, 프롬프트 중복 지시 제거, 
   concat 로직 수정 등 원인에 맞는 방식으로)
2. 자율신경균형도 유형과 맥박다양성 수치가 "이번 결과와 추이" 섹션에 명확한 형태로 
   반드시 언급되도록 프롬프트/로직 확인 및 수정 (예: "자율신경균형도는 [유형명]에 해당하며, 
   맥박다양성은 [수치]입니다" 형태로 명시적 인용)
3. 상담모드/일반모드 두 경로 모두 이 앵커가 누락되지 않도록 프롬프트 공통화 또는 
   양쪽 모두 수정
4. 수정 중 기존 안전장치(패턴명 매핑표 위반 감지 시 재시도, 최종 실패 시 코멘트 실패 처리)는 
   그대로 유지할 것 — 건드리지 말 것
