# 작업 지시서 — 검사(인바디/근력) 기능 구현 (13-6, B안 승인)

## 배경
직전 설계안(B안: 별도 테이블, 계산값 저장시점 고정, 판정기준 상수화, 독립화면
/examinations/new)으로 승인. 아래 스키마/로직으로 구현 진행.

---

## 1. 스키마
- StrengthTestRecord (제시된 필드 그대로: patientId, prescriptionId(nullable),
  measuredAt, gender, heightCm, armMuscleMassLeftKg/RightKg, legMuscleMassLeftKg/RightKg,
  smi, smiJudgement, gripLeftKg, gripRightKg, gripAvgKg, gripJudgement, staffUserId, createdAt)
- BodyCompositionRecord 신규 (설계안에 직접 언급 안 됐으나 인바디용으로 필요):
  { id, patientId, prescriptionId(nullable), measuredAt, weightKg, note(String?, 자유메모),
    staffUserId, createdAt }
  - 인바디는 요구사항상 "체중 위주 감량추이 확인"이 핵심이므로 이번엔 체중 중심의
    단순 구조로 시작 (체성분 세부수치까지는 이번 범위 아님, 필요시 향후 확장)
- prescriptionId는 두 테이블 모두 nullable 실FK로 연결

## 2. 판정 기준 상수 (src/lib/exam-thresholds.ts)
- SMI 기준: 남성 SMI < 7.0 → 근감소증(SARCOPENIA), 여성 SMI < 5.7 → 근감소증, 그 외 NORMAL
- 악력 기준표 (성별×연령대, kg 단위, WEAK/NORMAL/STRONG 3구간):

| 연령대 | 남성 WEAK | 남성 NORMAL | 남성 STRONG | 여성 WEAK | 여성 NORMAL | 여성 STRONG |
|---|---|---|---|---|---|---|
| 20-24 | <36.8 | 36.8-56.6 | >56.6 | <21.5 | 21.5-35.3 | >35.3 |
| 25-29 | <37.7 | 37.7-57.5 | >57.5 | <25.6 | 25.6-41.4 | >41.4 |
| 30-34 | <36.0 | 36.0-55.8 | >55.8 | <21.5 | 21.5-35.3 | >35.3 |
| 35-39 | <35.8 | 35.8-55.6 | >55.6 | <20.3 | 20.3-34.1 | >34.1 |
| 40-44 | <35.5 | 35.5-55.3 | >55.3 | <18.9 | 18.9-32.7 | >32.7 |
| 45-49 | <34.7 | 34.7-54.5 | >54.5 | <18.6 | 18.6-32.4 | >32.4 |
| 50-54 | <32.9 | 32.9-50.7 | >50.7 | <18.1 | 18.1-31.9 | >31.9 |
| 55-59 | <30.7 | 30.7-48.5 | >48.5 | <17.7 | 17.7-31.5 | >31.5 |
| 60-64 | <30.2 | 30.2-48.0 | >48.0 | <17.2 | 17.2-31.0 | >31.0 |
| 65-69 | <28.2 | 28.2-44.0 | >44.0 | <15.4 | 15.4-27.5 | >27.5 |
| 70-99 | <21.3 | 21.3-35.1 | >35.1 | <14.7 | 14.7-24.5 | >24.5 |

- 이 표를 그대로 상수 배열/객체로 코드에 반영. 나이가 해당 구간에 없으면(예: 20세 미만)
  에러 대신 "판정불가" 정도로 안전하게 처리

## 3. 계산 로직 (서버 재계산 원칙)
- SMI = (좌우 팔 골격근량 + 좌우 다리 골격근량 합) / (키(m)^2)
- 악력평균 = (좌+우)/2
- 클라이언트는 미리보기용으로만 같은 공식을 순수함수로 공유해서 즉시 계산 표시
- 서버는 클라이언트가 보낸 계산값을 신뢰하지 않고, 원본 입력값만 받아 항상 자체 재계산 후 저장

## 4. 화면 (/examinations/new)
- 환자 검색 (기존 /prescriptions/new 패턴 재사용)
- 검사 종류 선택: 인바디 / 근력검사
- 인바디 선택 시: 체중, 메모(선택) 입력
- 근력검사 선택 시: 성별(측정시점 스냅샷), 키, 팔(좌/우) 골격근량, 다리(좌/우) 골격근량,
  악력(좌/우) 입력
- 진행 중인 치료처방 있으면 드롭다운으로 선택 가능(선택사항, 자동매칭 없음)
- 저장 전 실시간 미리보기: SMI/판정, 악력평균/판정 즉시 표시
- 저장 완료 시 결과 화면에 다시 한번 명확히 표시 (환자 앞에서 바로 설명하는 용도이므로
  글자 크게, 판정 결과 눈에 띄게)

## 5. 목록 화면
- /examinations 에 전체 목록(환자별 필터 가능한 표 형태)
- 컬럼: 환자명, 검사종류, 측정일, 주요 수치(체중 또는 SMI/악력), 판정, 측정자
- 그래프/추이 시각화는 이번 범위 아님 (13-7에서 처리 예정)

## 범위 밖 (이번엔 하지 않음)
- 환자 통합뷰/그래프(13-7)
- 검사결과 ↔ 치료처방 자동연동(예: 강근단 등록 시 검사 알림 자동생성)
- 내원목록 뱃지 노출(13-7 관련)
