## 작업 지시서 — 치료처방(Prescription) 상세 페이지 신설 + 프로그램 배지 이동 버그 수정

### 배경
- 치료처방 목록(`/prescriptions`)과 환자 프로필(`/patients/[patientId]`)에서 
  프로그램 배지(예: "환약·3개월", "킬팻캡슐·3개월")를 클릭하면 상세 정보 없이 
  "오늘 할 일"(`/todo`)로 잘못 이동하는 버그 발견 (실사용 중 원장님 직접 확인)
- 근본적으로는 "상세 정보"라는 화면 자체가 아직 없음 — 지금은 차수/다음처방일/
  담당자 정도의 1줄 요약만 프로필에 노출되고, 그 이상 들여다볼 곳이 없음
- 환자 한 명의 흐름을 "치료처방 / 검사" 두 축으로 들여다볼 수 있게 하는 게 
  목표이며, 이번 작업은 그중 **치료처방 상세** 부분만 우선 진행
- **해피톡 AI 문구생성(탕약/환약/킬팻1·3개월)은 이번 범위 아님** — 별도 설계 
  후 추후 작업. 이번 상세페이지에는 톡 이력/문구생성 관련 섹션을 아예 넣지 
  않음 (나중에 그 기능 만들 때 같이 추가)

### 신규 화면
- `/prescriptions/[prescriptionId]` 신설 — 특정 환자의 특정 치료처방 1건에 
  대한 상세 페이지
- 표시 내용:
  - 환자명, 프로그램명, 카테고리(탕약/환/킬팻캡슐), 담당자, 상태(진행중/중단/완료)
  - 등록일(시작일), 총기간, 총회차(totalRounds)
  - **회차별 리스트**: 1차~N차 각각의 예정일/완료여부(완료일 포함) — 
    computeSplitSchedule() 등 기존 회차 계산 로직 재사용해서 조회
  - SINGLE 타입(S환 등 구형 프로그램) 및 FIXED_SEQUENCE(킬팻캡슐 3일체험)는 
    회차 개념이 다르므로 타입별로 적절한 표시 방식 분기 (SINGLE: 처방일+
    후속조치 예정일 1줄, FIXED_SEQUENCE: 웰컴/2일차/마감 각 이벤트 상태)
  - 이 처방으로 인해 생성된 "오늘 할 일" 항목들(NEXT_DOSE, FOLLOW_UP 등) 
    이력 — 완료된 것/예정된 것 구분 표시
  - 수정/중단 버튼 (기존 치료처방 목록에 있던 기능을 여기로 이동 또는 
    동일 기능 복제)

### 대상 파일 (Claude Code가 정확한 경로 재확인)
- 신규 페이지: `src/app/prescriptions/[prescriptionId]/page.tsx`
- 신규 API: `GET /api/prescriptions/[prescriptionId]` — 처방 기본정보 + 
  회차별 스케줄 + 관련 TodoTask 이력 조합해서 반환
- 치료처방 목록(`/prescriptions`) 컴포넌트 — 프로그램 배지 클릭 시 
  `/prescriptions/[prescriptionId]`로 이동하도록 링크 수정 (현재 `/todo`로 
  잘못 이동하는 버그 수정)
- 환자 프로필(`/patients/[patientId]`) 컴포넌트 — "진행중인 치료처방" 섹션의 
  프로그램 배지도 동일하게 상세페이지로 링크 수정
- 회차 계산 로직 파일(`computeSplitSchedule` 등) — 상세페이지에서 재사용 
  가능한 형태인지 확인, 필요시 조회 전용 함수로 분리

### 동작 규칙
1. 치료처방 목록/환자프로필 어느 경로로 들어와도 동일한 상세페이지로 연결
2. SPLIT 타입(탕약/환약/킬팻1·3개월): 회차 리스트가 전체 총회차만큼 
   나열되고, 지난 회차는 완료 상태(완료일 포함)로, 남은 회차는 예정일만 표시
3. SINGLE 타입(S환/하비환 등 구형): 처방일 + 후속조치 예정일/완료여부만 
   간단히 표시
4. FIXED_SEQUENCE(킬팻캡슐 3일체험): 웰컴(D0)/2일차(D2)/마감(D3) 각 이벤트의 
   상태(완료/예정/보류) 표시
5. 이 처방과 연결된 TodoTask 이력은 최신순으로, 완료여부와 함께 표시
6. 톡 문구생성/발송 관련 섹션은 이번엔 만들지 않음 (추후 별도 작업에서 추가)
7. 중단된 처방(중단/완료 이력)도 조회 가능해야 함 (프로필의 "중단/완료 이력 
   보기"에서 들어오는 경로도 고려)

### 검증 체크리스트
- [ ] 치료처방 목록에서 SPLIT 타입 배지(예: 환약·3개월) 클릭 → 상세페이지 
      정상 이동, 회차 리스트 정상 표시 확인
- [ ] SINGLE 타입(S환) 배지 클릭 → 상세페이지 정상 표시(처방일+후속조치) 확인
- [ ] FIXED_SEQUENCE(킬팻캡슐 3일체험) 배지 클릭 → 웰컴/2일차/마감 상태 
      정상 표시 확인
- [ ] 환자 프로필에서 동일하게 진입 시 정상 동작 확인
- [ ] 관련 TodoTask 이력이 정확하게(완료/예정 구분) 표시되는지 확인
- [ ] 중단된 처방도 프로필의 "중단/완료 이력 보기"에서 진입 시 정상 표시 확인
- [ ] "오늘 할 일"로 잘못 이동하던 기존 버그 완전히 해결됐는지 확인
- [ ] npx tsc --noEmit, npm run build 통과
