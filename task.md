## 작업: 환자별 통합 공유링크(티칭지+이벤트) 기능 + 티칭지 생성 유실 버그 수정

### 배경
- 현재 프로그램티칭(`/p/[token]`)과 이벤트(`EventImage`)가 완전히 분리된 별개 기능임
- 톡생성기(내원톡/2일/7일/3회차톡 등)에서 환자별 문구를 생성할 때, URL을 선택적으로
  포함할 수 있어야 함:
  1. URL 없음 (지금처럼 문구만)
  2. 저장된 프로그램티칭 URL만
  3. 저장된 이벤트 URL만
  4. 프로그램티칭 + 이벤트 통합 URL (화면 순서: 티칭지 먼저, 그 아래 이벤트)
- 최종적으로 나온 문구(+URL)는 지금 톡생성기와 동일하게 복사 → 담당자가 수동으로
  그 환자 카카오채널에 붙여넣어 발송

### 0. [선행] 티칭지 생성 유실 버그 원인 조사 및 수정
- 현재 프로그램티칭 생성 플로우에서, 환자를 선택하고 생성 과정 도중 화면을 벗어나면
  (또는 다른 곳으로 이동하면) 진행 중이던 내용이 저장 안 되고 사라지는 문제 있음
- **root cause 파악 우선**: 어느 시점에 상태가 날아가는지(로컬 state만 들고 있다가
  unmount로 유실되는 것인지, 저장 API 호출 자체가 마지막 "완료" 버튼에만 걸려있는
  구조인지 등) 먼저 확인하고, 패치가 아니라 구조적 원인을 고칠 것
- 이번 작업(아래 1~3번)에서 생성 플로우를 인라인 방식으로 재배치하면서 자연스럽게
  해결되는 부분도 있을 수 있으나, 별도로 남아있는 유실 지점이 있다면 함께 수정

### 1. 스키마 추가
- 신규 모델 `PatientShareLink`:
  - `id`, `token`(unique, 랜덤), `patientId`
  - `teachingPageId`(nullable, FK → PatientTeachingPage)
  - `eventImageId`(nullable, FK → EventImage)
  - `createdByStaffId`, `viewCount`(default 0), `firstViewedAt`(nullable), `createdAt`
  - 제약: teachingPageId/eventImageId 중 최소 1개는 있어야 생성 가능 (둘 다 null이면
    애초에 링크가 필요 없는 케이스이므로 서버단에서 막을 것)
- 기존 `PatientTeachingPage`, `EventImage`는 스키마 변경 없음 (참조만 추가)

### 2. API
- `POST /api/share-links` — patientId + (teachingPageId? / eventImageId?) 받아서
  생성 (이미 동일 조합의 링크가 있으면 재사용해서 중복 생성 방지)
- `GET /api/patients/[patientId]/teaching-pages` — 그 환자에게 이미 생성된 티칭지
  목록 (프로그램명 + 생성일 표시용, 드롭다운 채우기용)
- `GET /api/share-links/[token]` — 공개 조회용 (조회수/최초조회 기록 포함, 기존
  `/p/[token]` 조회 로직 참고)

### 3. 공개 페이지 `/s/[token]`
- 사이드바 없는 독립 화면 (기존 `/p/[token]` 스타일 재사용)
- teachingPageId 있으면 기존 티칭지 5필드 렌더링 컴포넌트 그대로 재사용해 위쪽에 표시
- eventImageId 있으면 그 아래에 이벤트 섹션 표시 (완성 이미지 or 합성 이미지 +
  title/intro, 전체 copy는 카톡 발송용이므로 여기선 이미지 위주로만 노출)
- 助 없으면(단일 컴포넌트만 있으면) 그 섹션만 표시, 나머지는 렌더링 안 함

### 4. 톡생성기 UI — 공유링크 선택 패널
- 환자별 톡 문구 생성 화면에 "링크 포함하기" 섹션 신규 추가:
  - 라디오/토글: [없음] [프로그램티칭] [이벤트] [통합]
  - 프로그램티칭 선택 시: 그 환자의 기존 저장된 티칭지 드롭다운
    (`GET /api/patients/[patientId]/teaching-pages` 결과)
    - 비어있거나 새로 만들고 싶으면 "새로 만들기" 버튼 → **기존 프로그램 티칭
      생성 플로우를 같은 화면에 인라인(모달 또는 확장 패널)으로 열기** (별도
      페이지 이동 없음) → 생성 완료 즉시 방금 만든 티칭지가 드롭다운에 자동
      선택된 상태로 반영
  - 이벤트 선택 시: 활성 이벤트(EventImage, isActive) 드롭다운
  - 선택 완료 후 "링크 생성" → `POST /api/share-links` 호출 → 생성된 URL을
    최종 톡 문구 하단에 자동 삽입, 복사 버튼으로 문구+URL 통째로 복사 가능

### 5. 검증 요청
1. 프로그램티칭만 포함 → `/s/[token]` 접속 시 티칭지만 정상 표시
2. 이벤트만 포함 → 이벤트만 정상 표시
3. 통합(둘 다) → 티칭지 위, 이벤트 아래 순서로 정상 표시
4. 같은 환자로 "새로 만들기" 인라인 플로우 실행 → 생성 도중 실수로 다른 곳
   클릭/이탈해도 유실 안 되는지 확인 (버그 재현 테스트)
5. 동일 조합으로 링크 재생성 시도 → 기존 링크 재사용되는지(중복 생성 안 되는지) 확인
6. `npx tsc --noEmit`, `npm run build` 통과 확인
7. 테스트 데이터(생성한 공유링크, 테스트 티칭지 등) 정리
