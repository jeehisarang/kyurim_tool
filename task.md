[작업지시서] HRV 코멘트 리뉴얼 — 카드형 레이아웃 + 헤드라인 + 명칭 변경 + 위험신호 고정문구

## 배경
오늘 실사용(김우석 테스트) 결과를 보고 원장님이 요청한 개선사항:
1. "AI 코멘트"라는 용어 제거 (AI라는 말이 안 보였으면 함)
2. 4개 섹션 나열식 → 카드형 매거진/리포트 스타일로 리디자인
3. 환자가 체크한 증상을 볼드/색상으로 강조해 "내 얘기"라는 인정감 부여
4. 훅이 되는 헤드라인 — 단, 질병명 직접 예측 아니라 "증상 인정 + 완만한 경과 언급" 혼합형
5. 치료 방향(화담이수/건비화습 등 처방 카테고리 + "구체적인 건 원장님과 상담") 포함
6. 위험신호(red flag) 안내 — 단, AI 자유생성 금지, 카테고리별 고정문구를 시스템이 삽입

## 명칭 변경
"AI 코멘트" → "건강 리포트" (기존 "환자와 함께보기" 버튼/기능과는 다른 명칭이니 혼동 주의 — 
그 기능명은 그대로 유지, 이번 라운드에서 바뀌는 건 이 카드 패널의 라벨과 버튼 텍스트만)
- "코멘트 수정" → "리포트 수정"
- "AI 코멘트 재생성" → "리포트 다시 만들기"
- 그 외 "삭제", "환자와 함께보기"는 유지

## 카드 구성 (7개, 기존 4개 섹션 대체)
1. **헤드라인 카드** (히어로 영역, 옅은 베이지/연녹색 배경)
   - 형태: "평소 [환자체크증상1]과 [환자체크증상2]이 반복된다면, [패턴명]의 미병 경향을 
     살펴볼 필요가 있습니다" → 자연스러운 문장으로 다듬어 출력
     예: "평소 몸이 무겁고 잘 붓는 증상은 수분대사가 원활하지 않은 담습 경향과 
     관련될 수 있습니다."
   - 헤드라인 뒤에 완만한 경과 언급 1문장 추가 (질병명 금지, 증상 연쇄만): 
     "이 상태가 계속되면 [일반 증상 1~2개]로 이어질 수 있어, 지금부터 관리가 필요합니다."
2. **내가 선택한 증상** — 상담설문에서 "경미하다/심하다"로 체크한 항목 원문 그대로 나열, 
   진한 남색/갈색 강조
3. **이번 검사에서 주목할 변화** — 직전 대비 변화 폭이 큰 지표 2~3개만 선별해 언급 
   (전부 나열 금지), 개선=녹색, 관찰필요=주황색으로 색상 구분
4. **한의 건강해석** — 기존 "한의학적 해석" 내용, "맥파검사만으로 확정한 것이 아니라 
   상담설문과 검사변화를 종합한 결과"라는 투명성 문구 유지
5. **이런 경향이 지속되면** — 질병명 아닌 증상 수준 경과만 서술 
   (예: "아침 피로, 부종, 소화지연, 몸의 무거움 등이 함께 나타날 수 있습니다")
6. **위험신호 안내** (조건부 노출, 시스템 고정문구) — 아래 "위험신호 고정문구" 항목 참고
7. **치료 방향 + 생활관리** — 치료원칙(화담이수/건비화습 등, TcmCategory.treatmentPrinciple 
   참조) + "구체적인 처방과 치료 기간은 문진·설진·맥진 후 원장님과 상담" 문구 고정 + 
   생활습관 안내(기존 학술근거 exam-guides 내용 활용)

## 위험신호(red flag) 고정문구 — 스키마 소규모 추가 필요
- TcmCategory에 redFlagNotice 필드(nullable) 추가
- 이 필드는 AI가 생성하지 않고, 원장이 각 카테고리별로 직접 입력 
  (오늘은 우선 비워두고, 담습·부종 카테고리 예시로 아래 텍스트를 기본값으로 시드):
  "부종이 갑자기 심해지거나 숨이 차거나 가슴 통증, 소변량 감소, 한쪽 다리만 붓는 증상이 
  동반되는 경우는 다른 원인 확인을 위해 진료가 필요합니다."
- 이 필드가 채워진 카테고리가 후보로 나왔을 때만 "위험신호 안내" 카드 노출, 
  값 없으면 카드 자체 숨김
- 기존 부정맥 안전장치(ensureArrhythmiaNotice)와 동일한 원칙: "시스템이 결정적으로 삽입, 
  AI 프롬프트 지시로만 맡기지 않음"

## 프롬프트 조립 순서 (기존 로직 대체)
1. 가장 높은 카테고리 점수 후보 선택 (동점이면 병렬)
2. 해당 카테고리에서 환자가 "심하다"로 체크한 문항 우선, 없으면 "경미하다" 문항 중 
   상위 2개 추출 → 헤드라인 재료로 사용
3. 검사 지표 중 직전 대비 변화폭이 큰 2~3개만 선별 (전체 나열 금지)
4. 카드 순서대로 프롬프트 조립, 기존 안전장치(반복문장 금지, 패턴명 매핑 위반 재시도, 
   부정맥 경고) 전부 유지
