# 작업지시서 — 카테고리 비중 막대, 100% 고정 노출 버그 조사+수정

## 배경
- 커밋 63a40e3에서 "전체 응답 문항 대비 원점수 비중" 방식으로 계산 전환을
  확정했으나(예: 최승희 케이스 스트레스 18%/기력 18%/담습 9%/순환 9% 처럼
  서로 다른 값이 나와야 함), 이후 색상/아이콘/한자 작업(커밋 d385232) 이후
  실사용 화면에서 확인해보니 4개 카테고리 전부 다시 100%로 표시되고 있음.
- 이전 방식(카테고리 내부 정규화, 후보는 항상 만점이라 전부 100%)으로
  회귀한 것으로 보임 — 원인 확인 필요.

## 조사 항목
1. 커밋 63a40e3와 d385232 사이(또는 그 이후) 비중 계산 로직 파일을 diff로
   비교 — 계산 함수 자체가 실수로 되돌려졌는지, 아니면 색상 매핑 작업 중
   다른 함수를 잘못 참조하게 됐는지 확인
2. 만약 계산 함수(전체비중 로직)는 그대로인데 프론트엔드가 옛 필드(카테고리
   내부 비율)를 계속 읽고 있는 거라면, 어느 필드를 참조 중인지 확인
3. 최승희 실제 데이터로 직접 계산해서 (전체 문항 만점 대비 원점수) 기대값과
   현재 노출값을 비교 — 기대값과 다르면 어느 단계(계산/저장/렌더링)에서
   틀어졌는지 특정

## 수정 규칙
- 원인 확정 후, "전체 응답 문항 대비 원점수 비중" 계산으로 재확정 반영
  (도넛/막대 둘 다 이 값을 참조해야 함 — 서로 다른 계산식을 쓰고 있었다면
  하나로 통일)
- 색상 매핑은 그대로 유지, 값 계산만 수정

## 검증 체크리스트

      (또는 현재 저장된 실제 응답 기준 값)로 서로 다르게 나오는지 확인
- [ ] 테스트환자(426), 문항 수 다른 카테고리 조합으로 재검증
- [ ] 도넛 조각 비율도 막대와 일치하는지 확인
- [ ] npx tsc --noEmit / npm run build 통과
