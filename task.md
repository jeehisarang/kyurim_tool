## 작업 지시서 — 이벤트 이미지(EventImage) 목록, 비활성 항목 기본 숨김 처리

### 배경
- EventImage는 이미 활성화/비활성화 기능은 있으나, 비활성화해도 목록 상단에 계속 
  노출되어 새 이벤트 만들 때 화면이 번잡함
- 기존 "/todo" 화면의 완료 항목 접기/펼치기 패턴(완료 항목 기본 숨김 + "완료된 항목 
  보기" 토글)과 동일한 방식으로 통일할 것

### 대상 파일
- EventImage 목록을 렌더링하는 컴포넌트/화면 (이벤트 이미지 생성기 관련 목록 화면 
  — 정확한 파일 위치는 Claude Code가 확인)

### 동작 규칙
1. 이벤트 이미지 목록에서 **isActive=true인 항목만 기본으로 표시**
2. 목록 하단(또는 상단)에 **"비활성화된 이벤트 보기 (N건)"** 형태의 토글 버튼 추가
   - N은 실제 비활성 이벤트 개수
   - 클릭 시 비활성 항목들이 펼쳐지고, 문구는 "비활성화된 이벤트 숨기기"로 전환
3. 비활성 항목이 펼쳐졌을 때도 시각적으로 활성 항목과 구분되도록 처리 
   (예: 흐림 처리, "비활성" 뱃지 — 기존 톤 유지)
4. 기존 활성화/비활성화 토글 버튼 자체의 동작(클릭 시 상태 전환)은 그대로 유지, 
   위치나 표시 방식만 추가
5. 완전삭제 기능은 이번 작업 범위 아님 (만들지 않음)

### 검증 체크리스트
- [ ] 신규 이벤트 생성 시 활성 목록 상단에 정상 노출되는지 확인
- [ ] 기존 이벤트 비활성화 시 목록에서 즉시 사라지고 "비활성화된 이벤트 보기" 
      카운트가 올라가는지 확인
- [ ] 토글 클릭 시 비활성 항목들이 펼쳐지고, 다시 클릭 시 접히는지 확인
- [ ] 비활성 상태에서 재활성화 시 다시 활성 목록으로 정상 복귀하는지 확인
- [ ] npx tsc --noEmit, npm run build 통과 확인
- [ ] 기존 활성/비활성 토글 기능 회귀 없음 확인
