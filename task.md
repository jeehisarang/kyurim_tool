# 작업 지시서 — 검사 상세 화면 "환자와 함께보기" 모드

## 배경
검사(근력검사/인바디) 상세 화면에서, 원장님이 환자와 함께 결과를 보며 상담하는
실제 워크플로우 지원. 새 창으로 열려 둘째 모니터에 띄우기 편하게 구성.

## 1. 신규 라우트: /patient-view/exam/[examType]/[id]
- 인증/사이드바 없는 완전 별도 레이아웃(내부 관리 화면과 분리된 layout)
- 기존 /examinations/[examType]/[id]의 데이터를 그대로 조회하되, 표시 항목은
  화이트리스트 방식으로 엄격히 제한:
  - 인바디: 체중, BMI, 체지방율, WHR, SMI 수치+판정(정상/근감소증 대신 부드러운
    표현으로), 측정일
  - 근력검사: 악력(좌/우/평균), 판정, 근력나이 — 이미 만들어둔
    gripAgePatientMessage() 문구 그대로 사용
  - 절대 포함 금지: 측정자 이름, 수정/삭제 버튼, 메모, 다른 검사 이력 목록,
    사이드바/내비게이션 전체
- 큰 글씨, 여유로운 여백, 청자색·한지 톤 유지 — 상담 중 화면 너머로 봐도
  잘 보이는 크기로 (제목/핵심수치는 특히 크게)
- 하단에 원장 전용 "직원화면으로 돌아가기/닫기" 버튼 (작게, 눈에 안 띄게)

## 2. 진입 방식
- 기존 /examinations/[examType]/[id] 상세 화면에 "환자와 함께보기" 버튼 추가
- 클릭 시 window.open()으로 새 창/탭에 /patient-view/exam/[examType]/[id] 오픈
  (팝업 차단 관련 안내 필요시 문구 추가)
- 새 창은 브라우저 기본 창 크기로 열리되, 사용자가 자유롭게 둘째 모니터로
  드래그해서 쓸 수 있으면 충분 (특수 fullscreen API 강제는 불필요)

## 3. 재사용을 위한 구조화
- 화이트리스트 필터링 로직(examination-format.ts 활용)과 레이아웃 컴포넌트를
  분리해서, 향후 환자통합프로필에도 동일 패턴("/patient-view/profile/[patientId]"
  형태)으로 확장 가능하게 준비 (이번 단계에서 프로필 페이지 자체는 만들지 않음)

## 검증
- 근력검사 상세에서 "환자와 함께보기" 클릭 → 새 창에 측정자/수정삭제/메모 없이
  악력+판정+근력나이(환자용 문구)만 큰 글씨로 표시되는지 확인
- 인바디도 동일하게 확인
- 새 창을 닫아도 원래 직원화면(원본 탭)에는 영향 없는지 확인
- 팝업 차단된 브라우저 환경에서 안내 문구 뜨는지 확인

## 검증
- 실데이터 환자(예: 최서현, 강래성 등 진행중 프로그램 있는 환자)로 재현:
  내원체크 목록 → 환자명 클릭 → 통합 프로필 화면 정상 이동, 진행중 프로그램/
  검사이력 요약 정상 표시 확인
- 내원체크 목록에 진행중 프로그램 배지 정상 노출 확인 (프로그램 없는 환자는
  배지 없이 정상 표시)
- 기존 검사/수정/삭제 버튼 등 기존 기능 회귀 없음 확인
- /examinations/patient/[patientId] 페이지 자체는 변경 없이 그대로 링크만
  연결되는지 확인
