# 작업지시서 — HRV 상세지표 시각화 (막대+참고범위) + 이상치만 코멘트

## 배경
- 기존엔 HRV 상세지표(TP/VLF/LF/HF/LF·HF비율/SDNN/RMSSD)가 AI 코멘트에서 줄글로
  나열되거나(GPT 스타일) 아예 생략되고 있었음.
- 이번에 "그래프로 한눈에 보이게 + 정상은 조용히, 참고범위 벗어난 것만 짧은 코멘트"
  방식으로 확정. **원장용/환자용 화면 모두 동일하게 노출.**
- 이 지표들의 "참고범위"는 학술 표준이 아니라 기기 제조사 해석 레이어라는 기존 원칙
  (7/17 확정)은 그대로 유지 — 표현은 "정상범위"가 아니라 "참고범위"로 통일.

## 1. 참고범위 상수 정의 (확인 필요)
아래 수치는 이번에 참고한 자료(GPT 분석, 출처 불명확) 기준입니다. 실제 구현 전에
**기존에 이미 저장된 참고범위 값(설정 > 검사 학술근거 관리 등)과 겹치는 게 있는지
먼저 확인**하고, 있다면 기존 값을 우선 사용할 것. 없다면 아래 값을 임시로 넣되
원장 검수 대기 항목으로 별도 표시.

| 지표 | 참고범위 | 우리말 부제 |
|---|---|---|
| 맥박변화도(Pulse Diversity) | 30~40 | 심박 리듬 다양성 |
| Total Power | 7.4~9.3 | 전체 활동량 |
| VLF | 6.8~8.8 | 장기 회복력 |
| LF | 6.1~8.1 | 자율조절 능력 |
| HF | 4.2~7.4 | 휴식·회복 능력 |
| LF/HF 비율 | 0.4~2.2 | 교감·부교감 균형 |
| SDNN | 31.2~100.9 | 전반적 회복력 |
| RMSSD | 16~74 | 부교감 회복능력 |

## 2. 컴포넌트 신규 개발
- `HrvDetailIndicatorChart` (가칭) — 위 8개 지표를 각각 가로 막대(음영 구간=참고범위,
  점=현재 측정값)로 렌더링. 값이 참고범위 안이면 점은 파란색, 범위 밖이면 주황색
  + 막대 바로 아래 1줄 코멘트("참고범위(X~Y)보다 다소 [높은/낮은] 편으로
  확인됩니다") 노출. 범위 안이면 코멘트 없음(침묵).
- 코멘트 판정은 **코드 레벨 결정론적 계산**(AI 아님) — 기존 `computeNotableChanges`와
  동일한 원칙. `isWithinReferenceRange(value, min, max)` 유틸 함수로 구현.
- 상단에 종합판정 4개 카드(스트레스지수/혈관건강지수/혈관건강타입/자율신경경향도)
  — 기존 값 그대로, 카드 형태로만 재배치.
- 하단 캡션 고정: "음영 구간 = 기기 제조사 참고범위, 점 = 현재 측정값" (+ 필요시
  기존 안전 문구 "이 해석은 기기 수치만으로 확정한 것이 아니라~" 재사용)

## 3. 적용 범위 — 원장용/환자용 공통
- 원장 확인화면(`/examinations/hrv/[id]`)과 환자용 화면(환자와 함께보기 팝업,
  `/s/[token]`, `/p/[token]`) **양쪽 모두에 동일한 컴포넌트로 렌더링**.
- 이 컴포넌트는 doctorText/patientText 분리 대상이 아님 — 그래프+범위 이탈 코멘트는
  전문용어가 없으므로 원장/환자 구분 없이 공통 사용 가능. (참고: 카드7의 doctorText/
  patientText 분리와는 별개 트랙)
- 카드4(한의 건강해석) 안에, 기존 도넛/막대(카테고리 비중) 아래에 이 컴포넌트를
  추가로 배치. 기존 텍스트 서술(자율신경균형도 판독 등)은 그대로 유지, 상세지표
  나열 줄글만 이 시각화로 대체.

## 4. 레거시 처리
- `HrvTestRecord`에 이미 저장된 7개 상세지표 컬럼을 그대로 사용 — 스키마 변경 없음.
- 상세지표 컬럼이 없는(구버전) 레코드는 이 컴포넌트 자체를 조건부로 숨김(기존
  "상세지표 있을 때만 노출" 패턴 재사용, 에러 아님).
