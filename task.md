# 작업지시서 — 검사톡 UX 개선 3건 + 건강 리포트 다운로드 기능

## 배경
- 검사톡(/s/[token])이 정상 작동 확인됨(7/17 건강리포트 신버전 + 7/16 구버전 병행 렌더링
  정상, aiCommentaryVersion 분기 회귀 없음).
- 실사용 화면 검토 중 발견된 개선 필요 사항 3건 + 원장 확인화면 다운로드 기능 1건을
  한 번에 정리.

---

## PART A — 이전 검사기록 접기/펼치기 (스크롤 단축)

### 사전확인 체크리스트 (설계 승인 완료)
- [x] 적용 범위: HRV + 인바디 + 근력검사 전체 동일 적용
- [x] 기본 노출: 최신 1건만 펼쳐진 상태로 노출
- [x] 이전 기록: 접힌 상태(제목/날짜/핵심 요약 정도만 보이는 헤더)로 두고,
      클릭 시 펼쳐지는 아코디언 방식

### 동작 규칙
- 검사종류별로 독립적으로 적용 (예: HRV 2건 + 인바디 3건이 함께 있으면,
  HRV 최신 1건 펼침 + 나머지 접힘, 인바디도 각각 별도로 최신 1건만 펼침)
- 접힌 항목 헤더에 최소 정보 표시: 검사일자 + 핵심 수치 요약(기존
  `hrvSummaryLabel` 등 기존 요약 헬퍼 재사용)
- 접힘/펼침은 페이지 내 상태(새로고침 시 초기화, 서버 저장 불필요)
- 기존 앵커 네비게이션(상단 이동 링크)과 충돌 없는지 확인 — 접힌 상태에서
  앵커 클릭 시 자동으로 펼쳐지며 스크롤 이동되어야 함

## PART B — 상담설문 섹션, 검사톡 링크에서 숨김

### 사전확인 체크리스트 (설계 승인 완료)
- [x] 숨김 기준: **검사기록이 하나라도 포함된 링크는, 티칭/이벤트와 묶여 있어도
      상담설문 섹션을 항상 숨김**
- [x] 검사기록이 없는 링크(티칭만/이벤트만/통합이지만 검사 미포함)는 기존대로
      상담설문 섹션 그대로 노출

### 동작 규칙
- `/s/[token]` 렌더링 시, 해당 링크에 연결된 검사기록(HRV/인바디/근력) 존재
  여부를 조건으로 상담설문 섹션 렌더링 여부 결정
- 섹션 순서 로직(검사결과→상담설문→티칭→이벤트) 자체는 유지하되, 검사기록
  포함 시 상담설문 자리만 건너뜀
- 기존에 이미 생성되어 발송된 링크에도 소급 적용됨(정적 스냅샷이 아니라
  매번 서버에서 렌더링하는 구조이므로 재생성 없이 자동 반영, 이 가정이
  맞는지 코드 확인 후 진행)

## PART C — "상담예약하기" CTA 버튼 (페이지 하단)

### 사전확인 체크리스트 (설계 승인 완료)
- [x] 위치: 페이지 맨 하단, 기존 "프로그램문의하기"/"이벤트문의하기"와 동일한
      위치·스타일
- [x] 노출 조건: 검사기록이 포함된 링크에서 노출 (티칭/이벤트만 있는 기존
      링크는 기존 버튼 그대로 유지, 신규 버튼과 중복 노출되지 않게 분기)
- [x] 문구: 버튼 위에 "내원하셔서 보다 자세한 상담 도와드리겠습니다" 안내문구,
      버튼 라벨 "상담예약하기"
- [x] 동작: 기존 `requestConsultCallback()` / `requestEventInquiryCallback()`과
      동일한 패턴 — 클릭 시 카카오채널 채팅 오픈 + 업무(TodoTask) 자동 생성 +
      버튼이 "카카오톡으로 상담 가능하십니다" 안내문구로 교체

### 동작 규칙
- 신규 함수 `requestExamConsultCallback()` (가칭) 신설, 업무 제목은 기존
  네이밍 규칙 따라 "검사상담 요청" 정도로 통일 (원장님 선호 문구 있으면 반영)
- 담당자 지정 없이 전체공통, 당일+동일환자 중복방지 로직은 기존
  이벤트문의하기와 동일하게 적용
- 통합 링크(검사+티칭+이벤트 모두 포함)인 경우, 기존 프로그램/이벤트문의
  버튼과 함께 상담예약하기 버튼도 나란히 노출 (배타적 아님, 병렬)

## PART D — 건강 리포트 PDF/이미지 다운로드 (원장 확인화면)

### 사전확인 체크리스트 (설계 승인 완료)
- [x] 형식: PDF + 이미지(PNG) 둘 다, 사용자 선택 가능
- [x] 버튼 위치: 원장 확인화면(`/examinations/hrv/[id]`)에만 추가

### 동작 규칙
- "다운로드" 버튼/아이콘 → PDF/이미지 선택 모달 또는 드롭다운
- 다운로드 대상: "환자와함께보기"에 표시되는 최신 건강 리포트 7카드 콘텐츠
  전체 (레거시 4섹션 레코드는 그 구조 그대로 다운로드 대상)
- 파일명 규칙 제안: `[환자이름]_건강리포트_[검사일자].pdf` / `.png`
  (원장님 선호 형식 있으면 반영)
- 비활성(소프트삭제) 레코드는 다운로드 버튼 비노출
