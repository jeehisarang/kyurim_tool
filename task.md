# 작업 지시서 — 증상 패턴 체크리스트 "질문 관리" 화면 신설 (원장 전용)
날짜: 2026-07-20

## 배경
- 50대 후반 남성 환자가 빈뇨/야간뇨(전립선 관련, 양허 증상) 호소했으나
  현재 "기력·냉증" 카테고리 체크리스트에 해당 질문이 없어 놓침
- 원장이 직접 카테고리 안에 질문을 추가/수정/삭제할 수 있는 화면이 필요
- 카테고리(7개: 스트레스·정서긴장/기력·냉증/열감·건조/소화기/담습·부종/
  혈허 경향/순환·어혈) 자체는 이번 범위에서 고정. 카테고리 안의 질문만 관리 대상.

## 범위 (이번에 하는 것)
1. 신규 설정 화면: `/settings/tcm-checklist-questions`
   - 원장 계정만 접근 가능 (기존 권한 체크 방식 그대로 재사용 — 직원 계정으로
     로그인 시 접근 차단 또는 메뉴 자체 미노출, 기존 설정 메뉴 권한 분기 로직 확인 후 동일 패턴 적용)
   - 7개 카테고리를 섹션으로 구분해서 표시 (카테고리명 헤더 + 그 안의 질문 리스트)
   - 각 카테고리 섹션 하단에 "질문 추가" 입력창 (텍스트 1줄 + 추가 버튼)
     → 추가 시 해당 카테고리 내 마지막 순서(order max+1)로 저장
   - 각 질문 항목: 텍스트 인라인 수정(클릭해서 편집 → 저장), 삭제 버튼(확인 모달 필수)
   - 삭제는 하드 삭제로 처리 (과거 응답은 `TcmChecklistAnswer`에 답변 당시 텍스트가
     스냅샷으로 남는 구조인지 반드시 먼저 확인 — 스냅샷이 아니라 FK 참조 방식이면
     삭제 시 과거 리포트 깨질 수 있으니, 그 경우 소프트 삭제(활성/비활성 플래그)로
     방식 변경해서 진행. 확인 결과와 실제 적용한 방식을 보고에 반드시 명시)
   - 질문 순서 조정 UI(드래그 등)는 이번 범위 밖 — "끝에 추가"만 지원
   - 카테고리 추가/삭제/이름변경은 이번 범위 밖

2. 반영 확인 (새 로직 개발 아님, 기존 로직이 자동 반영하는지 검증)
   - 새로 추가한 질문이 아래 3개 입력 경로 전부에서 노출되는지 확인:
     a. 독립 메뉴 "상담설문" (`/consultation-survey`)
     b. 검사등록화면(HRV 등) 바로가기
     c. `/consult-mode`(상담모드) "증상 패턴 체크" 드롭다운
   - 카테고리 점수 계산(문항점수합÷(문항수×2))이 새 문항 수를 자동으로 반영하는지 확인
   - 건강 리포트 카드2(선택 증상)에 새 질문이 체크됐을 때 원문 그대로 노출되는지 확인
   - 질문 삭제 후 과거에 이미 생성된 건강 리포트가 깨지지 않는지 확인 (실제 과거
     리포트 하나를 다시 열어서 확인, 캡처 또는 결과 서술로 보고)

3. 이번 세션에 바로 반영할 실사용 질문
   - 카테고리: 기력·냉증
   - 질문 문구: "소변을 자주 보시거나 밤에 자주 깨어 화장실에 가시나요? (남성의 경우
     전립선 관련 증상 포함)"
   - 새로 만든 관리화면을 통해 추가 (수동 스크립트 아닌 실제 UI 경로로 추가해서
     화면 자체도 함께 검증)

## 하지 않는 것 (범위 밖)
- 카테고리 자체 추가/삭제/이름변경
- 질문 순서 드래그 조정
- 질문에 대한 버전 관리/이력 추적
- 카드3/카드4/카드6 등 카테고리 단위 로직(레드플래그 문구, 변증 매핑, 치료원칙) 신규 작성
  — 이건 카테고리 단위라 이번 범위 밖, 그대로 유지

- 새 질문 체크 후 건강 리포트 재생성 → 카드2에 정상 노출, 카테고리 점수 계산에
  문항수 반영되는지 확인
- 질문 삭제 후 과거 리포트 재확인 (스냅샷 여부에 따라 안전한 삭제 방식 최종 확인)
- npx tsc --noEmit, npm run build 통과 확인
- 테스트 데이터 정리 확인
