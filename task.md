# 작업 지시서 — 체험 추천 적립금 "최대/확정" 이원화

## 배경
지금은 신청서 제출 즉시(ReferralCreditEntry TRIAL_SIGNUP) 5,000원이 확정
적립되는 구조. 하지만 신청만 하고 실제로 내원해서 3일체험을 받아가지 않은
사람도 있어 실제 사용 가능한 금액과 괴리가 생김.

## 결정된 방향
- "쌓이는 재미"(최대 적립금)는 그대로 유지 — 신청 즉시 반영되는 값은 계속
  보여줌
- 대신 "실제 사용 가능한 금액"(확정 적립금)을 별도로 추가 — 신청 목록에서
  직원이 "이 신청으로 3일체험 등록"을 눌러 실제 Prescription을 생성하는
  시점에만 확정
- 확정 시점은 추천링크의 7일 유효기간과 무관 (신청 자체가 유효기간 내에
  들어왔으면, 확정은 그 이후 아무 때나 일어나도 됨 — 예: 신청은 7일째,
  실제 내원은 9일째)
- 예시: 신청 5건(건당 5,000원) → 최대 적립금 25,000원, 그중 2건만 실제
  등록됨 → 확정 적립금 10,000원. 실제 사용 가능한 건 확정 적립금뿐

## 스키마 변경
- `ReferralCreditEntry`에 상태 개념 추가 필요:
  - 방법 A) `status` 필드 추가 (PENDING | CONFIRMED), 신청 시 PENDING으로
    생성, 실제 Prescription 연결 시 CONFIRMED로 전환
  - 방법 B) 신청 시점엔 크레딧 엔트리를 만들지 않고, TrialApplication에
    "최대 적립 반영 대상"임을 카운트로만 잡다가, 확정 시에만
    ReferralCreditEntry 생성 — 이 경우 "최대 적립금"은 ReferralCreditEntry가
    아니라 TrialApplication 건수 × TRIAL_REFERRAL_BONUS_AMOUNT로 매번 계산
  - Claude Code가 기존 코드 구조 보고 더 자연스러운 쪽으로 판단해 진행할 것
    (기존 마이그레이션/집계 로직 영향 최소화하는 방향 우선)

## 로직 변경
1. 신청 제출(`/refer/trial` 제출) 시점: 기존처럼 즉시 크레딧 반영하되, 이건
   "최대 적립금" 집계에만 포함되고 "확정 적립금"에는 포함 안 됨
2. 직원이 신청 목록(`/refer/applications`)에서 "이 신청으로 3일체험 등록" 실행
   → 실제 Prescription 생성 + `convertedPrescriptionId` 연결 → 이 시점에
   해당 신청 건의 크레딧을 "확정" 상태로 전환(또는 신규 생성)
3. 신청이 3일체험 등록으로 이어지지 못하고 그냥 남아있는 건(연락 안 됨, 취소 등)은
   계속 "최대 적립금"에는 남아있지만 "확정 적립금"에는 영원히 포함 안 됨 (별도
   만료/취소 처리 로직은 이번 범위에 넣지 않음 — 필요성 보이면 추후 별도 지시)

## 화면 변경
### `/refer/my/[token]` (환자가 보는 화면)

최대 적립금 25,000원 (신청 5건)
*실제 체험 진행 시 확정되며, 확정 적립금만 본프로그램 신청 시 사용 가능해요.

확정 적립금 10,000원 (확정 2건)

- 0건일 때 기존 경고배너("아직 추천으로 오신 분이 없어요...")는 최대 적립금
  기준으로 유지

### 처방 상세페이지의 "추천링크" 섹션
- 지금 "적립 현황: N건 · X원"으로 뭉쳐 있던 걸 최대/확정 두 줄로 분리 표시

### `/settings/referral-credits` (원장 전용 조회화면)
- 환자별 합계를 최대 적립금 합계 / 확정 적립금 합계 두 컬럼으로 분리
- MAIN_SIGNUP(본프로그램 추천)은 원래부터 직원 수동확정 방식이라 이번 변경과
  무관 — TRIAL_SIGNUP만 해당

## 검증
- 신청만 하고 등록 안 한 상태: 최대 적립금에는 반영, 확정 적립금에는 미반영 확인
- 신청 목록에서 3일체험 등록 실행 → 그 즉시 확정 적립금에 반영되는지 확인
- 유효기간(7일) 지난 뒤에 확정 처리해도 정상 확정되는지 확인 (신청 자체가
  유효기간 내였다면)
- 기존 테스트 데이터(임채원/이미정/권영옥 등)로 최대/확정 숫자가 실제 등록
  여부와 정확히 일치하는지 확인
- npx tsc --noEmit / npm run build 통과
