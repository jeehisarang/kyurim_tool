## 배경 (이번 대화에서 확정된 전체 설계)
- 목적: HRV 등 검사와 별개로, 환자에 종속된 독립적 증상 체크리스트("증상 패턴 프로필")를 
  만들어 AI 코멘트 생성 시 참조. "변증 진단표"가 아니라 참고용 프로필로 명명(과신 방지).
- 대상: 기본적으로 모든 환자(내과 계열) 적용
- 갱신 주기: 월 1회 권장(강제 아님), 재작성 시 이력 누적 보관, 최신 응답이 "현재 기준"
- 입력 경로: 
  (1) 원장실 — 독립 메뉴 "상담설문" + 검사 등록 화면(HRV 등)에서 환자 선택 시 바로가기
  (2) 공유링크(/s/) — 기존 검사톡/티칭/이벤트 3종 섹션에 "상담설문"을 4번째 섹션으로 추가
- 기존 tcmPatternMapJson(자유텍스트 증상기록 기반 판단): 완전 대체하지 않고 병행. 
  상담설문 응답이 있으면 우선 사용, 없으면 기존 방식 유지.

## 대분류 및 문항 (총 12문항 + 자유기록 1개)
7개 대분류, GPT 딥리서치가 제안한 학술 구조(EMOTION_STAGNATION 등)를 참고하되 
문항수는 원장실 실사용(구두 확인)에 맞게 대폭 압축함:

## 응답 방식 및 점수 계산
- 3단계: 없다(0점) / 경미하다(1점) / 심하다(2점)
- 카테고리 점수 = 해당 카테고리 문항 점수 합 ÷ (문항수 × 2) → 비율(0~100%)
- 환자 노출용 표시는 숫자 대신 3단계로만: 관련 증상 낮음 / 보통 / 뚜렷함 
  (내부적으로는 정확한 비율값 저장, 환자 화면에는 절대 숫자 노출 안 함)
- 최종 판정 문구는 "간기울결 확률 67%" 식 확정적 표현 금지, "관련 증상이 확인됩니다" 
  수준의 참고 표현만 사용

## 스키마 설계 원칙
- 지난 라운드 schema-proposal.md(90c7c87)의 5개 테이블 골자를 재사용하되:
  - 검사 종속(examRecordId) 대신 환자 종속(patientId)으로 FK 변경
  - 문항 테이블은 향후 확장을 고려해 다음 속성 포함: questionCode, patientQuestion, 
    categoryCode(위 7개 코드), weight(기본 1, 향후 조정 가능), displayOrder, isActive
  - 응답 테이블은 이력 누적 구조 (재작성 시 새 레코드 생성, deletedAt 없이 순수 append), 
    "가장 최근 것"을 현재 기준으로 조회하는 헬퍼 함수/쿼리 필요
  - 각 응답 항목 값은 0/1/2 정수로 저장 (없다/경미하다/심하다)
  - "그외" 자유기록은 별도 텍스트 필드 (점수화 테이블과 분리)
- 치료원칙 필드: 카테고리(7개)별로 텍스트 필드 추가, 초기값은 비워둠 
  (원장이 추후 직접 입력 예정 — 임의로 채우지 말 것)
