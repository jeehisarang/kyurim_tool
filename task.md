# 작업 지시서 — 프로그램 관리: 기간 단위(개월→주) 변경 + 목록에서 활성/비활성 토글

## 배경
"프로그램 관리"(/settings/program-teaching 내 또는 관련 설정 화면) 중 "새 프로그램 등록"
폼과 "프로그램 목록" 테이블 두 군데를 수정.
1. "총 기간(개월)" 입력을 "총 기간(주)"로 변경 — 자유 숫자 입력 유지, 단위만 개월→주
2. 프로그램 목록 테이블에서 "상태"(활성/비활성) 컬럼을 클릭 등으로 직접 토글 가능하게 변경
   (현재는 표시만 되고 이 화면에서 바꿀 방법이 없는 것으로 보임 — 사실 여부 확인 후 진행)

## 대상 파일
- "프로그램 관리" 화면 컴포넌트 (신규 프로그램 등록 폼 + 프로그램 목록 테이블이 있는 파일)
- 해당 폼/목록이 호출하는 API 라우트 (등록 API, 목록 조회 API, 활성/비활성 토글용 API —
  토글 API가 이미 다른 화면에 존재한다면 그대로 재사용)
- 관련 Prisma 스키마(총 기간을 개월 단위 정수로 저장하는 필드) 및 이 필드를 참조하는
  모든 로직(예: computeSplitSchedule 등 회차/기간 계산 로직) 전수 확인

## 사전 확인 사항 (작업 시작 전 먼저 조사·보고)
- 현재 프로그램 목록 화면에 활성/비활성을 바꿀 수 있는 버튼/기능이 정말 없는지, 아니면
  다른 화면(예: 치료처방 등록 화면 등)에만 그 기능이 있는지 확인
- "총 기간(개월)" 필드가 실제로 몇 군데 로직(회차 계산, 다음 처방일 계산 등)에서
  참조되고 있는지 전수 파악 — 단위를 개월→주로 바꿀 때 이 계산 로직들도 함께 맞춰야 함

## 동작 규칙

### 1) 기간 단위 개월→주 변경
- "새 프로그램 등록" 폼의 "총 기간(개월)" 라벨/입력을 "총 기간(주)"로 변경
- 입력값은 자유 숫자(제한 없음, 예: 4주/6주/8주/12주 모두 가능)
- 기존에 이미 개월 단위로 저장된 프로그램(예: 킬캡3개월=3개월, 킬캡1개월=1개월)은
  **데이터 마이그레이션으로 주 단위로 환산 저장** (1개월=4주 기준으로 환산: 3개월→12주,
  1개월→4주 등) — 절대 값 유실 없이 전수 확인 후 진행
- 회차/기간 계산 로직(2주 간격 분할처방 스케줄 등)이 이 필드를 참조하는 곳은 모두
  "개월 기준 계산"에서 "주 기준 계산"으로 함께 수정
- 화면 표시(프로그램 목록의 "총 기간" 컬럼 등)도 "N주"로 통일

### 2) 프로그램 목록에서 활성/비활성 토글
- 프로그램 목록 테이블의 "상태" 컬럼(현재 텍스트로만 "활성"/"비활성" 표시)을 클릭
  가능한 토글(버튼 또는 클릭 시 확인창)로 변경
- 클릭 시 활성↔비활성 전환, 저장 후 목록에 즉시 반영
- 비활성화 시 기존 원칙(소프트삭제, 치료처방 등록 화면 드롭다운에서 제외되지만 기존
  데이터는 보존) 그대로 유지 — 삭제 아님
- 실수 방지를 위해 비활성화 시에는 간단한 확인창("OO 프로그램을 비활성화하시겠습니까?")
  표시 권장

## 검증 체크리스트
- [ ] "사전 확인 사항" 조사 결과 보고
- [ ] 신규 프로그램을 주 단위(예: 6주)로 등록 → 정상 저장 확인
- [ ] 기존 개월 단위 프로그램들이 마이그레이션 후 올바른 주 수로 환산되어 표시되는지
      전수 확인 (킬팻캡슐/감비탕/황제감비탕/킬캡1개월/킬캡3개월 등)
- [ ] 회차 계산 로직이 마이그레이션 후에도 기존과 동일한 결과를 내는지 회귀 확인
      (기존 진행 중인 SPLIT 처방들의 다음 처방일이 변경 전과 동일하게 계산되는지)
- [ ] 프로그램 목록에서 활성→비활성, 비활성→활성 토글 모두 정상 동작 확인
- [ ] 토글 후 치료처방 등록 화면(/prescriptions/new)의 프로그램 선택 드롭다운에
      정상 반영되는지 확인 (비활성 시 제외, 활성 시 노출)
- [ ] npx tsc --noEmit, npm run build 통과

## 완료 보고 형식
- 수정된 파일 목록
- 마이그레이션 내용 및 환산 결과 (기존 20개 프로그램 각각 몇 개월→몇 주로 바뀌었는지)
- 회차 계산 로직 회귀 확인 결과
- 검증 체크리스트 결과
- 커밋 해시 (푸시는 원장님 확인 후 진행)
