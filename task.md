# 작업지시서 — 카테고리 비중 도넛/막대, 후보 카테고리 내 재정규화로 전환

## 배경
- "전체 문항 만점 대비 비중"(기타 포함) 방식이 후보 카테고리 수가 적을 때
  "기타"가 과도하게 커져(예: 46%) 1등/2등 비교의 의미가 퇴색된다는 판단.
- 대신 "후보 카테고리끼리만 합쳐서 100%로 재정규화"하는 방식으로 전환 —
  기타 없이 도넛이 꽉 차고, 카테고리 간 상대적 비중 비교는 그대로 유지됨.

## 사전확인 체크리스트 (설계 승인 완료)
- [x] "기타" 조각/범례 완전 제거
- [x] 분모를 "체크리스트 전체 만점(22점 등)"에서 "후보 카테고리 원점수 합"으로 변경
- [x] 예시(최승희 케이스): 후보 4개 원점수 합 = 4+4+2+2 = 12 →
      스트레스·정서긴장 4/12=33%, 기력·냉증 4/12=33%, 담습·부종 2/12=17%,
      순환·어혈 2/12=17% (합계 100%)

## 동작 규칙
- `buildCategoryVisualization`(hrv-health-report.ts) 계산식 변경:
  - 기존: `rawScore ÷ totalMaxScore(체크리스트 전체 만점)`
  - 변경: `rawScore ÷ sumOfCandidateRawScores(후보 카테고리들의 원점수 합)`
- `getCandidateCategoryShares`(tcm-checklist.ts)도 이에 맞게 반환값 조정
  (필요 시 "후보 원점수 합" 자체를 별도로 계산해 넘기도록)
- 도넛/막대 둘 다 이 함수 하나만 참조하는 기존 구조 유지 (계산 이원화 금지)
- 후보 카테고리가 1개뿐인 경우 → 자동으로 100%, 도넛은 단색 원 하나로 표시
  (범례도 카테고리 1개만 표시)
- 반올림 오차 처리: 퍼센트 합이 반올림으로 인해 99%/101%가 될 수 있으므로,
  기존 방식대로 자연스럽게 반올림하되 굳이 강제 보정 로직은 추가하지 않음
  (예: 33.3%+33.3%+16.7%+16.7%→반올림 시 100% 안 맞아도 무방, 근사치로 충분)
