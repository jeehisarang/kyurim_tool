# 작업지시서 — HRV 텍스트 코멘트 생성 모델 교체 (gpt-4o-mini → gpt-4.1-mini)

## 1. 배경

이미지 판독(자율신경균형도 구역 읽기) 호출은 이미 gpt-4.1(정식판)로
교체 완료됨. 이번 작업은 텍스트 코멘트 생성 호출(4개 섹션 문장을
작성하는 callHrvExplanationModel 등, 이미지 판독과는 이미 분리된 별도
함수)을 대상으로, 문구 품질(자연스러움) 개선 목적으로 gpt-4o-mini에서
gpt-4.1-mini로 교체합니다.

- 이미지 판독 호출(gpt-4.1)은 이번 작업과 무관, 건드리지 말 것
- CSV 자동임포트 경로도 동일한 텍스트 생성 함수를 재사용하고 있다면 자동
  으로 함께 적용됨 (별도 경로가 있다면 그것도 확인해서 동일하게 적용)

## 2. 변경 내용

- 텍스트 코멘트 생성 호출의 모델 문자열을 gpt-4o-mini에서
  gpt-4.1-mini로 변경 (정확한 모델 ID는 OpenAI 공식 문서 기준으로
  확인해서 적용)
- 기존 프롬프트 내용(미병 프레임, 자율신경균형도 앵커, 안전원칙 등)은
  변경하지 않음 — 모델 문자열만 교체

## 3. 검증 체크리스트

- 테스트 환자(426)로 코멘트 재생성해서 4개 섹션 모두 정상 생성되는지 확인
- 기존 안전장치들이 새 모델에서도 정상 작동하는지 확인:
  - 패턴명 오탐지 재시도/실패처리 (증상기록 없는 케이스)
  - 마커 파싱([기기판독]/[임상적의미]/[생활관리]/[한의학적해석] 등
    4개 마커)이 새 모델 출력에서도 깨지지 않는지 — 모델이 바뀌면 출력
    포맷이 미묘하게 달라질 수 있어 파싱 실패 여부를 특히 주의 깊게 확인
- 이전(gpt-4o-mini) 코멘트와 비교했을 때 문구가 실제로 더 자연스러워졌는지
  주관적으로도 확인해서 완료보고에 예시 포함
- 2회차(추이) 코멘트도 정상 생성되는지 확인
- CSV 자동임포트 경로에서도 동일 모델이 적용되는지 확인
