## Phase 2 — 마감설문 + 추천현황 배너 + 공유(복사)

### 2-1. 마감설문 페이지 (공개, 인증 없음)
- 라우트: `/refer/exit/[prescriptionId]` (3일차 마감톡에 삽입되는 링크,
  기존 forms.gle 링크 대체 — `TRIAL_DEADLINE_SURVEY_LINK` 환경변수 참조하던
  부분을 이 신규 라우트로 교체)
- 상단에 "내 추천 현황" 배너:
  - 해당 처방의 환자 명의 ReferralLink(TRIAL) 토큰 기준으로
    ReferralCreditEntry(TRIAL_SIGNUP) 건수/합계 조회
  - 0건이면: 경고색 강조 배너로 "아직 추천으로 오신 분이 없어요. {만료일}까지
    이 링크로 친구가 신청하면 적립금이 쌓여요" 문구
  - 1건 이상이면: "진행중 인원 N명 / 적립금 X원" 2분할 카드
  - 공통: "이 코드는 회원님만의 고유 코드이며, 개인정보가 없어 SNS·단톡방에
    자유롭게 공유해도 안전합니다" 안내 문구
  - 링크 복사 버튼 (기존 execCommand 폴백 공용 유틸 재사용 — LAN 환경 클립보드
    이슈 이미 해결된 패턴 그대로)
  - 카카오 공유 버튼은 Phase 4에서 추가(이번 Phase에서는 자리만 비워두거나
    숨김 처리)
- 이름 필드는 해당 Prescription의 환자명을 자동으로 채우고 수정 불가(readOnly)
  처리
- 나머지 4문항(복용여부/변화/상담희망/한줄소감)은 기존 구글폼 문항 그대로
- 제출 시:
  - ExitSurveyResponse 생성
  - consultInterest가 "네" 또는 "고민중"이면 기존 `requestConsultCallback()` 과
    동일 패턴으로 "본상담 예약 요청" WORK 업무 자동 생성(당일+동일환자+미완료
    중복방지 조건도 동일하게 적용)
- 완료 화면에도 동일한 추천현황 배너를 다시 한 번 보여줄 것 (제출 전/후 동일 정보,
  제출 후엔 "3만원 할인이 본프로그램 신청 시 적용됩니다" 안내 추가)

### 2-2. 2일차톡에 추천링크 자동 삽입
- 기존 2일차톡(AI 생성 로직)의 최종 발송 텍스트 뒤에, 고정 문구 블록 추가:

🎁 3일체험, 주변에도 추천해보세요!
아래 링크로 신청하시는 분이 생기면 5,000원씩 적립해드려요.
적립금은 나중에 킬팻캡슐 본프로그램 신청하실 때 사용하실 수 있어요.

👉 내 추천링크
{SHARE_BASE_URL}/refer/trial/{token}

({만료일}까지 신청 건에 한해 적립됩니다)

- 해당 환자의 활성 ReferralLink(kind=TRIAL, isActive, 미만료) 토큰 조회해서 삽입,
  없으면(체험 프로그램이 아닌 다른 톡이면) 이 블록 자체를 넣지 않음
- 웰컴톡(1일차)/마감톡(3일차)에는 넣지 않음 (기존 결정 유지 — 마감톡은 대신
  2-1의 마감설문 링크로 교체)

### 검증 (Phase 2)
- 실제 3일체험 환자로 2일차톡 생성 시 추천링크 블록 정상 삽입 확인
- 마감설문 진입 시 이름 자동채움+readOnly, 추천현황 배너(0건/N건) 각각 정상 표시
- 마감설문 제출 시 ExitSurveyResponse 저장 + 상담희망 시 WORK 업무 생성 확인
- 복사 버튼 정상 동작(LAN 환경 포함)
- npx tsc --noEmit / npm run build 통과
