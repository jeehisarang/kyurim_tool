# 작업 지시서 — 홈 화면 "오늘 할 일" 완료 항목 정리 UX 개선

## 배경 / 목적
현재 홈 화면에서 완료된 할 일 항목이 흐린 글씨로 미완료 항목과 함께 계속 나열되어
"오늘 할 일을 다 끝냈다"는 느낌이 들지 않음. 완료 항목을 접어두고, 진행 현황을
더 명확한 문구로 표시하며, 전부 완료 시 별도 안내를 보여주도록 개선.

## 변경 대상
`src/app/home/page.tsx` (및 필요시 `src/components/TodoTaskTable.tsx` 공유 컴포넌트).
`/todo` 화면의 동일 영역도 같은 방식으로 통일할지는 아래 참고사항 확인 후 판단.

## 1. 완료 항목 접기
- "오늘 할 일" 목록에서 완료된 항목은 기본적으로 숨김 처리
- 목록 하단(또는 상단)에 "완료된 항목 보기 (N건)" 토글 버튼 추가
- 토글 클릭 시 완료 항목이 펼쳐지고, 다시 클릭하면 접힘
- 미완료 항목은 항상 기본으로 노출 (숨김 대상 아님)

## 2. 진행 현황 문구 명확화
- 기존 "N/M건" 형태의 진행률 바 라벨을 더 풀어서 표기
  예: "오늘 할 일 처리 현황: M건 중 N건 완료" 형태로 변경
- 진행률 바 자체(시각적 바)는 유지, 텍스트 라벨만 보강

## 3. 전체 완료 시 안내 배너
- 미완료 항목이 0건인 경우, 목록 대신(또는 목록 위에) 완료 안내 배너 표시
  예: "오늘 할 일을 모두 처리했습니다 🎉"
- 이 경우에도 완료 항목 전체는 "완료된 항목 보기" 토글을 통해 확인 가능해야 함
  (완전히 숨겨서 안 보이게 하면 안 됨 — 필요시 다시 열람 가능해야 함)

## 4. /todo 화면과의 일관성 확인
- `/todo`도 동일한 완료 항목 다수 표시 문제가 있을 수 있으므로, 이번에 만든 접기
  토글/배너 로직을 공유 가능한 형태로 구현해 `/todo`에도 적용할지 판단
  (판단이 애매하면 우선 홈에만 적용하고, 이후 별도 확인 후 `/todo` 반영 여부 결정 —
  임의로 `/todo` 동작을 크게 바꾸지 말 것)

## 5. 회귀 테스트
- 완료 항목이 있는 날짜에서 기본적으로 숨겨지고 토글로 펼쳐지는지 확인
- 완료 항목이 없는 날짜(전부 미완료)에서는 토글이 없거나 "0건"으로 비활성 표시되는지 확인
- 전체 완료 시 안내 배너가 뜨고, 그 상태에서도 완료 항목 열람 가능한지 확인
- 완료체크/톡관리 버튼 동작 자체에는 영향 없는지 확인 (표시 방식만 변경)
- npm run build 통과 확인

## 완료 후 보고

- 실제 화면에서 접기/펼치기, 완료 배너가 어떻게 보이는지 텍스트로 요약
- /todo에도 적용했는지 여부와 이유
- npm run build 결과
