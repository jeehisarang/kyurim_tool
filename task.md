[작업지시서] 환자 상담모드 설문 — 스키마 설계 v2 + 구현 (환자 종속, 검사 비종속)

## 배경
- 지난 라운드(schema-proposal.md, 90c7c87)에서 "검사에 종속된 체크리스트" 구조로 
  설계 제안했으나, 원장 판단으로 방향 전환: 체크리스트는 검사(HRV 등)가 아니라 
  "환자"에 종속되는 독립적인 상담 기록으로 설계함.
- 이유: 맥파기(HRV)는 내과계열 1회성 검사인 반면, 변증 체크리스트는 환자의 지속적인 
  증상 프로필에 가까움. 검사 재실시 때마다 같은 질문을 반복할 필요 없이, 상담설문은 
  독립적으로 누적되고 여러 검사(HRV, 향후 인바디 등)의 AI 코멘트 생성 시 공통 참조됨.
- 최종 결정 사항:
  1. 대상: 기본적으로 모든 환자 적용(내과 계열)
  2. 갱신 주기: 월 1회 권장(강제 아님), 재작성 시 이력으로 누적 보관, 최신 응답이 
     "현재 기준"으로 AI 코멘트에 반영됨
  3. 원장실 입력 진입점: 독립 메뉴 "상담설문" + 검사 등록 화면(HRV 등)에서 환자 선택 시 
     바로가기 버튼
  4. 공유링크(/s/) 지원: 기존 검사톡 공유링크 시스템에 "상담설문"을 4번째 섹션 타입으로 
     추가 (기존 검사결과→프로그램티칭→이벤트 순서에 자연스럽게 편입)
  5. 점수 계산: 비율 방식 (체크한 문항 수 ÷ 해당 변증 전체 문항 수)
  6. 기존 tcmPatternMapJson: 완전 대체하지 않고 병행 — 상담설문 응답이 있으면 우선 사용, 
     없으면 기존 자유텍스트 증상기록 기반 판단 방식 유지
