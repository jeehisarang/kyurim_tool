# 작업지시서 — 건강 리포트 "치료 방향 & 생활관리" 섹션 카드형 재구성

## 배경
- 최승희 팀장 셀프테스트에서 발견: 여러 카테고리(4개)가 해당될 경우
  "치료 방향 & 생활관리" 섹션이 하나의 긴 문단으로 뭉쳐서 나와 상담 중
  가독성이 떨어짐 (어느 부분이 어느 증상 얘기인지 구분 안 됨).
- 원장 확인 결과: AI 재구성(다듬기)은 유지하되, 카테고리별로 분리된
  카드 형태로 각각 독립적인 문단을 갖도록 변경하기로 결정.

## 사전확인 체크리스트 (설계 승인 완료)
- [x] 표시 방식: 아코디언(클릭 펼침) 아님 — 전부 펼쳐진 카드형으로,
      상담 중 스크롤만으로 다 보이게
- [x] 문구 생성: AI가 카테고리별로 각각 다듬어서 작성 (원문 그대로 노출 아님)
- [x] 생활관리(공통 안내)는 기존처럼 마지막에 한 번만 별도 문단으로 유지

## 동작 규칙
- 해당 리포트의 활성 카테고리(예: 스트레스·정서긴장/기력·냉증/담습·부종/
  순환·어혈) 각각에 대해 독립된 카드 생성:
  - 카드 제목: 카테고리명 (예: "스트레스·정서긴장")
  - 카드 본문: 그 카테고리의 `treatmentPrinciple`을 AI가 환자용으로
    다듬은 1~2문장 (기존처럼 여러 카테고리를 한 문단으로 잇지 않고,
    카테고리 하나당 AI 호출 결과도 독립적으로 생성 — 프롬프트에서
    "다른 카테고리 언급 없이 이 카테고리만 다뤄라" 명시)
- 마지막에 공통 생활관리 문단(규칙적 생활, 스트레스 관리법 등) 1개 추가
  (기존과 동일한 자리, 이번 케이스에서는 문제 없었으므로 유지)
- 카드 스타일: "내가 선택한 증상"/"위험신호 안내" 등 기존 섹션과 톤 통일
  (색상 코딩 불필요, 단순 구획 분리면 충분)
- 카테고리명에 붙는 별표(`*카테고리명*`) 마크다운 렌더링 버그(이전 지시서
  건)도 이 작업 중 자연스럽게 해소되는지 함께 확인 — 카드 제목으로
  분리되면 아예 텍스트 안에 별표를 쓸 필요가 없어지므로

## 추가 — 팝업 차단 경고 수정 (경미)
- 다운로드 버튼이 새 창을 여는 방식이라 브라우저 팝업 차단에 걸림
  ("팝업이 차단되었습니다" 메시지 노출)
- 새 창 띄우기 대신, 같은 페이지 내에서 즉시 다운로드(blob URL + 자동
  클릭 트리거) 방식으로 변경 — 팝업 차단과 무관하게 동작하도록

## 검증 체크리스트
- [ ] 테스트환자(426)로 4개 카테고리 조합 재현, 카드별로 독립 문단
      생성되는지 확인 (카테고리 간 내용 섞임 없는지)
- [ ] 카테고리 1개만 해당하는 경우에도 정상 동작(카드 1개 + 생활관리)
- [ ] 카테고리명에 별표 등 마크다운 깨짐 없이 정상 렌더링 확인
- [ ] 다운로드 시 팝업 차단 경고 없이 정상 다운로드되는지 확인
- [ ] 기존 실사용 리포트(최승희 등) 재조회 시 캐시된 옛 문단형 텍스트가
      남아있는지, 재생성이 필요한지 확인 후 보고 (재생성은 원장 승인 후)
- [ ] npx tsc --noEmit / npm run build 통과
