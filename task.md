# 작업 지시: 오늘 할 일 - 밀림/오늘 구간 분리 + 2일톡 생성 로직 진단

## 배경
- /todo가 완료 안 된 항목 전부를 하나의 리스트(현재 35건)로만 보여주고 있어,
  "오늘 진짜 처리할 것"과 "밀려서 쌓인 것"이 구분되지 않음
- 2일톡이 유독 많이 쌓여있는데, 이게 (a) 정상적으로 밀린 것인지 (b) 생성 조건
  자체가 잘못돼서 오래된 내원 건까지 계속 재생성되는 버그인지 먼저 진단 필요

## 1. 진단 먼저 (코드 수정 전에 결과부터 보여줘)
현재 35건의 TodoTask를 dueDate 기준으로 그룹핑해서 보여줘:
- taskType, patient, dueDate, 원인이 된 Visit의 visitDate(2일톡의 경우)를 같이 조회
- 특히 2일톡 항목들의 "원인 내원일자"가 며칠에 걸쳐 분포되어 있는지 확인
  (예: 전부 7/6 내원자인지, 아니면 7/1~7/6 여러 날짜가 섞여 있는지)
- 이 진단 결과를 먼저 보여주고, 아래 2번(생성 로직 수정 필요 여부)을 같이 판단해줘

## 2. 2일톡 생성 로직 점검 (talk-todos.ts)
- 원래 의도: 2일톡은 "초진/재초진 내원 다음날" 1회성 알림
- 만약 dueDate 계산 없이 "초진/재초진 이력 있음 + DAY2 MessageLog 미발송"만 보고
  무한정 재생성되는 구조라면 버그 — dueDate를 반드시 (내원일+1일)로 고정해서 저장하고,
  그 시점이 지나면 "밀린 할 일"로만 남고 재생성은 안 되도록 수정
- 정상적으로 dueDate가 찍혀있는데 단지 담당자가 체크를 안 해서 계속 미완료로 남아있는
  것이라면 이건 버그 아님 — 3번 UI 개선만으로 충분

## 3. UI: 밀린 할 일 / 오늘 할 일 구간 분리
/todo 목록을 dueDate 기준 2개 섹션으로 분리:

📌 밀린 할 일 (N건)   — dueDate < 오늘, 지난 마감일도 같이 표시 (예: "7/5 마감, 2일 지남")
📅 오늘 할 일 (N건)   — dueDate = 오늘

- 담당자 필터는 두 섹션 공통으로 적용
- 이번주 처리 현황(N/M건) 계산 방식은 유지하되, 두 섹션 존재로 인해 오해 없도록
  숫자 옆에 "(밀린 일 포함)" 등 필요시 보조 문구 추가 검토
- /home의 "오늘 할 일 미리보기"도 상위 5개 뽑을 때 "오늘 할 일" 섹션 우선순위로
  뽑히도록 반영 (밀린 일이 너무 많아 미리보기가 전부 밀린 일로만 채워지지 않게)

## 4. 검증 요청
- 진단 결과: 2일톡 35건 중 몇 건이 실제 "어제(7/6) 내원" 건이고, 몇 건이 그 이전
  날짜 내원 건인지 수치로 제시
- 버그였다면 수정 후 재확인: 오늘(7/7) 기준 2일톡이 실제로 7/6 초진/재초진 내원자
  수와 일치하는지
- /todo 화면에서 밀린 할 일 / 오늘 할 일 섹션 분리 정상 표시 확인
- npm run build 통과 확인
