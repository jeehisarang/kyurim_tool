## 작업: 톡생성기(2일차/7일차/3회차톡) AI 프롬프트에 상담기록(ConsultationNote) 반영

### 문제
- 2일차톡/7일차톡/3회차톡 AI 문구생성 시 현재 PatientNote(누적메모)만 입력재료로
  사용하고 있고, ConsultationNote(상담모드에서 기록되는 초진상담 등 상담기록)는
  전혀 참조하지 않고 있음
- ConsultationNote는 7/10 세션에서 신설된 기능이라, 그보다 먼저 만들어진
  톡생성기(2일/7일/3회차톡) 프롬프트에는 반영이 안 된 상태로 남아있었던 것
- 티칭지(ProgramTeaching) 프롬프트에는 7/11에 이미 ConsultationNote를 입력재료로
  추가한 선례가 있음 — 그때와 동일한 원칙으로 톡생성기에도 적용할 것

### 수정 대상
- `src/lib/ai-message.ts` (또는 2일차/7일차/3회차톡 프롬프트를 생성하는 실제 파일 —
  파일명이 다르다면 해당 파일 기준으로 진행)
- 메시지 생성 시 환자의 최신 ConsultationNote 1건을 조회하는 쿼리 추가
  (patientId 기준, 날짜 내림차순 1건)

### 프롬프트 수정 원칙
- 입력재료에 "최근 상담기록(ConsultationNote 최신 1건)" 추가
- 반영 규칙: 상담기록 내용이 해당 톡 맥락(2일차/7일차/3회차)과 관련 있을 때만
  자연스럽게 녹여서 반영, 무관하면 억지로 끼워 넣지 말고 생략
  (티칭지 프롬프트의 "관련성 없으면 억지로 끼워 넣지 않기" 원칙과 동일)
- ConsultationNote가 없는 환자는 기존처럼 PatientNote만으로 정상 생성되어야 함
  (필수 입력재료 아님, 있으면 참고하는 보조 재료)
- 자체검토(self-check) 항목에 "상담기록 반영 시 관련성 있는지" 확인 항목 추가

### 검증 요청
1. ConsultationNote가 있는 실제 환자로 2일차/7일차/3회차톡 각각 생성 →
   상담기록 내용이 자연스럽게 반영되는지 확인
2. ConsultationNote가 없는 환자로도 동일하게 생성 → 기존처럼 정상 동작(회귀 없음) 확인
3. 상담기록이 있지만 해당 톡 맥락과 무관한 경우 → 억지로 끼워 넣지 않고
   자연스럽게 생략되는지 확인 (티칭지 때 "엠타이트" 케이스처럼)
4. `npx tsc --noEmit` 통과 확인
5. 검증 후 테스트 데이터(생성한 테스트 톡 등) 정리
