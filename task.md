# 작업지시서 — 건강 리포트 점수 시각화 재설계 + 볼드 강조 결정론적 보강

## 배경
- 카테고리 점수 막대 시각화(커밋 3b0d6b6)가 실제로는 무의미했음 — 후보 카테고리
  선정 자체가 "동점(최대 비율) 병렬 선정" 방식이라, 화면에 뜨는 카테고리들은
  항상 막대 길이가 동일(예: 전부 100%)해서 변별력이 없음.
- 볼드 강조도 5회 중 4회 미적용으로 사실상 작동 안 함 — AI 프롬프트 지시만으로는
  신뢰 못 한다는 기존 원칙(HRV 안전문구, 패턴명 안전장치 등과 동일)에 따라
  코드 레벨 결정론적 후처리로 전환.

---

## PART 1 — 카테고리 점수 시각화 재설계

### 사전확인 체크리스트 (설계 승인 완료)
- [x] 막대 길이로 카테고리 간 비교하는 방식 폐기
- [x] 대신, 카테고리별 막대 **내부**를 "심하다"/"경미하다" 응답 비율로
      두 색구간(진한 색상 강조) 표시 — 카테고리 간 비교가 아니라 카테고리
      **내부** 심각도 구성을 보여주는 용도로 전환

### 동작 규칙
- 각 후보 카테고리의 `TcmChecklistAnswer` 응답값(0=없다/1=경미하다/2=심하다)을
  집계해, 해당 카테고리 문항 중 "심하다(2)" 비율과 "경미하다(1)" 비율을 계산
- 막대 하나를 두 구간으로 분할 렌더링: 진한 색(심하다 비율) + 옅은 색(경미하다
  비율) — "없다(0)" 응답은 막대에 반영 안 함(카테고리가 후보로 뜬 이상 0 문항은
  자연히 소수이므로 굳이 표시 안 함)
- 막대 길이 자체(전체 폭)는 모든 카테고리 동일하게 유지 — 길이 비교가 아니라
  "이 카테고리 안에서 얼마나 심한 문항이 많은지"만 보여주는 것이 목적임을
  각주에 명시
- 각주 문구 수정: 기존 "임상 기준이 아닌 체크 문항 대비 응답 비율" →
  "카테고리 간 비교가 아닌, 해당 카테고리 내 증상 심각도 분포를 나타냅니다"
  로 오해 방지 문구 보강

### 검증 체크리스트
- [ ] 테스트환자(426), "심하다" 응답 비율이 서로 다른 카테고리 조합으로
      테스트 — 막대 내부 색구간 비율이 실제 응답값과 일치하는지 확인
- [ ] 전부 "심하다"만 체크된 카테고리 (100% 진한색), 전부 "경미하다"만
      체크된 카테고리 (100% 옅은색) 각각 케이스 확인
- [ ] 1개 카테고리만 있는 경우도 정상 동작

## PART 2 — 볼드 강조 결정론적 후처리

### 사전확인 체크리스트 (설계 승인 완료)
- [x] AI 프롬프트 지시만으로는 부족(5회 중 4회 미적용 확인됨) → 코드 레벨
      후처리로 보강

### 동작 규칙
- AI 생성 텍스트에서 아래 대상 문자열을 찾아 **코드에서 결정론적으로** 볼드
  마크업(`**단어**`)을 삽입 (AI가 이미 볼드 처리했으면 중복 처리 안 하도록 체크):
  - 해당 리포트의 후보 카테고리명 전체 (예: "과로형만성스트레스", "스트레스·정서긴장" 등
    카테고리명 및 자율신경균형도 유형명)
  - 문장당 최초 등장 1회만 볼드 처리 (같은 단어 반복 등장 시 전부 볼드 처리하면
    오히려 산만해지므로)
- 기존 AI 프롬프트의 "볼드 표시" 지시는 유지하되(이중 안전장치 개념), 최종
  적용 여부는 코드 후처리가 보장
- 카드4/카드5 두 섹션 모두 동일 로직 적용

### 검증 체크리스트
- [ ] 테스트환자(426), 5회 반복 생성 후 5/5 모두 핵심 패턴명 볼드 적용
      확인(AI가 이미 볼드 처리한 경우, 안 한 경우 둘 다 결과적으로 볼드 노출)
- [ ] 이중 볼드(`****단어****`) 등 마크업 충돌 없는지 확인
- [ ] npx tsc --noEmit / npm run build 통과

- Part 2: 5회 반복 생성 결과 중 볼드 적용 확인 스크린샷 또는 텍스트 스냅샷
