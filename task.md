# 작업 지시서 — 신규 프로그램 등록 화면 해피톡 주기 옵션 확장

## 배경
현재 "프로그램 관리" 화면(치료처방 신규 프로그램 수동등록 기능)에서 해피톡 주기가
2주 / 4주(1달) 중 택1(라디오버튼)로 고정되어 있음. 이를 1주 / 2주 / 3주 / 4주
4개 중 택1로 확장 요청.

## 대상 파일
- "프로그램 관리" 화면 컴포넌트 (스크린샷 기준 "새 프로그램 등록" 폼이 있는 파일 —
  코드베이스에서 "해피톡 주기" 또는 관련 라디오버튼 UI로 검색해서 특정할 것)
- 해당 폼이 호출하는 API 라우트
- 관련 Prisma 스키마(현재 해피톡 주기를 2주/4주 값만 받는 필드/enum이 있다면 해당 부분)
- 기존 등록된 프로그램 목록/배지 등에 주기 표시가 있다면 그 부분도 확인

## 사전 확인 사항 (작업 시작 전 먼저 조사·보고)
- 이 화면에서 등록한 "해피톡 주기" 값이 실제 해피톡 발송 타이밍 계산 로직(다음 처방
  알림 등)에 연동되어 쓰이고 있는지 코드 상에서 확인할 것
- 연동되어 있다면: 1주/3주 선택 시에도 그 계산 로직이 정상 동작하도록 함께 처리
- 연동되어 있지 않다면(즉 등록 화면에만 존재하고 실제로 쓰이는 곳이 없다면): 이번
  작업 범위는 등록 화면 옵션 확장까지만 진행하고, "연동 안 됨" 사실을 완료 보고에
  명시

## 동작 규칙
1. 해피톡 주기 선택 옵션을 1주 / 2주 / 3주 / 4주(1달) 4개로 확장
2. UI는 기존과 동일한 형태(라디오버튼) 유지 — 드롭다운으로 바꿀 필요 없음, 다만
   4개가 한 줄에 들어가기 부담스러우면 줄바꿈 등 레이아웃만 자연스럽게 조정
3. 여전히 자유입력은 허용하지 않음(4개 중 반드시 택1)
4. 기존에 이미 2주/4주로 등록된 프로그램 데이터는 그대로 유지되어야 함 (마이그레이션
   시 값 손실 없어야 함)
5. 스키마 변경이 필요하다면(예: enum에 값 추가) 기존 값과 호환되는 방식으로 진행
