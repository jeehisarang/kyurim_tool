# 작업지시서 — 환자 티칭지 5단 구조화 + 검사판정 4단계 세분화

## 배경
- 기존 aiPersonalizedText는 단일 텍스트 블록으로 저장되어 화면 순서/스타일 통제가
  어렵고, 헤드라인 같은 요소를 별도로 다루지 못함
- 검사판정이 이진법(정상/약함)이라 "경계"에 있는 환자를 다 "정상"으로 뭉뚱그려
  업셀링 기회를 놓치고 있음
- 이번 작업은 향후 모든 프로그램 티칭지(강근단뿐 아니라 검사무관 프로그램 포함)의
  기본 골격이 되므로, 특정 프로그램에 종속되지 않는 범용 구조로 설계할 것

## 1. DB 스키마 변경

`PatientTeachingPage`의 `aiPersonalizedText: String` 단일 필드를 폐기하고,
아래 5개 필드로 구조화:

| 필드명 | 타입 | 설명 |
|---|---|---|
| `headline` | String | AI 자동생성 헤드라인 1개 (5~12자 내외, 후킹용) |
| `personalSubtopic` | String | 이 환자에게 해당되는 핵심 소주제 1문장 |
| `bodyText` | String | 직원 셀링포인트 기반 공감형 본문 (2~4문장) |
| `examSummary` | String? | 검사요약 (검사무관 프로그램이면 null) |
| `academicHook` | String | 원장 학술문구 기반 신뢰 마무리 1~2문장 |

- 기존 데이터 마이그레이션 불필요 (실사용 등록 0건 상태이므로 스키마만 교체)
- `/p/[token]` 렌더링 순서: headline → personalSubtopic → bodyText → examSummary
  (있을 때만) → academicHook

## 2. 검사판정 4단계 세분화 (마진 기반)

기존 이진/삼진 판정을 "약함 / 경계 / 양호 / 우수" 4단계로 확장.
`exam-thresholds.ts`에 마진 계산 함수 추가:

- **SMI (근감소증 판정)**: 성별 기준선(남 7.0 / 여 5.7) 대비 비율로 계산
  - 약함: 기준선 미만
  - 경계: 기준선 ~ 기준선×1.10
  - 양호: 기준선×1.10 ~ 기준선×1.25
  - 우수: 기준선×1.25 이상
- **악력**: 기존 연령대×성별 WEAK/NORMAL/STRONG 표의 경계값을 그대로 두고,
  WEAK/NORMAL 경계 위아래 ±5%를 "경계" 밴드로 분리 (STRONG은 "우수"로 매핑)
- 두 계산 모두 **서버가 원본 수치로 재계산**해서 저장 (클라이언트값 불신 원칙 유지)
- ⚠️ 위 비율(1.10, 1.25, ±5%)은 초안 수치이며, 실제 임상 감각과 맞는지 원장님
  1차 검토 필요 — 확정 전까지 코드 상수로 분리해 나중에 쉽게 조정 가능하게 할 것

## 3. AI 프롬프트 스펙 (5단 출력)

- 입력재료: targetSymptomKeywords, 직원 셀링포인트 7종, 원장 학술 3종, 검사 최신값+
  4단계 판정+직전 대비 변화량(있으면), ConsultationNote 최신 1건(관련부분만),
  PatientNote 누적메모
- 출력: JSON 5개 필드 (headline/personalSubtopic/bodyText/examSummary/academicHook)
- 검사 무관 프로그램: examSummary는 프롬프트에서 아예 제외 요청 (빈 문자열 유도 금지,
  필드 자체를 생성 안 하도록 지시)
- 절대 금지: 특정 한약명 노출, 과장된 효과 단정 표현("100% 낫습니다" 류),
  원장 학술문구의 "신중한 단서"(예: "~일 수 있습니다" 톤) 임의 삭제/과장
- 자체검토(self-check) 지시문 유지: 비문 여부, 5개 필드 모두 채워졌는지(검사무관 시
  examSummary 제외), 학술 문구가 본문과 자연스럽게 이어지는지

## 4. 검증 항목
- 검사 이력 有/無 각각 5필드 정상 생성 (無일 때 examSummary 없이)
- 4단계 판정 각각(약함/경계/양호/우수) 최소 1건씩 테스트 데이터로 확인
- `/p/[token]` 화면에서 5단 순서대로 시각적 구분(간격/스타일) 적용 확인
- npm run build 통과
