# HRV 검사 화면 개선 작업지시서

## 배경
오늘(7/15) 구현한 HRV 5단 구조 AI 코멘트 기능을 원장님이 실사용 테스트(환자 426 또는
테스트 환자)하면서 4가지 개선점을 발견함.

## 1. 검사 이미지 다중 업로드 (2장 고정)
- 배경: HRV 기기 출력 리포트는 항상 2장(1페이지: 결과요약, 2페이지: 상세결과 등)으로
  나오는데, 현재 업로드는 파일 1개만 선택 가능한 구조로 보임 (전태균맥파1.jpg만 저장됨)
- 요청: HRV 검사 등록 화면에서 이미지 첨부를 정확히 2장 선택/업로드 가능하도록 수정
  - 파일 선택 input을 2개로 분리하거나, multiple 첨부 후 2장까지만 허용하는 방식 중
    기존 코드 구조에 맞는 쪽으로 판단해서 진행
  - DB 스키마 확인 필요: 현재 이미지 경로 저장 필드가 1개(String)인지 확인, 1개라면
    배열/JSON 또는 필드 2개(imagePath1, imagePath2)로 확장 필요 — 마이그레이션 시
    기존 저장된 1장짜리 데이터 손실 없도록 처리
  - "환자와 함께보기" 화면에서도 2장 모두 표시되도록 수정 (탭 전환 또는 세로 나열
    중 UI에 맞는 방식으로, 세로 나열 우선 검토 — 별도 인터랙션 없이 스크롤로 둘 다
    보이는 게 가장 단순함)

## 2. 검사 이미지 확대/축소 — 마우스 드래그(팬) 미지원
- 현상: "환자와 함께보기"(/patient-view/exam/...) 화면에서 이미지 확대/축소 버튼(+/-)은
  동작하나, 확대된 상태에서 마우스 드래그로 화면 이동(팬)이 안 됨. 원장님이 진료실에서
  환자에게 확대해서 보여줄 때 필수적으로 필요한 기능
- 요청: 확대 상태일 때 마우스 드래그(모바일이면 터치 드래그도 포함)로 이미지를 이동할 수
  있도록 팬 기능 추가. 흔한 패턴(CSS transform + drag 이벤트, 또는 가벼운 라이브러리)
  중 기존 코드 스타일에 맞는 방식으로 구현

## 3. 검사 이미지 기본 표시 크기가 모니터 대비 작음
- 현상: 팝업/독립화면 진입 시 이미지가 작게 표시되어, 확대하지 않으면 리포트 안의
  작은 글씨(측정치, 판정 등)를 읽기 어려움
- 요청: 기본 표시 크기를 더 크게 조정 (화면 폭에 맞춰 꽉 채우는 정도로), 또는 진입 시
  자동으로 1단계 확대된 상태로 시작하도록 수정. 2번 팬 기능과 함께 적용해 사용성 확보

## 4. AI 코멘트(HRV 해설) 구조화 표시 + 수작업 편집 기능
- 현상: 현재 AI 생성 코멘트가 편집 불가능한 하나의 긴 줄글로 표시되어 가독성이 낮음.
  프로그램티칭(ProgramTeachingCreator, PatientTeachingPage)에서 이미 적용한 "섹션별
  카드 구분 + 필드별 수작업 편집(textarea 저장)" 패턴을 HRV 코멘트에도 동일하게 적용
- 요청:
  - HRV AI 코멘트 저장 구조 확인: 현재 DB에 단일 텍스트 필드로 저장되는지, 아니면
    이미 섹션별로 나뉘어 저장되는지 먼저 확인 후 보고
  - 화면(원장 확인용 화면 + "환자와 함께보기" 화면 둘 다) 표시를 섹션별 카드/구분선
    형태로 재구성 (예: 판정 요약 / 한의학적 해석 / 생활관리 안내 등 논리적 단위로
    시각적 구분 — 정확한 구획은 현재 프롬프트 출력 구조 확인 후 판단)
  - "환자와 함께보기" 화면은 기존 화이트리스트 방식(수정/삭제 버튼 등 내부기능 노출
    차단) 원칙 유지, 편집 기능은 원장 전용 확인 화면에만 추가
  - 안전안내(HRV_SAFETY_NOTICE) 고정 블록은 지금처럼 별도 박스로 계속 분리 유지

## 검증 체크리스트
- 2장 이미지 업로드 → 저장 → 재조회 시 둘 다 정상 표시
- 기존에 이미 저장된 1장짜리 검사기록 조회 시 에러 없이 정상 표시 (회귀 없음)
- 팬 기능: 확대 후 마우스 드래그로 이미지 이동 확인
- 기본 크기 확대 후 리포트 글씨 가독 확인
- AI 코멘트 섹션 구분 표시 확인, 원장 화면에서 편집→저장 확인
- 테스트는 테스트 전용 환자(426)로만 진행, 완료 후 테스트 데이터 정리
- npx tsc --noEmit, npm run build 통과
