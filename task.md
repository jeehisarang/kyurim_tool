# 작업 지시서 — 치료처방 환자 리스트/통계 화면 신설 + 등록 피드백 추가

## 배경
- 치료처방(구 처방등록) 메뉴가 등록 폼만 있고, 등록된 환자를 모아 보는 화면이 없음
- 처방 등록 시 성공 피드백이 없어 등록됐는지 알 수 없음
- 한 환자가 여러 프로그램(예: 킬팻캡슐 3일체험 + 감비탕)에 동시 등록되어도,
  "오늘 할 일"에는 그날 마감인 항목만 노출되므로 환자의 전체 진행 프로그램을 한눈에 볼 곳이 없음

## 1. 처방 등록 성공 피드백
- POST /api/prescriptions 성공 시 화면에 명확한 성공 표시 추가
  (예: "처방이 등록되었습니다" 토스트 또는 배너, 기존 SealStamp 컴포넌트 재사용 가능하면 재사용)
- 등록 후 폼을 초기화하거나, 방금 등록한 처방 정보를 화면에 요약으로 보여주는 것도 고려

## 2. 치료처방 환자 리스트 화면 (신규)
- /prescriptions (또는 치료처방 메뉴 하위) 에 리스트 화면 추가:
  - **전체보기**: 현재 하나 이상의 프로그램에 등록되어 진행 중인(active) 환자 전체 목록
    - 각 행: 환자명, 차트번호, **진행 중인 모든 프로그램 뱃지(복수 표시)**, 등록일, 현재 상태(차수/다음 예정일 등)
    - 한 환자가 여러 프로그램에 등록되어 있으면 뱃지 여러 개 나란히 표시 (예: [킬팻캡슐 3일체험] [감비탕])
    - 프로그램명이 길면 뱃지 내에서 축약 표시 가능 (예: "감비탕", "킬팻 3일체험" 등 — 툴팁이나 title 속성으로 전체명 확인 가능하게)
  - **프로그램별 리스트보기**: 특정 프로그램(예: 킬팻캡슐 3일체험) 선택 시 그 프로그램에 등록된 환자만 필터링해서 표시
  - 환자 클릭 또는 뱃지 클릭 시 해당 환자/프로그램의 상세 진행상태로 이동 (기존 딥링크 구조 재사용 가능하면 재사용)
- 상단에 간단한 통계 요약 카드 추가 (예: 프로그램별 현재 진행 환자 수, 이번달 신규 등록 수 등 — 기존 대시보드 통계 계산 로직 재사용 가능한 부분은 재사용)

## 3. 확인
- 테스트: 김우석 환자로 킬팻캡슐 3일체험 + 감비탕 중복 등록 후, 신규 리스트 화면에서
  두 프로그램 뱃지가 모두 표시되는지 확인
- 기존 "오늘 할 일" 화면 로직은 건드리지 않음 (그날 마감 항목만 보여주는 현재 방식 유지)

## 완료 후
- 수정/신규 파일 목록
- 등록 피드백 동작 확인 방법
- 중복 등록 환자의 다중 뱃지 노출 테스트 결과
- 빌드 통과 확인

# 작업 지시서 (이어서) — 치료처방 가상 시드 데이터 추가

## 배경
- 직전 지시서(치료처방 환자 리스트/통계 화면 신설)와 함께 진행
- 기존 prisma/seed-demo.ts(환자 60명, 내원기록 87건) 패턴을 재사용해서 치료처방 데이터도
  가상으로 채워, 신규 리스트/통계 화면이 실제 데이터가 쌓인 것처럼 보이도록 함

## 요구사항
- 기존 seed-demo.ts에 있는 가상 환자 중 일부를 골라 치료처방(Prescription) 데이터 추가 시딩:
  - 킬팻캡슐 3일체험: 5~8명 (일부는 등록만, 일부는 마감톡까지 완료된 상태로 다양하게)
  - 킬팻캡슐 3개월 / 1개월 (9-1 SPLIT): 각 3~5명, 서로 다른 차수(1차/2차/진행중 등)로 분산
  - 감비탕/황제감비탕: 3~5명
  - 환약(S환/하비환): 2~3명
  - **중복 등록 케이스 최소 2~3명 포함** — 한 환자가 2개 이상 프로그램에 동시 등록된 상태
    (방금 원장님이 테스트하신 "킬팻캡슐 3일체험 + 감비탕 중복" 같은 케이스가 시드에도 있어야
    리스트 화면에서 다중 뱃지 표시를 바로 확인 가능)
- 재실행 방어 로직은 기존 seed-demo.ts 패턴(마커 확인 후 스킵) 그대로 적용
- 명령어는 기존 db:seed-demo에 포함시키거나, 필요시 별도 스크립트(예: db:seed-prescriptions)로
  분리 — 편한 쪽으로 진행하되 어느 쪽으로 했는지 보고할 것

## 완료 후
- 시딩된 환자/프로그램 조합 요약 (몇 명이 몇 개 프로그램에 어떻게 분산됐는지)
- 신규 치료처방 리스트 화면에서 시드 데이터가 정상적으로 보이는지 확인 (전체보기 + 프로그램별 필터 둘 다)
- 중복 등록 환자의 다중 뱃지 노출 확인
