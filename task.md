# 작업 지시서 — 추천현황 페이지 분리 (/refer/my/[token])

## 배경
지금까지는 "내 추천 현황" 배너(코드/적립현황/복사/카톡공유 버튼)가 신청폼
페이지(`/refer/trial/[token]`) 상단과 마감설문 페이지에 같이 떠 있었음.
실사용 확인 결과 문제 발견:
- 2일차톡은 텍스트를 복사해서 그대로 붙여넣는 방식이라, 애초에 그 안에
  카톡공유/채널추가 같은 "버튼"을 넣을 수 없음
- 톡에 넣은 신청폼 링크(`/refer/trial/[token]`)를 환자 본인이 확인차 눌러보면,
  자기 이름으로 자기가 신청하는 화면이 보이는 부자연스러운 상황 발생
- 그 링크를 그대로 친구에게 전달하면, 친구도 원래 신청자 전용인 공유배너/
  적립현황을 불필요하게 보게 됨

## 해결 방향
페이지를 역할별로 완전히 분리:
- **`/refer/trial/[token]`**: 순수 신청폼 전용. "OOO님 추천으로 오셨네요" 배지만
  유지, 공유배너·적립현황·카톡공유버튼·채널추가버튼 전부 제거
- **`/refer/my/[token]`** (신규): "내 추천 현황" 전용 페이지. 추천코드, 적립
  건수/금액(0건일 때 경고배너 포함), 안전 안내문구, 링크 복사 버튼, 카톡 공유
  버튼을 여기로 이동. 카톡 공유 버튼 클릭 시 공유되는 링크는 본인의
  `/refer/trial/[token]`(신청폼) 주소

## 작업 범위

### 1. `/refer/my/[token]` 신규 페이지
- 라우트 신규 생성, token으로 ReferralLink 조회 (TRIAL/MAIN 공용)
- 기존 신청폼/마감설문에 있던 "내 추천 현황" 배너 UI(코드 표시, 0건/N건 분기
  문구, 안전 안내문구, 링크복사 버튼, 카톡공유 버튼)를 그대로 이 페이지로 이전
- 카톡 공유 버튼(`KakaoShareButton`)의 공유 대상 링크를 신청폼
  (`/refer/trial/[token]`) 주소로 설정 (webUrl/mobileWebUrl 둘 다, PC 버그
  수정사항 유지)
- `isPublicPatientFacing` 판단 로직에 `/refer/my/` 경로 추가 (사이드바/활동피드
  노출 방지 — 기존 `/s/` 누락 버그 전례 반드시 참고)

### 2. `/refer/trial/[token]` 페이지 정리
- 상단 "내 추천 현황" 배너 완전 제거 (추천코드 배지 "OOO님 추천으로 오셨네요"만
  유지, 이건 친구 입장에서 의미 있는 정보이므로 남김)
- 카톡공유버튼/채널추가버튼 등 신청자 본인용 요소 제거 (채널추가버튼은 신청
  완료 화면에만 있었다면 그대로 유지 — 이건 신청자 본인이 봐도 되는 요소이므로
  구분해서 남길 것)

### 3. 마감설문 페이지(`/refer/exit/[prescriptionId]`) 정리
- 상단에 직접 띄우던 "내 추천 현황" 배너를 제거하고, 대신 "내 추천 현황
  보기 →" 링크(버튼)로 교체 → 클릭 시 해당 환자의 `/refer/my/[token]`으로 이동
- 완료 화면도 동일하게 배너 대신 링크로 교체

### 4. 2일차톡 링크 교체
- 지금 2일차톡에 자동 삽입되던 링크를 `/refer/trial/[token]`에서
  `/refer/my/[token]`로 교체
- 문구도 자연스럽게 수정:

🎁 3일체험, 주변에도 추천해보세요!
아래 내 추천페이지에서 링크를 공유하시면, 신청하는 분마다 적립금이 쌓여요.

👉 내 추천 현황 보기
{SHARE_BASE_URL}/refer/my/{token}


### 5. 처방 상세페이지의 "추천링크" 섹션
- 지금 신청폼 링크(`/refer/trial/[token]`)를 보여주던 걸, 직원이 환자에게
  전달할 때는 상황에 따라 둘 다 필요할 수 있음:
  - "내 현황 페이지 링크"(`/refer/my/[token]`) — 환자에게 안내할 때 기본으로 추천
  - "신청폼 직링크"(`/refer/trial/[token]`) — 직원이 특정 지인에게 바로 신청서만
    보내고 싶을 때 쓰는 보조용
  - 두 링크 모두 표시하되, "내 현황 페이지"를 기본/강조로 배치

## 검증
- `/refer/my/[token]`에서 코드/적립현황/복사/카톡공유 정상 동작 확인, 공유되는
  링크가 신청폼 주소인지 확인
- `/refer/trial/[token]`에 배너/공유버튼이 더 이상 안 뜨는지, 추천배지만 남는지 확인
- 마감설문 페이지에서 "내 추천 현황 보기" 링크 클릭 시 정상 이동 확인
- 2일차톡 생성 시 링크가 `/refer/my/[token]`로 나오는지 확인
- 처방 상세페이지에 두 링크 모두 정상 표시되는지 확인
- 기존 테스트 데이터로 전체 플로우(수신→내현황확인→카톡공유→친구신청) 재검증
- npx tsc --noEmit / npm run build 통과
