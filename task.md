# 작업지시서 — "치료 방향" 카드 내부 키워드 불릿 분리 (고정사전 방식, AI 호출 제거)

## 배경
- 카드형 재구성(커밋 48dbc6e) 이후에도, 카드 하나 안에 3~4개 치료원칙
  키워드가 문단으로 이어져 있어 상담 중 가독성이 부족하다는 피드백.
- 결정: 이 부분은 AI가 매번 다듬지 않고, 원장이 최종 확정한 고정 설명
  사전을 그대로 노출하는 방식으로 전환 (AI 자유생성 원칙에 더 부합, 속도도
  개선, 일관성 완전 보장).
- 대표처방(시호소간산 등)은 환자용 화면에서는 노출하지 않기로 확정
  (전문용어라 혼란 유발 가능, 상담 시 원장이 구두로 설명).

## 사전확인 체크리스트 (설계 승인 완료)
- [x] 카드 본문 표시 방식: 키워드별 불릿 리스트, AI 재구성 없이 고정 텍스트
- [x] 대표처방은 카드에 노출하지 않음 (원장 확인화면에는 계속 노출 — 전체
      데이터 자체는 삭제하지 않음, 표시만 제한)
- [x] 아래 7개 카테고리 28개 키워드 설명은 확정 완료, 그대로 코드에 반영

## 고정 사전 (그대로 코드에 입력)

스트레스·정서긴장
- 소간해울·이기해울: 몸 안에 막힌 기운을 풀어 답답함을 덜어줍니다
- 청간사화: 치밀어 오르는 열감과 짜증을 가라앉혀 줍니다
- 양심안신: 마음을 안정시켜 편안하게 해줍니다

기력·냉증
- 보기익기: 부족한 기운을 채워 활력을 되찾게 도와줍니다
- 건비익기: 소화 기능을 도와 에너지 생성을 돕습니다
- 온양산한: 몸을 따뜻하게 데워 냉증을 완화합니다
- 온보비신: 몸의 근본적인 체력을 보강해줍니다

열감·건조
- 자음생진: 부족한 진액을 채워 건조함을 촉촉하게 해줍니다
- 자음청열: 몸에 쌓인 열을 내려 건조·열감을 가라앉힙니다
- 자음강화: 몸의 수분과 열의 균형을 맞춰줍니다
- 보익간신: 간과 신장의 기능을 보강해 회복을 돕습니다

소화기
- 건비화위: 비위 기능을 튼튼히 해 소화를 돕습니다
- 이기화중: 막힌 기운을 풀어 속을 편하게 합니다
- 소도화적: 정체된 음식물을 내려 더부룩함을 덜어줍니다
- 화담강역: 위로 치받는 기운을 가라앉혀 줍니다

담습·부종
- 건비화습: 소화 기능을 강화해 습기가 쌓이지 않게 돕습니다
- 화담이수: 끈적한 습기를 몸 밖으로 배출시켜줍니다
- 이수소종: 고인 물기를 내보내 부기를 가라앉힙니다
- 온양화수: 몸을 데워 수분 대사를 원활하게 합니다

혈허 경향
- 보혈양혈: 부족한 혈을 채워줍니다
- 익기생혈: 기운을 보강해 혈이 만들어지도록 돕습니다
- 양심안신: 마음을 안정시켜 편안하게 해줍니다
- 건비생혈: 소화 기능을 도와 혈이 잘 생성되게 합니다

순환·어혈
- 활혈거어: 혈액순환을 원활하게 해 어혈을 풀어줍니다
- 행기활혈: 기와 혈의 흐름을 함께 개선해줍니다
- 통락지통: 막힌 순환으로 인한 통증을 덜어줍니다
- 온경산한: 몸속 냉기를 풀어 순환을 촉진합니다

## 동작 규칙
1. 위 28개 키워드-설명 쌍을 코드 내 정적 상수(TypeScript 객체, 예:
   `TREATMENT_PRINCIPLE_KEYWORD_GLOSSARY`)로 추가. DB 스키마 변경 불필요
   (기존 `TcmCategory.treatmentPrinciple` 필드는 그대로 두되, 카드 렌더링
   시에는 참조하지 않음 — 원장 확인화면/설정화면에서는 기존 필드 그대로 유지)
2. 카드 렌더링 로직 변경: 기존처럼 AI를 호출해 문단을 생성하지 않고,
   해당 카테고리에 매핑된 키워드 배열을 순회하며 불릿으로 출력
   - 카드 제목: 카테고리명
   - 카드 본문: 키워드별 불릿 (`키워드 — 설명` 형식)
3. 사전에 없는 키워드가 향후 추가될 경우(예: 카테고리 확장 시) 대비해,
   매핑 안 된 키워드는 콘솔 경고만 남기고 화면에는 키워드명만 노출(설명
   없이) — 조용히 빠지는 것보다 눈에 띄게 처리
4. 마지막 "생활관리" 공통 카드는 기존처럼 AI 생성 유지 (이번 변경 대상 아님)
5. AI 호출 제거로 인한 부수효과 확인: 카드 생성 속도 개선 여부, 기존
   "치료원칙 미입력 시 카드 생성 안 함" 안전장치는 계속 유지(사전에 없는
   게 아니라 카테고리 자체에 키워드가 비어있으면 카드 자체를 스킵)
