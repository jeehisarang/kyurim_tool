# 작업 지시: 7일톡 스킵 + 환자 키워드 메모 누적 구조 + AI 프롬프트 재구성

## 배경
- 7일톡은 "명단을 몰라서"가 아니라 "2일톡 무반응 후 또 말 거는 부담"으로 실제 발송이
  잘 안 되고 있음 → 완료가 아닌 "보류(스킵)"라는 제3의 상태가 필요
- 3회톡은 반대로 "명단 파악 자체가 안 돼서" 못 하고 있었음 → 이미 구축된 오늘할일
  자동생성으로 해결된 상태, 이번 작업 범위 아님
- 2일차/7일차/3회차 톡 발송 시 매번 키워드를 새로 입력하는 게 부담스러움 →
  환자 첫 내원 시점부터 누구나(원장/간호/실장) 생각날 때마다 가볍게 한 줄씩 남겨두고,
  발송 시점엔 AI가 누적된 메모 중 필요한 것만 골라 반영하는 구조로 전환
- 실제 사용 중인 GPT 프로젝트 프롬프트(1년 전 버전, 대화형 5단계 구조)를 첨부함 —
  이번 기회에 "1회 호출로 완성 문구 생성"하는 구조로 재구성

## 0. 먼저 할 일: 계획만 제시
PatientNote 신규 모델 스키마안, MessageLog에 추가될 필드(스킵 상태 등)를 먼저 요약
제시해줘. 승인 후 진행.

## 1. 7일톡 스킵 기능
- TodoTask/MessageLog에 완료(DONE)와 별개로 SKIPPED 상태 추가
  (완료도 아니고 미처리도 아닌 제3의 상태 — 어떤 필드/enum으로 표현할지는 계획에서 제안해줘)
- /todo, /messages의 7일톡 항목에 "체크" 버튼 옆 "보류" 버튼 추가
- 보류는 재생성되지 않고 "보류됨"으로 계속 표시 (내원 재개 전까지)
- 알림톡 발송 충실도 지표(추후 대시보드 반영 예정, 이번 작업 범위는 데이터 구조까지만)에
  완료/보류/미처리 3분류가 가능하도록 상태값 설계

## 2. PatientNote 모델 (신규)
- 환자 1명당 여러 건 누적 (기존 Patient.memo 단일 필드와 별개, 대체 아님)
- 필드: 내용, 작성자(StaffUser), 작성시각
- 입력 위치: 내원체크 목록의 환자 행 옆 "메모+" 아이콘, /messages 톡 생성 패널 내에서도
  즉석 추가 가능
- 완전히 선택사항 — 아무도 안 남기면 빈 상태로 AI가 일반 문구 생성

## 3. /messages 톡 생성 패널 개선
- 톡 생성 버튼 클릭 시 뜨는 패널에 "기존 메모" 목록을 먼저 보여줌 (작성자·날짜 포함)
- 그 아래 새 키워드 입력창 (기존처럼 이번 발송에만 쓸 추가 키워드, 선택사항)
- 3회차톡의 경우 키워드 입력 위에 "호전도" 선택 드롭다운 추가: 상(60%↑) / 중(30~50%) / 하(0~30%)
  → 이 값에 따라 완전히 다른 메시지 구조(안정화/조정/재평가) 사용

## 4. AI 프롬프트 재구성 (2일톡/7일톡/3회톡 공통 엔진, src/lib/ai-message.ts)
- 기존 GPT 프로젝트의 5단계 대화 흐름 구조는 제거 (버튼 클릭 시 이미 톡종류/환자정보가
  정해져 있으므로 불필요) — 1회 호출로 완성 문구만 출력
- 공통 프롬프트 원칙:
  - 자연스러운 회화체, 4~6줄 한 문단, 과도한 이모지/느낌표 자제(🙂 정도만)
  - "복붙 느낌" 금지 — 누적된 메모 중 관련 있는 것만 선별해 문장에 자연스럽게 녹일 것
    (전부 다 넣으려 하지 말 것을 명시)
  - 마지막은 예약 링크 안내로 마무리
  - 출력은 메시지 본문만 (안내 문구 없이 바로 텍스트 시작)
- 2일차: 어제 내원 시 증상/상황 언급 + 안부 확인 + 초기 치료 중요성 자연스럽게 강조 + 재예약 유도
- 7일차: 2일톡의 연장(계속 말 거는 톤)이 아니라 독립된 안부 확인으로 톤 재설계 —
  "오랜만에 안부 여쭙는" 느낌, 재촉 아닌 걱정 톤
- 3회차: 호전도(상/중/하) 값에 따라 완전히 다른 구조 사용 (첨부 예시의 A/B/C 버전을
  few-shot으로 프롬프트에 포함) — 안정화(상) / 조정(중) / 재평가(하)
- 첨부 프롬프트의 실제 예시 문구들은 few-shot 예시로 프롬프트 안에 포함시켜 톤 학습시킬 것

## 5. 검증 요청
- 7일톡 보류 처리 → 재생성 안 되고 "보류됨" 상태 유지되는지 확인
- 시드 데이터 환자에 메모 2~3건 추가 후 톡 생성 → 메모가 자연스럽게 문구에 녹아드는지,
  전부 나열되지 않고 선별되는지 확인
- 3회차 호전도 상/중/하 각각 생성 결과가 첨부 예시와 톤/구조 면에서 유사한지 확인
- npm run build 통과
