## 작업 지시서 A — CTA 버튼: "프로그램문의하기" 명칭변경 + "이벤트문의하기" 신규 추가

### 배경
- 기존 티칭지(PatientTeachingPage) 공개페이지의 CTA 버튼("본상담 예약하기")은 
  카카오채팅 오픈 + 오늘 할 일 콜백업무 자동생성까지 정상 동작 중 (7/13 야간 구현분)
- 문제: 이벤트(EventImage)만 단독으로 발송하는 경우(공유링크 4가지 조합 중 "이벤트만"), 
  환자가 액션을 취할 CTA 버튼이 아예 없음
- 요청사항: ① 기존 버튼 명칭 변경 ② 이벤트에도 동일 프로세스의 버튼 신규 추가 
  ③ 클릭 후 환자가 보는 화면에 안내 문구 노출

### 명칭/텍스트 변경
- 티칭지 버튼: "본상담 예약하기" → **"프로그램문의하기"**
- 이벤트 버튼(신규): **"이벤트문의하기"**
- 생성되는 업무 제목도 함께 변경: 기존 "OOO님 본상담 예약 요청 — 연락 필요" → 
  **"OOO님 프로그램문의 요청 — 연락 필요"** (이벤트는 "OOO님 이벤트문의 요청 — 연락 필요")
- 클릭 후 버튼 자리에 노출할 안내 문구: **"카카오톡으로 상담 가능하십니다"** 
  (토스트처럼 사라지지 않고, 버튼을 이 문구로 교체하는 방식 — 팝업이 차단돼도 환자가 
  안내를 놓치지 않도록)

### 대상 파일 (Claude Code가 정확한 경로 재확인)
- `src/lib/teaching-pages.ts` — `requestConsultCallback()` 내 업무 제목 텍스트 변경, 
  중복방지 판별 조건 문자열("본상담 예약 요청" 포함 체크)도 "프로그램문의 요청"으로 
  동일하게 변경
- 이벤트용 신규 함수 추가 (같은 파일 또는 `src/lib/event-images.ts`): 
  `requestEventInquiryCallback()` — patientId 기준 당일+동일환자+"이벤트문의 요청" 
  포함 조건으로 중복방지, 담당자 없이 전체공통 WORK 업무 생성 (프로그램문의와 동일 패턴)
- 신규 공개 API: `POST /api/share-links/[token]/event-consult-request` — 인증 없음, 
  토큰으로 PatientShareLink 조회 → patientId + eventImageId 확인 → 
  `requestEventInquiryCallback()` 호출
- `src/components/TeachingPageContent.tsx` — 버튼 텍스트 변경 + 클릭 후 안내문구 
  교체 로직 추가
- `/s/[token]` 렌더링 컴포넌트(이벤트 섹션 부분) — "이벤트문의하기" 버튼 신규 추가, 
  `handleCtaClick` 패턴 재사용(팝업 차단 회피 위해 window.open을 클릭 핸들러 내 
  동기적으로 먼저 호출 → 이후 cta-click 로그 + event-consult-request 병렬 fetch)
- `.env`의 `NEXT_PUBLIC_KAKAO_CHANNEL_CHAT_URL` 그대로 재사용 (신규 환경변수 불필요)

### 동작 규칙
1. 프로그램/이벤트 버튼 모두 클릭 시: 카카오채팅 새 탭 오픈 + 해당 업무 자동생성 + 
   ActivityLog(CTA 클릭) 기록 — 3가지 동시 처리
2. 클릭 즉시 버튼을 "카카오톡으로 상담 가능하십니다" 텍스트로 교체 (재클릭 방지 겸 
   안내 역할)
3. 통합(프로그램+이벤트 둘 다) 공유링크의 경우, 두 버튼이 각 섹션 아래 독립적으로 
   존재 — 각각 별도로 클릭 가능해야 함
4. 이벤트 단독 공유링크(`eventImageId`만 있고 `teachingPageId` 없음)에서도 정상 
   동작해야 함
5. 당일+동일환자+동일유형(프로그램/이벤트) 중복 클릭 시 새 업무 생성 안 됨 (기존 
   프로그램문의 로직과 동일 기준)
