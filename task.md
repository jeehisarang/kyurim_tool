# 작업 지시서 — 자율신경맥파기(HRV) 검사 기능 신규 추가 (최종 통합본)

## 배경
검사실 노트북의 HRV 기기가 결과를 이미지/PDF로 자동 출력하며, 그 폴더는 구글드라이브와
동기화됨. 기존 인바디/근력검사와 달리 이 검사는 기기가 이미 판정까지 낸 완성된 결과지가
산출물이므로, 규림툴은 판정 계산 로직을 새로 만들 필요 없이 다음 역할만 담당함:
① 구글드라이브에서 결과 파일 목록 조회 후 선택 가져오기
② 핵심 수치 4개 기록(추이 추적용) + 원본 이미지 보존
③ 그 수치+학술근거+환자 증상기록을 조합한 5단 구조 AI 코멘트 생성
④ "환자와 함께보기" 화면에서 원본 이미지(확대/축소/드래그 가능)+AI 코멘트 노출

## 사전 확인 사항 (작업 시작 전 조사·보고)
- 기존 구글폼 연동(13-3)의 서비스계정(`secrets/google-service-account.json`) 방식 확인,
  이번엔 Drive API 필요 — 가벼운 전용 패키지 존재 여부 확인 (googleapis 전체번들 문제
  있었던 사례 참고, `@googleapis/sheets`처럼 Drive 전용 경량 패키지 검토)
- 원장님이 구글드라이브 폴더를 서비스계정 이메일과 공유(읽기 권한)하는 절차를 별도
  안내문서로 정리해 전달 (폴더 ID 확인 방법 포함)
- 기존 검사 이미지 업로드 자동 리사이즈/압축 유틸(7/15 세션 완료분) 재사용 가능 여부 확인
- 기존 PatientNote, ConsultationNote 조회 함수(티칭지에서 쓰는 것과 동일) 재사용 검토

## 대상 파일 / 신규 구조

### DB 스키마
- `HrvTestRecord` 신규:
  - patientId, testDate(자정 정규화), vascularHealthIndex(숫자), vascularHealthType(문자열,
    기기 판정 그대로 A~G), avgPulse(숫자), stressIndex(숫자)
  - sourceImagePath(원본 결과지 이미지, 리사이즈 저장)
  - aiCommentary(Text, AI 생성 5단 코멘트 전체)
  - measuredByStaffId, createdAt
- `ExamAcademicGuide` 신규 (향후 다른 검사종류 확장 고려한 범용 테이블):
  - examType(현재 "HRV" 고정값), content(원장 작성 학술근거, 자유 서술 — 아래 초안 참고)
  - tcmPatternMapJson(한의학적 가능성 매핑표 — 증상키워드/한의학적가능성/표현문구 배열,
    아래 초안 표 구조로 저장, 원장님이 설정화면에서 수정 가능하게)

### 화면/API
1. **결과지 가져오기 모달** (구글폼 가져오기 UX 패턴 재사용)
   - "HRV 결과지 가져오기" 버튼 → 지정 구글드라이브 폴더 파일 목록(파일명/수정일/썸네일)
   - 파일명 기반 환자 매칭 추정 + 직접 재검색 가능
   - 선택 시 이미지 미리보기
2. **수치 입력 폼**
   - 미리보기 옆에 혈관건강지수/혈관건강도타입/평균맥박/스트레스지수 입력
   - 저장 시 이미지 로컬 다운로드+리사이즈, HrvTestRecord 생성
3. **AI 코멘트 생성 (5단 구조)**
   - 입력재료: 이번 회차 4개 수치 + 직전 기록 대비 변화(있는 경우) + ExamAcademicGuide.content
     + PatientNote/ConsultationNote 중 HRV·스트레스와 관련성 있는 부분만 선별 + tcmPatternMapJson
   - 출력 구조(고정 순서):
     1) 기기 판독 요약 (수치 그대로 인용)
     2) 임상적 의미 ("기기 기준상" 명시, 지수/타입/스트레스 각각 별도 축으로 설명,
        확정형 표현 금지)
     3) 생활관리 포인트 (ExamAcademicGuide.content에 있는 내용만 근거로, 창작 금지)
     4) 한의학적 해석 — 관련 증상 있으면 tcmPatternMapJson에서 매칭되는 패턴을
        "가능성 시사" 형태로 언급 + "최종 변증은 문진·설진·맥진 후 확정" 문구 필수,
        관련 증상 없으면 "동반 증상 확인 후 판단 가능" 정도로 유보
     5) 안전 안내 — **AI가 생성하지 않고 시스템이 고정 텍스트를 그대로 덧붙임**
        (아래 고정문구 그대로, 자살사고 등 정신건강 위기신호 포함)
   - 프롬프트 공통 원칙: 실제 수치 뭉뚱그리지 않기, 신중한 단서("~가능성이 있습니다")
     유지, 새로운 의학적 효능/통계/변증 창작 금지, 관련성 없는 내용 억지로 끼워넣지 않기,
     자체검토(self-check) 지시문 포함
4. **환자와 함께보기 화면** (`/patient-view/exam/hrv/[id]` 신규)
   - 독립화면(사이드바 없음, 팝업 오픈), 원본 이미지 + AI 코멘트(5단, 안전안내 포함) 표시
   - **이미지 확대/축소/드래그(팬) 기능**: 마우스 휠 확대축소, 드래그로 이동, 화면 내
     +/-/원래크기 버튼도 제공, 기본 진입시 전체 이미지가 보이는 크기로 시작, 터치 핀치
     제스처 지원, 브라우저 자체 줌과 충돌 없어야 함, CSS transform 기반 직접구현 또는
     가벼운 라이브러리 검토 (신규 의존성 추가는 필요성 판단 후)
   - **공용 컴포넌트로 분리** (`ImageZoomViewer` 등) — 추후 다른 검사 이미지에도 재사용
   - 화이트리스트 방식으로 내부정보 노출 차단(기존 원칙 재사용)
5. **설정 화면**
   - HRV 학술근거(ExamAcademicGuide.content) + 한의학적 매핑표(tcmPatternMapJson) 작성/수정
     화면, 원장 전용 (기존 설정 메뉴에 탭 추가 또는 신규 경로)
6. **환자 검사이력**
   - `/examinations/patient/[patientId]` 통합 타임라인에 HRV 회차도 노출
     (썸네일+4개 수치 요약+"환자와 함께보기" 바로가기)

### 초기 시딩 데이터 (원장님 검토용 초안 — 그대로 저장 후 화면에서 다듬을 수 있게)

**ExamAcademicGuide.content (HRV) 초안:**
■ 검사 성격
손끝 말초맥파 기반 APG·HRV 측정 도구로, 혈관 건강과 누적 스트레스를 비침습적으로
선별하는 참고도구. 확진/질병진단 대체 불가. 한 시점의 스냅샷으로 해석하는 것이 안전.
검사 전 5분 안정, 흡연·음주 직후 회피, 손 차가우면 데운 후 측정 권장.
■ 혈관 건강 지수
낮을수록(음수일수록) 좋음. -30~+5는 안전범주.
생활관리: 혈압·혈당·지질 관리, 금연·절주, 규칙적 운동, 저염식.
■ 혈관 건강도 타입 (A~G)
A-B 안전, C-D 경계, E-G 위험. 고령일수록 자연히 불리해지는 경향 고려.
■ 스트레스 지수
<25 양호 / 25~35 일시적 / 35~45 초기 / 45~60 축적 / 60↑ 만성.
생활관리: 유산소운동(주 150분 이상), 수면 규칙화, 느린 호흡훈련.
■ 평균맥박
일반적 안정시 정상범위 60~100회/분. 반복측정 추세로 활용, 단독 진단 아님.
■ 해석상 주의
젊은 성인 대상 검증 중심 자료라 고령환자는 참고자료로만 활용.
LF/HF 등 세부지표는 단독으로 균형/불균형 단정 근거로 쓰지 않음.

## 검증 체크리스트
- [ ] 구글드라이브 폴더 파일 목록 정상 조회
- [ ] 파일명 기반 환자 매칭 + 매칭 실패/재검색 케이스 확인
- [ ] 결과지 이미지 다운로드+리사이즈 저장 확인
- [ ] 4개 수치 입력→저장→검사이력 반영 확인
- [ ] AI 코멘트 5단 구조 순서대로 생성되는지, 각 단계 내용 검증
- [ ] 한의학적 해석: 관련 증상 있는 케이스/없는 케이스 둘 다 확인 (억지 매칭 없는지)
- [ ] 안전안내 문구가 AI 생성이 아니라 고정 텍스트 그대로 붙는지 확인
- [ ] 직전 기록 없는 최초 검사 시 추이 언급 없이 정상 생성
- [ ] 이미지 확대/축소/드래그, 버튼 조작, 터치 핀치, 브라우저 줌과 비충돌 확인
- [ ] "환자와 함께보기" 화면 내부정보 노출 없음 확인
- [ ] npx tsc --noEmit, npm run build 통과
- [ ] 검증은 테스트 전용 환자(426번)로만 진행, 실데이터 화면에서 자동화 스크립트 금지
