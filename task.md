# 작업지시서 — HRV 이미지 판독 호출 모델 교체 (gpt-4o-mini → GPT-4.1)

## 1. 배경

방금 완료된 "자율신경균형도 앵커" 작업 검증 중, 반복 테스트(5회 이상)에서
gpt-4o-mini가 이미지 속 자율신경균형도 구역명을 읽을 때 가끔 5개 정식
명칭(과로형만성스트레스/질병형만성스트레스/급성스트레스/초기부정맥/
심한부정맥)에 없는 합성 표현을 만들어내는 경우가 발견됨. 코드 안전장치
(정식 명칭 매칭 검증 후 재시도)로 최종 출력은 항상 안전(5개 정식 명칭 중
하나 또는 생략)하게 나오지만, 판독 자체의 신뢰도가 들쭉날쭉함.

이번 작업은 이미지 판독(비전) 호출 1곳만 더 강력한 모델로 교체합니다.
텍스트 생성(코멘트 문장 작성) 등 나머지 호출은 gpt-4o-mini 그대로 유지 —
환자당 이미지 판독은 1회만 발생하므로 비용 영향은 미미함.

## 2. 변경 내용

- HRV 이미지 판독에 쓰이는 OpenAI 호출의 모델 문자열을 gpt-4o-mini에서
  gpt-4.1로 변경 (정확한 모델 ID는 Claude Code가 OpenAI 공식 문서
  기준으로 확인해서 적용 — "GPT-4.1"과 "gpt-4o" 계열은 모델 ID 표기가
  다를 수 있으니 실제 사용 가능한 정확한 문자열로 확인할 것)
- 기존 detail: "high" 옵션 등 비전 관련 파라미터는 유지
- 이 변경이 이미지 판독 호출에만 적용되고, 다른 OpenAI 호출(텍스트
  코멘트 생성 등)에는 영향 없어야 함 — 코드에서 호출이 분리되어 있는지
  확인 후 해당 지점만 정확히 수정

## 3. 검증 체크리스트

- 테스트 환자(426)로 동일한 이미지를 5회 이상 반복 재생성해서, 자율신경
  균형도 구역명이 이전보다 안정적으로(정식 명칭만) 나오는지 확인
- 기존 안전장치(정식 명칭 검증 재시도)는 그대로 유지한 채로 테스트 —
  모델을 바꿨다고 안전장치를 제거하지 말 것 (모델이 좋아져도 100% 보장은
  아니므로 이중 안전장치 유지)
- 비용에 큰 변화가 없는지 간단히 확인 (환자당 1회 호출이므로 미미할 것으로
  예상되지만, 실제 토큰 사용량 로그가 있다면 확인)
- 텍스트 코멘트 생성(다른 섹션들)은 이번 변경과 무관하게 정상 동작하는지
  확인
