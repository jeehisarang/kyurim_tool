# 작업지시서 — "검사톡" 신규 기능 (독립 검사결과 알림톡)

## 1. 배경 및 목적

- 현재 검사(인바디/근력/HRV, 향후 종류 계속 추가 예정)는 특정 프로그램에
  종속되어 결과가 다뤄지는 구조 — 검사 자체가 독립적으로 환자에게 원격
  공유되는 알림톡 유형은 없음
- 검사 종류가 앞으로 계속 늘어날 예정이므로(자율신경맥파/소변검사 등),
  검사결과를 환자에게 알려주는 것을 프로그램과 무관한 독립 알림톡
  유형("검사톡")으로 신설
- 한 환자가 여러 검사(예: 인바디+근력+HRV)를 받은 경우, 그 중 이번에
  보낼 것만 담당자가 체크해서 선택 가능해야 함
- 기존 티칭톡/이벤트톡과 마찬가지로 프로그램티칭 + 이벤트도 함께 결합
  가능해야 함 (기존 PatientShareLink의 티칭/이벤트 조합 구조를 검사결과까지
  포함하도록 확장)

## 2. 확정된 설계 방향 (원장님 결정사항)

1. 발송 방식: 기존 2일차톡/7일차톡/3회차톡과 동일하게 톡생성기의
   6번째 유형으로 편입 — 발송여부 추적(발송함/안함 상태 관리) 필요.
   기존 알림톡 발송 충실도 지표에도 포함되어야 함
2. 환자용 화면: 기존 "환자와함께보기"(원장실 팝업, 같은 자리에서
   보여주는 용도) 레이아웃을 재사용하지 않고 신규로 디자인.
   원격/모바일 열람을 전제로 한 디자인이어야 함 (카카오톡 링크 클릭 →
   모바일 브라우저로 보는 상황을 기본 가정)
3. 레이아웃: 검사 종류가 여러 개 선택된 경우, 한 페이지에 세로
   스크롤로 검사 종류별 섹션을 순서대로 배치. 단, 검사 개수가 늘어날
   상황(4~5개 이상)을 대비해 페이지 상단에 앵커 링크(예: "인바디 |
   근력검사 | 맥파")를 두어 클릭 시 해당 섹션으로 스크롤 점프하는 절충
   구조 적용 (완전한 탭 전환 방식은 채택하지 않음)

## 3. 착수 전 확인 필요 (설계 먼저 제시)

구현 전 아래 사항을 코드/스키마에서 확인 후 계획을 정리해서 먼저 보고할 것
(승인 후 실제 구현 진행):

1. 기존 PatientShareLink 구조 재활용 가능성 확인
   - PatientShareLink가 현재 "티칭지 없음/티칭지만/이벤트만/통합" 4가지
     고정 조합으로 되어 있는데, 이를 "티칭(0or1) × 이벤트(0or1) × 검사결과
     (0개 이상 다중선택)" 구조로 일반화할 수 있는지, 아니면 검사결과 포함
     여부만 별도 필드/테이블로 추가하는 게 나을지 스키마 변경안 제안
   - 기존 티칭/이벤트만 있는 공유링크(검사결과 미포함)의 동작에 영향이
     없어야 함 (하위호환 필수)

2. 검사 기록 다중선택 UI 위치 확인
   - 톡생성기에서 "검사톡" 유형 선택 시, 해당 환자의 인바디/근력/HRV
     기록 목록(최근 몇 건까지 보여줄지 포함해서 제안할 것 — 예: 각 종류별
     최근 1건만 기본 체크, 과거 기록도 선택 가능하게 할지 여부)을 어떤
     화면/컴포넌트에서 체크박스로 노출할지 설계안 제안
   - 향후 검사 종류가 늘어나도(자율신경맥파/소변검사 등) 자동으로 목록에
     포함되도록 검사 종류에 하드코딩 의존하지 않는 구조로 설계할 것

3. 검사 종류별 환자용 표시 항목 확인
   - 각 검사 종류(인바디/근력/HRV)마다 환자용으로 보여줄 항목이 다름
     (예: HRV는 4개 지표+AI 코멘트, 인바디는 체성분 수치, 근력은 SMI+
     악력+근력나이) — 기존 "환자와함께보기" 화이트리스트 로직(민감정보
     차단, 원장 전용 필드 노출 금지)을 신규 검사톡 화면에도 동일하게
     적용할 것. 컴포넌트를 완전 새로 만들되, 화이트리스트 변환 로직은
     기존 것 재사용 가능한지 확인

4. 발송추적/통계 반영 방식
   - 기존 알림톡 발송 충실도 지표 계산 로직에 "검사톡" 유형을 어떻게
     끼워넣을지 확인 (기존 로직이 5종 고정으로 하드코딩돼 있는지, 유형
     추가에 열려있는 구조인지부터 확인)

## 4. 동작 규칙 (설계 승인 후 적용)

- 검사톡 생성 화면: 환자 선택 → 해당 환자의 검사기록 목록에서 포함할 검사
  체크 → (선택) 프로그램티칭/이벤트 결합 여부 선택 → 공유링크 생성
- 공유링크(/s/[token]) 접속 시 신규 디자인의 환자용 화면 렌더링:
  - 상단: 앵커 네비게이션(선택된 검사 종류만 표시, 1개만 선택 시 네비
    생략 가능)
  - 이후 세로 스크롤로 각 검사결과 섹션 순서대로 배치
  - 티칭/이벤트가 결합된 경우 기존 PatientShareLink 표시 순서/스타일
    참고해서 배치
- 발송 여부는 톡생성기 목록에서 다른 유형과 동일한 방식(발송함 체크)으로
  추적
- 원장 전용 데이터(측정자, 메모, 수정/삭제 버튼 등)는 이 화면에 절대
  노출되지 않도록 화이트리스트 방식 유지

## 5. 검증 체크리스트

- 테스트 전용 환자(예: 426번)로 인바디+근력+HRV 3종 검사기록 준비 후
  검사톡 생성 → 3종 모두 체크 → 링크 생성 → 모바일 화면 크기로 실확인
- 검사 1종만 선택한 경우 앵커 네비 없이 정상 표시되는지 확인
- 티칭+이벤트+검사결과 통합 조합도 실제로 생성해서 확인
- 기존 티칭/이벤트만 있는 기존 공유링크(과거 생성분)가 이번 변경 이후에도
  깨지지 않는지 확인 (하위호환)
- 발송여부 체크 후 알림톡 발송 충실도 통계에 정상 반영되는지 확인
- 실데이터 화면에서 자동화 스크립트 실행 금지 — 테스트 전용 환자로만 검증

## 6. 완료보고 형식

- 3번(착수 전 확인) 각 항목의 코드 확인 결과 + 설계안을 먼저 보고
  (구현 전 승인 요청)
- 승인 후 구현 완료 시: 스키마 변경사항, 수정/신규 파일 목록, 실확인 결과,
  git commit 해시
