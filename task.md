# 작업지시서 — 건강 리포트 가독성 개선 (Part 1: 해석 섹션 시각화+강조 / Part 2: 치료방향 롤백+다듬기)

## 배경
- 최승희 팀장 실사용 피드백: "한의 건강해석"·"이런 경향이 지속되면" 두 섹션이
  문장만 쭉 나열돼 있어 여러 카테고리가 겹칠 때 핵심이 한눈에 안 들어옴.
  또한 이 부분이 "결국 리포트의 핵심"이라, 환자에게 보여줄 때 그럴듯하고
  전문적으로 보이는 인상이 중요함.
- "치료 방향" 카드는 고정사전(커밋 d5fd073) 전환 후 오히려 "교과서적이고
  도움 안 되는" 밋밋한 느낌이 됐다는 피드백 — AI 개인화 버전(커밋 48dbc6e)
  으로 롤백하되, 불필요한 justification 문장은 덜어내기로 확정.

## PART 1 — 한의건강해석 / 경향지속 섹션 개선

### 버그 수정 (최우선)
- "한의건강해석" 섹션에서 발견된 **중복 문장 버그**: "이번 검사 구역이
  과로형만성스트레스에 가까운 패턴으로, 기기 분석 기준으로는
  과로형만성스트레스 구역에 가까운 패턴입니다" 처럼 같은 내용이
  한 문장 안에서 반복됨. 프롬프트 검토 후 수정, 재발 방지용 안전장치
  (기존 `violatesDuplicateSentenceRule` 패턴 재사용 가능한지 검토) 고려

### 사전확인 체크리스트 (설계 승인 완료)
- [x] 카드4(한의건강해석) 상단에 카테고리별 점수 시각화 요소 추가
      (`TcmCategoryScore` 활용, 막대 또는 도넛 형태 — 구현 편의에 따라
      Claude Code가 제안, 원장 확인 후 확정)
- [x] 핵심 패턴명·핵심 증상 조합에 볼드 마크업 적용 (AI 프롬프트에
      "중요 키워드에 **볼드** 표시" 지시 추가)
- [x] "이런 경향이 지속되면" 섹션도 동일하게 핵심 문구(예: "과로형만성스트레스
      경향이 지속되면") 볼드 처리

### 동작 규칙
1. **시각화**: 리포트가 참조하는 활성 카테고리들(예: 스트레스·정서긴장/
   기력·냉증/담습·부종/순환·어혈)의 `TcmCategoryScore` 값을 가져와
   카드4 최상단에 간단한 바(bar) 형태로 표시
   - 각 바에 카테고리명 + 상대적 비율 표시 (임상 기준이 아닌 "체크 문항
     대비 비율"이라는 점을 작게 각주로 명시 — 기존 안전 원칙 유지)
   - 색상은 기존 celadon/한지 팔레트 내에서 처리 (튀지 않게)
2. **볼드 강조**: AI 프롬프트에 "카테고리명, 핵심 패턴명(예:
   과로형만성스트레스), 결정적 증상 조합에 한해 볼드 마크업 사용,
   남용하지 말 것(문단당 1~2곳)" 지시 추가. 렌더러는 이미 볼드를
   정상 처리하므로 프론트엔드 수정 불필요, 프롬프트만 수정
3. 두 섹션 모두 문장 자체는 유지 (없애는 게 아니라 시각화+강조로 보조)

### 검증 체크리스트
- [ ] 테스트환자(426), 4개 카테고리 조합으로 중복 문장 재발 안 하는지
      5회 반복 생성 확인
- [ ] 카테고리 점수 시각화가 실제 값과 일치하는지 확인 (스크린샷 대조)
- [ ] 볼드 마크업이 과하지 않게(문단당 1~2곳) 적용되는지 확인
- [ ] 1개 카테고리만 있는 경우에도 시각화 정상 동작(바 1개)

## PART 2 — 치료 방향 카드, AI 개인화 버전으로 롤백 + 다듬기

### 사전확인 체크리스트 (설계 승인 완료)
- [x] 커밋 d5fd073(고정사전) 되돌리고, 48dbc6e 방식(카테고리별 독립 AI
      호출)으로 복귀
- [x] 대표처방(시호소간산 등)도 다시 포함
- [x] 단, "이 치법이 왜 이 증상에 맞는지" 식의 불필요한 근거 설명 문장은
      제거 — 키워드 + 대표처방을 담백하게 언급하는 수준으로 축소
- [x] 카테고리별 독립 AI 호출 원칙(다른 카테고리 정보 프롬프트에 안 줌)은
      그대로 유지 — 이건 d5fd073에서도 유지됐던 좋은 구조이므로 롤백 대상 아님

### 동작 규칙
- `generateCategoryTreatmentCards` 복원 (또는 신규 재작성), 프롬프트 수정:
  - 입력: 해당 카테고리의 `treatmentPrinciple`(키워드+대표처방 원문)
  - 출력 형식 예: "소간해울·이기해울·청간사화·양심안신 치료 방향을
    고려할 수 있으며, 시호소간산·가미소요산 계열이 참고될 수 있습니다"
    정도로 **1문장 내외**, 근거·이유 설명 문장 없이
  - 명시적으로 금지: "~때문에", "~이유로", "~증상에 맞춰" 같은
    justification 연결어 사용 금지 지시
- 고정사전(`TREATMENT_PRINCIPLE_KEYWORD_GLOSSARY`)은 코드에서 제거하지
  말고 남겨두되, 카드 렌더링에는 더 이상 참조하지 않음 (추후 다른 용도로
  재사용 가능성 있으니 삭제는 보류)

### 검증 체크리스트
- [ ] 테스트환자(426), 4개 카테고리 조합에서 카드 간 섞임 없음 확인
      (48dbc6e 때와 동일 기준)
- [ ] 각 카드 문장이 근거설명 없이 담백한 1문장 내외로 나오는지 확인
- [ ] 대표처방 정상 노출 확인
- [ ] npx tsc --noEmit / npm run build 통과
