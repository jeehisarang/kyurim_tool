# 작업지시서 — 프로그램 선택 UX 개선 + 티칭지 수작업 편집 기능

## 배경
- 실사용 중 발견: 직원이 "AI 생성 > 환자 티칭지"에서 프로그램 선택 시, 환자가 이미
  진행 중인 프로그램(예: 황제감비탕 5/7차)과 무관한 다른 프로그램(예: S환 다이어트)을
  실수로 선택해 티칭지를 생성한 사례 발생 (김경자님 케이스)
- 콘텐츠 품질(프롬프트)은 검증 완료 — 이번 작업은 UX/기능 보완에 집중

## 1. 프로그램 선택 화면 — 진행 중 프로그램 우선 표시 + 확인 알림

`/consult-mode` 또는 톡생성기 내 "환자 티칭지" 생성 플로우에서 프로그램 선택 시:

- 선택된 환자의 **현재 진행 중(ACTIVE) Prescription이 있으면**, 프로그램 목록 상단에
  "현재 진행 중" 배지와 함께 해당 프로그램을 우선 노출
- 직원이 **진행 중이 아닌 다른 프로그램**을 선택하려 하면, 생성 버튼 누르기 전
  confirm 성격의 안내 문구 노출:
  예) "이 환자는 현재 [황제감비탕]을 진행 중입니다. [S환 다이어트] 티칭지를
  만드시겠습니까?" — "예, 계속" / "취소" 선택
- 진행 중인 프로그램이 없는 환자(신규 등)는 기존처럼 자유롭게 선택 가능, 알림 없음

## 2. 티칭지 5필드 수작업 편집 기능

- 티칭지 생성 완료 화면(현재 "링크 복사" / "다른 프로그램으로 새로 만들기" 버튼이
  있는 화면)에 **"수정"** 버튼 추가
- 클릭 시 headline/personalSubtopic/bodyText/examSummary/academicHook 5개 필드를
  각각 텍스트 영역(textarea)으로 바로 편집 가능하게 전환
- "저장" 클릭 시 PatientTeachingPage 레코드 업데이트 → `/p/[token]` 페이지에 즉시 반영
- examSummary가 null인 경우(검사 무관 프로그램) 그 필드는 편집 UI 자체를 노출하지 않음
- 수정 이력은 별도로 남기지 않음 (현재상태 덮어쓰기, ConsultationNote와 달리 진료기록
  아님 — 단순 마케팅 카피이므로)
- 저장 실패 시 명확한 에러 표시 (기존 저장실패 무음처리 버그 수정 원칙 동일 적용)

## 3. 검증 항목
- 진행 중 프로그램 있는 환자 / 없는 환자 각각 정상 동작 확인
- 무관한 프로그램 선택 시 확인 알림 정상 노출, "취소" 시 생성 안 됨 확인
- 수작업 편집 → 저장 → `/p/[token]` 반영 확인 (실제 환자 아닌 테스트 환자로,
  검증 후 즉시 삭제)
- `npx tsc --noEmit` 및 `npm run build` 둘 다 통과 (dev 서버 내리고 실행)
