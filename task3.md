# 작업지시서 — HRV 검사등록 화면 업로드/입력 흐름 개선

## 1. 배경 (실사용 중 발견된 불편사항)

원장님이 오늘 직접 HRV 검사를 해보며 느낀 문제 2가지:

1. 업로드 방식 선택이 무의미함 — 현재 검사지 이미지 등록 시 "구글드라이브
   업로드"와 "직접 등록" 두 가지 방식을 선택하게 되어 있는데, 실제로는 저장
   경로(폴더)만 달라질 뿐 사용자 입장에서 구분할 실익이 없음. 하나로 통일 필요.
2. 사진 업로드와 수치 입력이 분리되어 있어 종이 검사지를 봐야 함 — 현재
   흐름은 (a) 검사지 사진을 업로드하고 (b) 별도로 혈관건강지수/평균맥박/
   스트레스지수 등 수치를 수기 입력해야 등록이 완료되는데, 수치 입력 시
   업로드한 이미지가 화면에 보이지 않아 결국 인쇄된 종이 검사지를 보면서
   입력해야 함. 원장실은 프린트를 최대한 안 하려는 방침이라 이게 불편함.

목표 흐름: 검사지 사진 업로드 → 업로드한 이미지가 등록화면에 바로
미리보기로 표시됨 → 그 이미지를 보면서(종이 없이) 수치를 입력 → 버튼 클릭
시 코멘트(AI 해석) 생성. 즉 "이미지 미리보기 + 수치 입력 폼"이 한 화면에
동시에 떠 있어야 함.

적용 범위: HRV 검사등록 화면만 (인바디/근력검사는 이번 작업 대상 아님,
건드리지 말 것)

## 2. 착수 전 확인 필요 (설계 먼저 제시)

구현 전에 아래 사항을 코드에서 확인하고, 계획을 정리해서 먼저 보고할 것
(승인 후 구현 진행):

1. 업로드 방식 선택 UI의 실제 구조 확인
   - "구글드라이브 업로드"가 실제로 Google Drive API 연동인지, 아니면 단순히
     저장 폴더 경로 옵션(A/B) 정도의 차이인지 코드로 확인
   - 폴더가 실제로 다른 용도(예: 백업, 외부공유 등)로 쓰이고 있는지도 확인 —
     단순 UX상 불필요한 선택지인지, 아니면 폴더 구분에 실질적 이유가 있는지
     파악한 뒤 결과를 원장님께 먼저 보고
   - 통일하기로 결정되면, 어느 저장 방식(기존 "직접 등록" 쪽 로직)으로
     합칠지 제안할 것 (기존 데이터/파일 경로에 영향 없는 방향 우선)

2. 현재 등록 화면(이미지 업로드 → 수치 입력) 컴포넌트 구조 확인
   - 현재 이미지 업로드 슬롯(1페이지 필수/2페이지 선택)과 수치 입력 폼이
     같은 화면의 다른 섹션에 있는지, 아니면 서로 다른 스텝/페이지로 분리돼
     있는지 확인
   - 이미지 업로드 직후 프리뷰가 이미 있는지(있다면 위치만 재배치),
     없다면 신규로 이미지 미리보기 컴포넌트를 추가해야 하는지 확인

3. 레이아웃 제안
   - 데스크톱 기준 좌측(또는 상단) = 업로드된 이미지 미리보기(확대/축소
     가능하면 더 좋음), 우측(또는 하단) = 수치 입력 폼 형태를 기본안으로
     제안하되, 기존 화면 톤/컴포넌트 재사용 가능 여부 먼저 검토
   - 이미지 2장(1페이지/2페이지) 업로드 시 미리보기에서 페이지 전환/동시
     노출 방식도 같이 제안할 것

## 3. 동작 규칙 (설계 승인 후 적용)

- 업로드 방식 선택 UI(구글드라이브 vs 직접등록) 제거, 단일 업로드 버튼으로 통일
- 이미지 업로드 완료 시 같은 화면에 즉시 미리보기 표시 (별도 페이지 이동 없이)
- 수치 입력 필드는 미리보기 이미지와 동시에 화면에 노출되어야 함 — 입력 중
  스크롤해서 이미지가 안 보이는 배치는 지양
- 기존 수치 입력/미리보기 계산 로직(정상/비정상 판정 등)은 그대로 유지,
  이번 작업은 레이아웃/업로드 흐름 개선에 한정
- 기존에 등록된 HRV 기록(이미지 경로 등)에는 영향 없어야 함

## 4. 검증 체크리스트

- 테스트 전용 환자(예: 426번)로 신규 HRV 등록 시나리오 실행: 업로드 → 미리보기
  노출 확인 → 미리보기 보면서 수치 입력 → 코멘트 생성까지 한 화면에서 끊김
  없이 진행되는지 확인
- 2페이지 이미지 업로드 케이스도 동일하게 확인
- 기존 등록된 HRV 기록 조회 시 이상 없는지 확인 (레이아웃 변경이 조회 화면에
  영향 주지 않는지)
- 실데이터 화면에서 자동화 스크립트 실행 금지 — 테스트 전용 환자로만 검증
