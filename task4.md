# 작업지시서 — HRV 상세지표 해석 방향 전환 (자율신경균형도 유형 + 맥박다양성 앵커)

## 1. 배경

이전에 드린 "HRV 상세지표 표준범위 조사" 작업은 그대로 계속 진행해주세요
(연령별 표준범위 근거가 코드/설치파일에 있는지 확인하는 작업 — 취소 아님).

다만 그 작업과 별개로, 상세지표(TP/VLF/LF/HF/SDNN/RMSSD) 코멘트를 만드는
방향 자체를 아래처럼 전환하기로 했습니다. 개별 지표를 하나하나 정상/비정상
판정해서 나열하는 방식은 "결과 재탕"이라는 문제를 그대로 반복하는 것이라
채택하지 않습니다.

새 방향: 유비오맥파 리포트("수양명경락기능검사 결과표")의 "자율신경
균형도" 표와 "맥박다양성" 값을 해석의 중심 앵커로 삼습니다.
- "자율신경균형도"는 기기가 이미 교감/부교감 관련 여러 지표를 종합해서
  "과로형 만성스트레스 / 질병형 만성스트레스 / 급성스트레스 / 초기부정맥 /
  심한부정맥" 중 하나의 사분면 유형으로 분류해주는 종합판정입니다
- "맥박다양성"은 심장박동 변화의 다양성을 하나의 숫자로 보여주며, 기기
  자체에 "성인평균 30~40, 20이하면 만성스트레스"라는 해석 기준이 이미
  붙어있습니다
- 개별 세부지표(TP/VLF/LF/HF 등)를 낱낱이 재해석하지 않고, 이 종합판정
  2가지를 코멘트의 뼈대로 삼아 (a) 미병 프레임으로 재서술, (b) 지난 검사
  대비 추이 서술, (c) 한의학적 해석(간기울결/심비양허 등 기존 패턴명 로직)과
  자연스럽게 연결하는 것이 목표입니다

## 2. 착수 전 확인 필요 (조사만, 구현 착수 금지)

1. 현재 HRV 등록 시 업로드되는 이미지에 이 정보가 이미 포함되는지 확인
   - 현재 수동 등록 화면에서 업로드하는 1~2페이지 이미지가 "수양명경락기능
     검사 결과표"(이미지1 예시 — 맥박변화도/상세분석/심박분포도/자율신경
     균형도/스트레스지수/가속도맥파분석이 다 담긴 종합 리포트)와 같은
     레이아웃인지, 아니면 더 단순한 페이지("혈관 건강도 측정 결과"류,
     이미지2 예시)만 업로드되고 있는지 확인
   - 만약 이미 종합 리포트 이미지가 업로드되고 있고 AI 프롬프트에 그 이미지가
     전달되고 있다면 → 이번 작업은 프롬프트 로직 변경만으로 가능 (인프라
     변경 불필요)
   - 만약 지금은 단순 페이지만 업로드되고 있다면 → 업로드 요구사항을 "종합
     리포트 페이지 포함"으로 바꿔야 하므로 별도 설계가 필요함을 보고

2. "자율신경균형도" 유형과 "맥박다양성" 값이 CSV에 있는지 재확인
   - 지난 조사에서 확인한 CSV 컬럼 목록에는 이 두 항목이 없었음 — 재확인
     결과도 마찬가지라면, 이 두 값은 이미지에서만 얻을 수 있다는 뜻이므로
     CSV 자동연동 설계 시 "이미지 캡처 파일 매칭"이 선택이 아니라 필수
     항목으로 격상되어야 함 (이전 CSV 자동연동 설계지시서 내용 업데이트 필요)

## 3. 목표 코멘트 구조 (조사 결과 확인 후 별도 지시로 설계 착수 예정)

아래는 방향성 공유 목적이며, 이번 작업에서 바로 구현하지 않습니다:

- 자율신경균형도 사분면 유형 → 미병 신호로 재서술 (예: "과로형 만성스트레스"
  쪽으로 위치한 경우 "몸이 소진되기 쉬운 상태로 기울어져 있다는 미병 신호로
  볼 수 있습니다" 류)
- 지난 검사 대비 사분면 위치가 어느 방향으로 이동했는지 추이 서술 (이미
  구현된 "결과와 추이" 섹션 로직 확장)
- 맥박다양성 수치와 기기 자체 해석 기준("30~40 정상, 20이하 만성스트레스")을
  참고해 보조 설명으로 연결
- 한의학적 해석 섹션과 연결 시, 기존 안전원칙(증상 텍스트 매칭 시에만 패턴명
  언급) 그대로 유지 — 이번에도 예외 없음
