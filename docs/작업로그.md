### 7. 7일톡 스킵 + 환자메모 누적구조 + AI 프롬프트 재구성
**버그 3건 발견 및 수정** (이번 작업 진행 중 확인):
1. `/api/visits`가 `visitDate`를 자정 시각과 **정확 일치**로 조회하는데, 시드 데이터가 임의 시각으로 들어가 있어 "오늘 체크된 내원 목록"이 0건으로 잘못 표시됨 → 시드 스크립트를 자정 고정으로 수정
2. "이번주 처리 현황" 집계가 `TodoTask.isDone`만 보는데, 톡 할일은 애초에 이 필드를 쓰지 않는 구조라 톡 완료 건이 진행률에 반영되지 않음 → `MessageLog.sentDate` 기준으로 톡 완료를 별도 집계하도록 수정
3. `/api/messages/confirm`이 `aiDraftContent`를 안 넘기면 기존 저장된 AI 초안을 `null`로 덮어써버리는 문제 → 값이 없으면 기존 값을 유지하도록 수정 (`/todo`에서 톡 체크 시 초안이 사라지지 않게)

**신규 기능**:
- `PatientNote` 모델 신규 (환자 1명당 여러 건 누적, `Patient.memo`와 별개) — `/visit-check` 목록의 "메모+" 아이콘, `/messages` 톡 생성 카드에서 즉석 추가 가능
- `MessageLog`에 `skippedAt`/`skippedByUserId` 추가 → 완료/보류/미처리 3분류. 7일톡에 한해 "보류" 버튼 제공, 보류해도 재생성되지 않음
- `/messages` 톡 생성 패널 개선: 누적 메모 목록 → 추가 키워드 입력 → (3회톡만) 호전도(상/중/하) 드롭다운 → 문구생성
- **AI 프롬프트 구조 논의 및 재구성**: 기존 GPT 프로젝트(1년 전, 5단계 대화형: 시작→톡종류선택→환자정보입력→문구생성→수정확인)의 few-shot 예시(`docs/kakao-prompt-original.md`)를 그대로 반영해, 버튼 클릭 시 이미 톡종류/환자정보가 정해져 있으므로 1회 호출로 완성 문구를 뽑는 구조로 재설계. 2일톡/7일톡 톤을 명확히 분리(7일톡은 "오랜만의 안부, 걱정" 톤으로 재설계)하고 3회톡은 호전도별 A/B/C 구조를 few-shot으로 학습시킴
- 실제 GPT 호출로 검증: 누적 메모 중 관련 있는 것만 선별 반영되는지, 3회톡 상/중/하가 예시와 톤/구조가 일치하는지 확인 완료
