# 규림한의원 통합 툴 프로젝트 — 요구사항 누적 정리본
### (최종 업데이트: 2026-07-06)

---

## 1. 시스템 성격

- 구글시트를 대체하는, 규림한의원 **로컬 네트워크 내**에서 원장님과 직원들이 함께 쓰는 **통합 실무 툴**
- 한차트(정식 진료기록 프로그램)와는 별개로 운영
- 단순 통계 대시보드가 아닌 **환자관리 중심 통합 실무 툴**
- 진료분야, 진료구분 등 항목은 원장님이 직접 추가/수정/삭제 가능한 **커스텀 구조**

---

## 2. 핵심기능 ① 내원체크

- 차트번호 / 이름
- 진료분야 (급여치료, 자보, 한약, 비만, 피부·한방성형 — 커스텀 가능)
- 진료구분 (초진, 재초진, 재진, 전화상담 — 커스텀 가능)
- 날짜별 내원 체크 (예약함/안함) → 예약율 산출

---

## 3. 핵심기능 ② 통계·CRM 지표

- 누적환자수, 진료분야별 환자수
- 일별/평균 예약율, 일평균 내원수
- **인당 내원수** — 월 경계 없이 누적 추적 (기존 구글시트 한계 개선 포인트)
- **7일 재방문율** — 월 경계 오차 최소화 (월말 초진→다음달 재진 누락, 전화상담 후 재내원 누락 문제 해결)
- **3회 이상 내원 초진율** — 동일한 월 경계 오차 문제 해결

---

## 4. 핵심기능 ③ 문자발송(알림톡) 관리

### 4-1. 알림톡 5종 상세
| 유형 | 발송 시점/조건 | 생성방식 |
|---|---|---|
| 웰컴톡 | 한차트에서 발송 (초진/재초진), 카카오채널 대화창 자동생성 목적 | 고정 |
| 만남톡 | 고정 템플릿 2종 보유, 저장/추출 기능 필요 | 고정 |
| 2일차톡 | 첫내원 다음날 점심 발송 | 환자메모 기반 AI 생성 |
| 7일차톡 | 7일 미내원 환자 대상 | 환자메모 기반 AI 생성 |
| 3회차톡 | 3회 내원 완료 축하 + 향후안내 | AI 생성 |

- 2일차/7일차/3회차톡은 **공용 AI 문구생성 엔진**으로 설계
- 발송여부 추적 (기존 시트 AL~AP열 대응)
- **알림톡 발송 충실도 지표**: 대상환자(초진/재초진) 대비 실제 발송률, 5종 각각 % 통계화
- 향후 메시지 템플릿 등록·관리 기능으로 확장 (웰컴톡 예시 템플릿 확보됨)

### 4-2. AI 문구생성 UX 흐름 (신규, 오늘 확정)
- 문자발송 관리 화면에서 환자별 **알림톡 유형 버튼** 클릭
- 클릭 시 해당 환자의 누적 정보(메모, 체중/인바디, 내원이력 등) 기반 **AI가 적당량의 맞춤형 문구 생성**
- 담당자가 결과를 **복사(copy) → 카카오톡에 직접 붙여넣기 → 수동 발송**
- 자동발송이 아닌 **"AI 초안 생성 + 수동 발송"** 구조 (발송 최종 컨트롤은 사람이 유지)
- 알림톡 유형별 프롬프트는 추후 개별 전달 예정

## 5. 핵심기능 ④ 목표 관리

- 모든 통계지표의 목적은 "수치를 늘리는 것" → 목표관리가 핵심 축
- 지표별 주간목표/월간목표 입력 칸
- 목표 대비 달성률 + 남은 수치 상시 표시
- 목표는 언제든 수정 가능

---

## 6. 핵심기능 ⑤ 사용자 식별 / 작업이력

- 원장/직원 각자 로그인 또는 체크한 사람 식별 가능
- "누가 무슨 일을 했는지" 확인 가능한 구조
- 처방주기 체크(⑧) 등 담당자 지정·확인 기능과 직결

## 7. 핵심기능 ⑥ 프로젝트 관리 시스템 (오픈구조)

- "프로젝트" = 환자 기본정보 + 커스텀 체크항목 + 통계뷰의 공통 틀
- 프로젝트 자유 생성/삭제/편집 가능
- 기존 "내원체크"(①)는 기본형(디폴트 프로젝트)으로 재정의
- 배경: 구글시트로 MVP성 프로젝트를 여러 개 운영하며 시트가 분산되던 문제 해결 목적
- 신규 프로젝트 생성 시 별도 구글시트 없이 템플릿에서 spin-up

### 7-1. 프로젝트 사례 ① 킬팻캡슐 (현재 운영 중)
- 플로우: QR/링크 → 구글설문 → 시트저장 → 직원 확인(현재 수작업) → 타입판단 → 상품발송(3일무료/1달/3달)
- 웰컴/2일차/3일차(마감) 알림톡 별도 운영
- AI 문구생성 특이구조: **환자발송용 메시지 + 원장용 내부분석(타입/처방방향/상담포인트) 2단 출력**
- NotebookLM으로 시트 실시간 동기화하여 프롬프트 기반 생성 중 (프롬프트 추후 공유)
- 새 응답 자동감지 필요 (현재 수작업)
- 마감설문(후기) 연동 필요
- 무료체험→유료전환 시 중복 작성으로 임시 대응 중 → 향후 정식 구현 필요

### 7-2. 프로젝트 사례 ②③
- 나머지 구글시트 프로젝트들 — **추후 별도 세션에서 요구사항 수집 예정** (오늘은 보류)

---

## 8. 핵심기능 ⑦ 목표 진행 / 반성 / 크로스체크 (방향성 핵심)

- 문제의식: 지금까지 "기입"과 "확인"에만 머물러 있었음
- 목표관리(④)에 사고 사이클 추가:
  - **진행**: 목표를 향해 지금 무엇을 하고 있는지
  - **반성**: 미달성 시 원인 되짚기
  - **크로스체크**: 여러 지표를 교차 검토 (예: 예약율↑ 재방문율↓ 원인 연결)
- → 대시보드가 "숫자 나열"이 아닌 **"목표 달성을 돕는 사고 도구"**가 되어야 함

## 9. 핵심기능 ⑧ 처방주기 관리 / "오늘 할 일" 알림

### 9-1. 프로그램별 처방 단위 (오늘 확정)
| 프로그램 | 처방 단위 | 후속조치 텀 |
|---|---|---|
| 킬팻캡슐 | 3달 프로그램, 2주치씩 분할처방 | 남은 차수 자동계산 → 다음 처방일 알림 |
| 탕약 (감비탕, 황제감비탕) | 3달 프로그램, **2주 기본단위** 분할처방 | 다음 처방일 자동계산 |
| 환약 (S환, 하비환) | 한번에 처방 (분할 없음) | 종료 후 **1달** 기준 후속조치 (오늘 확정) |

### 9-2. 알림 형태 (오늘 확정)
- 별도 배지/탭이 아닌 **"오늘 할 일" 목록에 통합 배치**
- 핵심: **직관적 노출 + 담당자 지정 + 체크 여부(했음/안했음) 상호 확인** 가능한 구조
- 기능 ⑤(사용자 식별/행동 로깅)와 직접 연결

---

## 10. 로드맵 — Phase 2 (다음 단계, 오늘 신규 확정)

**비만관리 환자 통합 케어**
- 현재 이원화 문제: 킬팻캡슐은 구글폼/시트 + 한차트 메모(체중 등) 이중 관리
- Phase 2에서는 캡슐뿐 아니라 **환약/탕약 비만 환자까지 포함**하여
  - 체중, 인바디 결과 등 **정량 데이터 입력 필드**를 통합툴 내에 추가
  - 환자별 진행 추이(체중 변화 그래프 등) 확장 가능
- 역할 분리 원칙: **한차트 = 정식 진료기록(처방/진단)**, **통합툴 = 비만관리 정량데이터**
- 순서: 1단계(구글시트 대체 통합 실무 툴)를 먼저 완성 후 착수
