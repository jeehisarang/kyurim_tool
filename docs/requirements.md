# 규림한의원 통합 툴 프로젝트 — 요구사항 누적 정리본
### (최종 업데이트: 2026-07-07)

---

## 1. 시스템 성격

- 구글시트를 대체하는, 규림한의원 **로컬 네트워크 내**에서 원장님과 직원들이 함께 쓰는 **통합 실무 툴**
- 한차트(정식 진료기록 프로그램)와는 별개로 운영
- 단순 통계 대시보드가 아닌 **환자관리 중심 통합 실무 툴**
- 진료분야, 진료구분 등 항목은 원장님이 직접 추가/수정/삭제 가능한 **커스텀 구조**

## 4. 핵심기능 ③ 문자발송(알림톡) 관리

### 4-1. 알림톡 5종 상세
| 유형 | 발송 시점/조건 | 생성방식 |
|---|---|---|
| 웰컴톡 | 한차트에서 발송 (초진/재초진), 카카오채널 대화창 자동생성 목적 | 고정 |
| 만남톡 | 고정 템플릿 2종 보유, 저장/추출 기능 필요 | 고정 |
| 2일차톡 | 첫내원 다음날 점심 발송 | 환자메모 기반 AI 생성 |
| 7일차톡 | 7일 미내원 환자 대상 | 환자메모 기반 AI 생성 |
| 3회차톡 | 3회 내원 완료 축하 + 향후안내 | AI 생성 |

- 2일차/7일차/3회차톡은 **공용 AI 문구생성 엔진**으로 설계
- 발송여부 추적 (기존 시트 AL~AP열 대응)
- **알림톡 발송 충실도 지표**: 대상환자(초진/재초진) 대비 실제 발송률, 5종 각각 % 통계화
- 향후 메시지 템플릿 등록·관리 기능으로 확장 (웰컴톡 예시 템플릿 확보됨)
- **[2026-07-07 확정]** AI 문구 품질 기준: 어떤 경우에도 비문(어색한 문장)이 나오면 안 됨 —
  담당자가 다시 손볼 정도면 AI 도입 의미가 없다는 것이 원칙. 자체검토(self-check) 지시문을
  프롬프트에 포함해 반복검증

### 4-2. AI 문구생성 UX 흐름
- 문자발송 관리 화면에서 환자별 **알림톡 유형 버튼** 클릭
- 클릭 시 해당 환자의 누적 정보(메모, 체중/인바디, 내원이력 등) 기반 **AI가 적당량의 맞춤형 문구 생성**
- 담당자가 결과를 **복사(copy) → 카카오톡에 직접 붙여넣기 → 수동 발송**
- 자동발송이 아닌 **"AI 초안 생성 + 수동 발송"** 구조 (발송 최종 컨트롤은 사람이 유지)
- 알림톡 유형별 프롬프트는 추후 개별 전달 예정

## 7. 핵심기능 ⑥ 프로젝트 관리 시스템 (오픈구조)

- "프로젝트" = 환자 기본정보 + 커스텀 체크항목 + 통계뷰의 공통 틀
- 프로젝트 자유 생성/삭제/편집 가능
- 기존 "내원체크"(①)는 기본형(디폴트 프로젝트)으로 재정의
- 배경: 구글시트로 MVP성 프로젝트를 여러 개 운영하며 시트가 분산되던 문제 해결 목적
- 신규 프로젝트 생성 시 별도 구글시트 없이 템플릿에서 spin-up

### 7-1. 프로젝트 사례 ① 킬팻캡슐 (현재 운영 중)
- 플로우: QR/링크 → 구글설문 → 시트저장 → 직원 확인(현재 수작업) → 타입판단 → 상품발송(3일무료/1달/3달)
- 웰컴/2일차/3일차(마감) 알림톡 별도 운영
- AI 문구생성 특이구조: **환자발송용 메시지 + 원장용 내부분석(타입/처방방향/상담포인트) 2단 출력**
- NotebookLM으로 시트 실시간 동기화하여 프롬프트 기반 생성 중 (프롬프트 추후 공유)
- 새 응답 자동감지 필요 (현재 수작업)

## 13. [신규, 2026-07-07] 치료처방 확장구조 + 검사 + 환자 통합뷰

### 13-1. 치료처방 — 확장 가능한 통합 프로그램 구조
- "치료처방"(구 처방등록) 메뉴 아래, 서로 다른 성격의 프로그램들을 하나의 구조로 통합:
  - 다이어트(비만치료): 킬팻캡슐 체험3일 / 1개월 / 3개월
  - 강근단 (근감소증 처방약)
  - 향후 확장 예정: 치료한약, 보약, 첩약 등
- 원칙: 환자가 "내원체크"에서 특정 진료(예: 초진/급여치료)를 받으면 자동으로 그에 맞는 톡 사이클(예: 2일톡)이
  생성되는 것과 마찬가지로, **"치료처방"에서 특정 프로그램(예: 킬팻캡슐 3일체험)에 등록되면
  자동으로 그 프로그램 전용 톡/할일 사이클이 생성**되어야 함
- 내원체크 트리거와 치료처방 트리거는 독립적으로 각자의 할 일 사이클을 만듦 (동시에 발생 가능)

### 13-2. 킬팻캡슐 3일체험 — 최우선 구현 대상 (구조가 가장 복잡)
- 참고 시트: "규림비만다이어트캡슐시트" — 구분(초진/재진/비대면), 웰컴톡/2일차/마감설문 컬럼 존재
- 톡 구성: 웰컴톡(처음 템플릿) → 2일차톡 → 3일차 마감톡(마감설문 링크 포함), 현재 전부 수작업 발송+체크
- **AI 문구 톤 원칙**: 3일체험은 무료 상품이라 안 먹는 사람이 많음 → 목적은 학술적 설명이 아니라
  "내 설문을 보고 맞춤으로 짚어줬다"는 인상을 짧게 주고 마감설문 작성 → 본상담(유료 티켓팅) 유도.
  기존 NotebookLM 기반 예시(권정은님 케이스)보다 더 간결하게 다듬을 것.
- **[2026-07-07, 구현 착수]** A안(Program.type="FIXED_SEQUENCE" + ProgramEventTemplate/
  ProgramEventLog 신규 테이블)으로 설계 승인, 구현 진행 중.

### 13-3. 구글폼 실시간 연동 (가능성 확인됨)
- 현재: 구글폼 응답 시트를 NotebookLM에 실시간 동기화 → 원장님이 추출해 수작업으로 AI 활용 중
- **결론: 우리 시스템 내에서도 가능** — 구글시트 API(서비스 계정, 읽기 권한)로 연결 후,
  서버가 주기적으로(폴링 방식) 새 응답을 확인해 파싱 → 기존 AI 엔진(OpenAI)으로 3종 메시지 일괄 생성
- 로컬 서버 특성상 웹훅(즉시알림)은 불가하여 폴링 방식으로 처리 (체감 차이 적음)
- 준비물: 구글 클라우드 서비스 계정 생성 + 해당 시트 보기 권한 공유 + 인증키 `.env` 등록
- 추후 이 구글폼 자체를 시스템 내부 정보취합 기능으로 대체하는 것도 검토 가능

### 13-4. 3일체험 → 본상담 → 본프로그램 전환 추적
- 3일체험 마감설문 응답 시 "본상담 희망" 여부(현재 시트상 노란색 표시) 확인
- 본상담 희망 시, 이후 **킬팻캡슐 1개월 / 3개월** 두 본프로그램 중 실제로 어디로 전환되는지
  (또는 전환 안 됨) 추적
- **전환 퍼널 통계** 필요: 체험→본상담희망→실제유료전환 비율, 전환까지 걸린 기간 등
- 프로그램 명칭(3개월/1개월/TEST)은 확정 아님, 추후 조정 가능

### 13-5. 해피톡 (본프로그램 처방주기 관리)
- 킬팻캡슐 1개월/3개월은 기존 탕약(감비탕 등)과 처방주기 관리 방식 동일 (9-1 참고, 2주 단위 분할처방)
- "해피톡" = 다음 처방(다음 차수) 안내 메시지. 현재는 한차트 메모에 수기로 남겨 누락은 방지 중
- **타이밍 원칙**: 정확히 2주 되는 날이 아니라, 주말/개인사정 등 변수를 감안해 **2주 되기 1~2일 전** 발송
  ("다 드셔가는데 다음 처방 하셔야죠?" 개념)
- **미해결 문제**: 해피톡 발송 여부는 챙기지만, 그 이후 **환자 응답/실제 다음 처방 연결 여부는 추적이 안 되고
  그냥 넘어가는 경우가 있음** → 시스템으로 추적 보완 필요

### 13-6. 검사 (신규 핵심 기능)
- **인바디(체성분) 검사**: 비만 프로그램(특히 3개월) 대상, 내원 시에만 측정 가능
  - 정석은 2주 1회(처방주기와 일치), 최소 월 1회가 암묵적 룰
  - 최소 3회(초기/중간 2달차/마지막) 비교로 감량 추이 확인
- **근력검사 (강근단/근감소증·노쇠 프로그램 대상, 월 1회)**:
  - SMI지표: 키 + 사지(양팔/양다리) 골격근량 수기 입력 → 자동 합산 및 SMI 계산 →
    성별 기준(남 SMI<7.0, 여 SMI<5.7 kg/m²) 대비 근감소증 여부 자동 판정
  - 악력: 왼손/오른손 수기 입력 → 평균 자동계산 → 성별·연령대별 기준표(WEAK/NORMAL/STRONG) 대비 판정
  - 사용 방식: 환자 앞에서 그 자리에서 수치 입력 → 결과/판정 즉시 설명
- **누적 기록 필수**: 현재 구글시트는 1회성 계산기라 흐름(개선/유지 추이)을 못 보여줌 →
  시스템에서는 검사 이력이 누적되어 추이를 볼 수 있어야 함
- 원장님 판단: 이 "검사" 기능까지 구현되면, 한의원에서 실제로 하는 치료처방/해피톡/CRM 개념이
  구조적으로 거의 다 커버된다고 봄 (13장 전체가 핵심 완성 지점)

### 13-7. 환자 통합뷰 (상세 프로필)
- 내원목록에서 환자별로 현재 진행 중인 치료처방/검사가 뱃지 등으로 직관적으로 노출
- 특정 항목(예: "킬팻캡슐 3개월" 뱃지) 클릭 시 치료처방 화면으로 이동해 해당 환자의 현재 진행 상태
  (차수, 다음 처방일 등) 확인 가능
- 환자를 클릭하면 그 환자의 치료처방/검사 데이터가 누적되어 **그래프 등 시각적 통계**로 표시되는
  통합 프로필 화면
- **UI 품질 중요 포인트**: 규림한의원은 환자가 직접 화면을 보는 경우가 많음 →
  이 화면은 상담 시 환자에게 직접 보여주는 용도로도 쓰이므로 디자인 완성도가 중요
