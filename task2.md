## [긴급] /s/[token] 페이지가 내부 AppShell에 노출되는 문제 수정

### 문제 (심각도 높음 — 정보노출)
- /s/[token] 공유링크 페이지가 사이드바+현재사용자 헤더+우측 실시간 활동피드가
  포함된 일반 내부 화면(AppShell) 레이아웃으로 렌더링되고 있음
- 실시간 활동피드에는 실제 환자 이름(초진상담 기록, 톡발송 확인 등)이 그대로
  노출되는데, 이 페이지는 환자에게 직접 전달되는 공개 링크이므로 절대 안 됨
- 기존 /p/[token](프로그램티칭 공개페이지)은 "사이드바 없는 독립 화면"으로
  올바르게 예외처리 되어 있었는데, 이번에 새로 만든 /s/[token]은 그 예외
  목록에 빠진 것으로 추정됨

### 확인 및 수정
1. AppShell(또는 사이드바+활동피드 레일을 렌더링하는 루트 레이아웃/컴포넌트)이
   어떤 기준으로 "독립 화면"(/p/[token], /consult-mode, /patient-view/* 등)을
   판단해 사이드바/활동피드를 숨기는지 확인
2. /s/[token]도 동일한 예외 목록/로직에 포함시켜, 사이드바·현재사용자 헤더·
   실시간 활동피드 전부 렌더링되지 않도록 수정
3. 로그인 여부와 무관하게(직원 세션이 있는 브라우저로 열어도) 항상 독립 화면으로
   렌더링되는지 반드시 확인 — 지금 문제는 로그인 상태에서 테스트해서 드러난 것이므로,
   조건이 "로그인 안 했을 때만 숨김"처럼 잘못 걸려있었을 가능성도 확인할 것

### 검증 요청 (필수)
1. 직원 로그인된 브라우저에서 /s/[token] 접속 → 사이드바/활동피드/현재사용자
   헤더 전부 안 보이는지 확인
2. 시크릿모드(비로그인 상태 시뮬레이션)에서도 동일하게 확인
3. 티칭지만/이벤트만/통합 3가지 조합 모두 같은 방식으로 안전하게 렌더링되는지 확인
4. 기존 /p/[token]도 회귀 없이 여전히 독립 화면으로 잘 나오는지 확인
5. npx tsc --noEmit, npm run build 통과 확인

### 완료 후 보고
- 근본 원인이 정확히 무엇이었는지 (AppShell 판단 로직의 어느 부분이 문제였는지)
- 위 검증 결과
- 혹시 이미 실제 환자에게 발송된 /s/[token] 링크가 있었는지도 다시 한번 확인 요망
  (현재까지는 원장님 본인 테스트 목적으로만 발송된 것으로 파악되나, 재확인 필요)
