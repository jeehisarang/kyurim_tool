# 작업 지시서 — 톡생성기 "링크 포함하기"에 추천링크 체크박스 추가

## 배경
현재 톡생성기(TalkGroupManager, AI 생성 화면)의 "링크 포함하기" 섹션에는
프로그램티칭/이벤트/검사결과(검사톡) 3개 체크박스만 있음. 추천링크는 지금
2일차톡에만 자동 삽입되고, 그 외 톡(웰컴톡 등)에서 필요할 때는 처방 상세
페이지로 돌아가 복사해야 하는 번거로움이 있음.

## 수정
- "링크 포함하기" 섹션에 4번째 체크박스 "추천링크" 추가
- 현재 환자의 활성 ReferralLink(만료 전, isActive) 조회:
  - TRIAL 타입이 있으면 "추천링크(체험)"으로 표시
  - MAIN 타입이 있으면 "추천링크(본프로그램)"으로 표시
  - 둘 다 있으면 두 개 다 노출(체크박스 2개로 분리) — 흔치는 않겠지만 가능한
    케이스이므로
  - 활성 링크가 아예 없으면 체크박스 자체를 숨김(선택 못하게 — 없는 링크를
    보내는 상황 방지)
- 체크 시 "선택 문구 생성" 버튼 클릭 결과에, 2일차톡에서 쓰던 것과 동일한
  고정 문구 블록(안내멘트+링크+만료일)이 최종 텍스트 뒤에 자동 삽입됨
- 이미 2일차톡에는 자동으로 삽입되고 있으므로, 2일차톡 생성 시엔 이 체크박스가
  기본으로 체크된 상태로 시작(중복 방지 목적이 아니라 일관성 위해) — 체크
  해제하면 안 들어가게 유지
- 톡 종류(웰컴/2일차/마감) 구분 없이 어떤 톡에서든 이 체크박스가 뜨도록 구현

## 검증
- 킬캡3체험 웰컴톡 화면에서 "추천링크(체험)" 체크박스가 뜨고, 체크 후 문구
  생성 시 링크가 정상 삽입되는지 확인
- 만료된 링크만 있는 환자(또는 링크 자체가 없는 환자)는 체크박스가 안 뜨는지 확인
- 2일차톡에서는 기본 체크 상태로 시작하는지, 해제 시 정상적으로 안 들어가는지 확인
- 기존 프로그램티칭/이벤트/검사결과 체크박스 동작에 회귀 없는지 확인
- npx tsc --noEmit / npm run build 통과
