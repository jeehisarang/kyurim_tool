## 작업 지시서 — 치료처방(Program) 원장 전용 수동 등록 기능

### 배경
- 현재 Program(치료처방 프로그램 마스터)은 seed 스크립트로만 생성되어 있어, 
  원장님이 UI에서 직접 새 프로그램(예: 강근단, 숨한약)을 추가할 방법이 없음
- 다이어트(킬팻캡슐 3체험/1개월/3개월)류만 등록되어 있고, 검사연동/카테고리 
  설정이 필요 없는 단순 프로그램(이름+기간+해피톡주기)부터 원장님이 직접 
  추가할 수 있게 할 것
- 검사연동, 카테고리 분류 등은 이번 범위 밖 (다음 단계에서 별도 작업)

### 스코프 결정사항 (확정)
- 해피톡 주기는 **2주 또는 4주(1달) 중 선택**만 가능 — 자유 입력 아님
- 신규 등록 시 입력 항목: **프로그램 명칭 / 총 기간(개월) / 해피톡 주기(2주|4주)** 
  3가지만
- 검사연동(linkedTestType), 카테고리(탕약/환/킬팻캡슐 분류) 등은 이번엔 
  받지 않음 — 등록 후 기본값(연결검사 없음, 카테고리 미분류 또는 "기타") 
  으로 생성, 추후 별도 화면/작업에서 보완
- **원장 계정에서만 접근 가능** (기존 설정 메뉴처럼 role 서버단 재검증 필수)

### 설계 확인 필요 (Claude Code가 기존 코드 확인 후 아래 가정 검증)
- 기존 SPLIT 타입 프로그램(탕약/환약/킬팻캡슐)의 "2주 간격" 이 `splitPeriodDays` 
  같은 필드값으로 관리되는지, 아니면 계산 로직(computeSplitSchedule 등)에 
  하드코딩되어 있는지 확인 필요
  - 필드로 관리 중이면: 신규 등록 시 14 또는 28을 그대로 저장하면 됨 (스키마 
    변경 불필요할 수 있음)
  - 하드코딩되어 있으면: `splitPeriodDays`(Int) 필드를 Program에 추가하고, 
    관련 계산 로직(다음 회차 예정일, computeSplitSchedule 등) 전체가 이 
    필드값을 읽도록 일반화 — **기존 프로그램(탕약/킬팻캡슐 등) 전부 14로 
    마이그레이션하여 회귀 없도록 할 것**
- "총 기간(개월)" 입력이 `totalRounds`(총 회차)로 자동 환산되는 방식인지 확인 
  (예: 3개월 + 2주 주기 = 약 6~7회차, 기존 킬팻캡슐 방식과 동일 계산식 재사용)

### 대상 파일 (Claude Code가 정확한 경로 재확인)
- Prisma 스키마 — Program에 `splitPeriodDays` 필드 없다면 추가 + 마이그레이션 
  (기존 레코드 기본값 14로 백필)
- 신규 API: `POST /api/programs` (원장 전용, 서버단 role 재검증), 
  명칭 중복 체크(대소문자/공백 정규화 후 비교 — 기존 "강근단" vs "강근단 " 
  같은 중복 방지)
- 신규 화면: `/settings/programs` (또는 기존 설정 메뉴 하위) — 등록된 프로그램 
  목록(활성/비활성) + "새 프로그램 등록" 폼(명칭/총기간/해피톡주기 라디오)
- 기존 회차 스케줄링 로직 파일 — `splitPeriodDays` 필드값을 읽도록 일반화

### 동작 규칙
1. 명칭은 필수, 공백만 있거나 빈 값 불가, 기존 활성 프로그램과 중복(정규화 
   비교) 시 저장 차단 + 안내 메시지
   - 단, **비활성화된 기존 프로그램과 이름이 겹치는 경우** 신규 생성을 막지 
     말고 경고만 표시 ("비활성 상태의 동일한 이름의 프로그램이 있습니다. 
     계속하시겠습니까?") — 실수로 옛날 걸 재활성화하는 대신 새로 만드는 걸 
     막지 않기 위함
2. 총 기간(개월) → 해피톡 주기 기준으로 총 회차(totalRounds) 자동 계산 
   (기존 킬팻캡슐 계산 방식과 동일 로직 재사용)
3. 신규 프로그램은 기본적으로 카테고리 미지정/"기타", 연결검사 없음(null) 
   상태로 생성 — 치료처방 목록 화면에서 기존 3개 카테고리(탕약/환/킬팻캡슐) 
   탭에 안 걸리는 프로그램은 별도로 노출되도록 확인만 (탭 구조 자체를 
   바꾸는 건 이번 범위 아님)
4. 등록된 프로그램은 즉시 `/prescriptions/new`(치료처방 등록)의 프로그램 
   선택 목록에 나타나야 함
5. 원장 계정이 아닌 경우 이 화면 접근 시 403 (기존 설정 메뉴 패턴과 동일)

### 검증 체크리스트
- [ ] 원장 계정으로 "강근단", 기간 3개월, 해피톡 2주 등록 → 저장 성공, 
      회차 자동계산값 확인
- [ ] 직원 계정으로 동일 화면 접근 시 차단(403) 확인
- [ ] 동일 이름(공백만 다르게) 재등록 시도 시 중복 차단 확인
- [ ] 비활성 상태의 동일 이름 있는 경우 경고 후 계속 생성 가능한지 확인
- [ ] 신규 등록한 프로그램이 `/prescriptions/new`에서 바로 선택 가능한지 확인
- [ ] 신규 등록 프로그램으로 실제 처방 등록 후, 해피톡 알림(2주 또는 4주 
      기준)이 정확한 날짜에 오늘 할 일에 생성되는지 확인
- [ ] 기존 탕약/킬팻캡슐 등 프로그램의 회귀(기존 2주 스케줄링 정상 동작) 확인
- [ ] npx tsc --noEmit, npm run build 통과
