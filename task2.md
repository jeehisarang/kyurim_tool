# 작업 지시서 — 오늘 할일 톡/업무 분리 표시 + 홈 화면 폭 불일치 수정

## 1. /todo 페이지: 톡/처방 vs 업무 좌우 분리
- 현재 하나의 표에 톡/처방/업무가 섞여 있고, 업무 항목은 환자명/프로그램명이 항상
  "-"로 비어 공간 낭비 발생
- 좌우 2단 레이아웃으로 분리 (좌측 넓게, 우측 좁게 — 항목 수 비율 고려):
  - 좌측 "톡/처방": 기존 컬럼 그대로 (환자명/프로그램명/할일종류/마감/담당자/완료여부)
  - 우측 "업무": 환자명/프로그램명 컬럼 제거, 할일종류/마감/담당자/완료여부만 표시
    (수정/삭제 버튼 포함)
- 밀린 할일 / 오늘 할일 구간 분리는 기존 그대로 유지, 각 구간 내에서 좌우 분리 적용
- 반응형: 화면 폭이 좁아지면(모바일 등) 좌우 분리 대신 세로로 스택되도록 처리
- 완료된 항목 보기 토글도 좌우 각각 독립적으로 동작 (또는 공통 토글 하나로 둘 다 열고
  닫아도 무방 — 구현 편한 쪽으로 판단)

## 2. 홈 화면 위젯 폭 불일치 수정
- 홈 화면 "오늘 할일 미리보기" 위젯이 `/todo` 전체 페이지와 폭이 다르게 렌더링되고 있음
  (예전 표 여백 확장 작업에서 이 위젯이 누락된 것으로 추정)
- `/todo` 페이지와 동일한 폭 기준을 홈 화면 위젯에도 동일하게 적용
- 홈 위젯 자체는 상위 N개만 보여주는 미리보기 성격은 유지 (좌우 분리까지는 필요 없음,
  기존처럼 통합 리스트 형태 유지해도 무방 — 폭만 맞추면 됨)

## 검증 요청
1. /todo에서 톡/처방과 업무가 좌우로 분리되어 표시되는지, 업무 쪽에 불필요한 빈 컬럼이
   없는지 확인
2. 화면 폭 좁힐 때(모바일 등) 세로 스택으로 정상 전환되는지 확인
3. 홈 화면 위젯과 /todo 페이지 폭이 시각적으로 일치하는지 확인
4. npm run build 통과 확인
