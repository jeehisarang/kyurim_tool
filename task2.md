[긴급 추가 지시 — 작업 중 반영] 상담모드 화면에 증상 패턴 체크리스트 진입점 추가

방금 원장님이 실제 환자 정보를 가장 많이 입력하는 화면이 "/consultation" (상담모드 — 
환자검색 → 상담유형 드롭다운 → 자유텍스트 → AI 차팅변환) 이라는 걸 확인했습니다. 
이 화면이 세 번째 진입점으로 추가되어야 합니다 (기존 계획: 독립메뉴 "상담설문" + 
검사등록화면 바로가기, 여기에 상담모드 진입점 추가).

## 반영 방식
1. "상담유형" 드롭다운(현재 초진상담 등)에 "증상 패턴 체크" 옵션 추가
2. 이 옵션 선택 시, 기존 자유텍스트 textarea를 12문항 체크리스트 UI로 교체 표시
   (다른 상담유형 선택 시엔 기존 자유텍스트 그대로 유지)
3. "저장" 버튼 클릭 시, 증상 패턴 체크 데이터는 기존 상담기록 테이블이 아니라 
   TcmChecklistResponse에 독립 저장 (patientId 기준, source: IN_CLINIC, 
   submittedByStaffId는 "현재 사용자" 드롭다운 값 사용)
4. 기존 상담 기록(자유텍스트+AI차팅변환) 로직은 전혀 건드리지 말 것 — 완전히 
   별개의 저장 경로로 추가만 할 것

이미 진행 중인 스키마/UI 작업과 병행 가능한 범위면 이번 라운드에 같이 반영하고, 
너무 크게 얽히면 다음 라운드로 분리해도 됩니다 — 판단해서 진행하세요.
