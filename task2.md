# 작업 지시서 — 신청폼(TrialApplicationForm) 디자인 개선

## 배경
실사용 확인 중 발견된 문제 + 디자인 개선 필요:
1. 설정화면 "설명 문구"에 줄바꿈으로 입력해도 페이지에서 한 줄로 뭉쳐 나오는
   버그 (white-space 미보존)
2. 문구 자체(헤드라인/설명/체크리스트)는 그대로 유지 — 텍스트 내용은 안 바꿈
3. 기본정보 12개 필드가 세로로 쭉 나열되어 길게 느껴짐
4. 몸타입 6문항이 진행상황 표시 없이 끝없는 리스트처럼 느껴짐
5. 선택지 버튼이 밋밋한 리스트형이라 선택 여부가 잘 안 보임
6. CTA 버튼("신청하기")이 존재감 없는 회색 톤

## 1. 설명 영역 재구성 (줄바꿈 버그 수정 + 시각 계층 + 색상)

### 줄바꿈 버그
- 설명 문구 렌더링 부분에 `white-space: pre-line` (또는 동일 효과의 스타일)
  적용해 설정화면에서 입력한 줄바꿈이 그대로 반영되게 수정

### 구조 재편 (텍스트는 그대로, 표시 방식만 변경)
- 현재 하나의 긴 문단으로 렌더링되는 설명 문구를, 아래 3부분으로 자동 분리해서
  스타일 차등 적용:
  1. 첫 문장(따옴표로 감싸인 캐치프레이즈, "변화는 몸에서...")과 이어지는 소개
     문장 → 이탤릭체, 진한 적갈색(`#712B13`), font-weight 500
  2. 중간 설명 문단 → 일반 본문 스타일 그대로(`var(--text-secondary)`)
  3. "✓"로 시작하는 문장들 → 한 줄로 이어붙이지 않고 각각 별도 줄의 체크리스트로
     분리 렌더링, 골드 톤 배경 카드(`#FAEEDA`) 안에 배치, 체크 아이콘 색상
     `#BA7517`, 텍스트 색상 `#633806`
  4. 마지막 문장("~ 먼저 경험해보고 결정하세요") → 가운데 정렬, 진한 적갈색
     (`#4A1B0C`), font-weight 500
- 문장 분리 로직: 원장님이 입력한 설명 문구 텍스트를 파싱해서 "✓"로 시작하는
  줄들을 자동으로 체크리스트 블록으로 분리(정규식 등으로 처리), 나머지는
  본문/캐치프레이즈로 표시 — 설정화면 자체는 지금처럼 자유 텍스트 입력 그대로
  유지(별도 필드 분리 안 함)

## 2. 기본정보 섹션 레이아웃 개선
- 필드들을 "기본 정보" 섹션 카드로 묶고, 짧은 값끼리는 2열 배치
  (키/체중 + 감량목표 나란히 배치)
- 나머지 필드(연락처/체중변화/복용약물/병력/가족력/다이어트경험)는 기존처럼
  세로 배치 유지

## 3. 몸타입 6문항 — 진행표시 + 카드형 선택지
- 각 문항 상단에 "N / 6" 진행 텍스트 + 얇은 진행바(전체 진행률 표시)
- "최대 2개" 안내를 작은 알약(pill) 배지 스타일로 변경
- 선택지 버튼을 카드형으로 재설계:
  - 좌측에 원형 배지(A/B/C/D/E)
  - 미선택 시: 무채색 배지+테두리
  - 선택 시: 강조색 배지(`var(--fill-accent)`)+강조 테두리+연한 배경
- "기타" 옵션은 평소엔 작은 링크형("+ 기타 직접 입력")으로만 표시, 클릭 시에만
  텍스트 입력창이 나타나도록 변경 (기존엔 항상 빈 입력창이 보이던 것 개선)

## 4. CTA 버튼
- "신청하기" 버튼을 강조색 채움 스타일로 변경 (`background: var(--fill-accent)`,
  `color: var(--on-accent)`), 기존 회색 아웃라인 스타일에서 교체

## 검증
- 설정화면에 줄바꿈 포함 문구 입력 → 페이지에서 줄바꿈 그대로 반영되는지 확인
- "✓"로 시작하는 문장이 자동으로 체크리스트 카드로 분리되어 렌더링되는지 확인
- 기본정보 2열 배치, 모바일 폭에서 깨지지 않는지 확인
- 6문항 진행바가 문항 이동할 때마다 정상 갱신되는지 확인 (스크롤 방식 유지
  이므로 viewport 진입 감지 또는 폼 스크롤 위치 기준 처리 — 방식은 Claude Code
  판단)
- 선택지 카드 선택/해제 시 강조 스타일 정상 토글 확인
- "기타" 입력창이 클릭 전엔 숨겨져 있다가 클릭 시 나타나는지 확인
- 기존 테스트 데이터로 실제 신청 제출까지 전체 플로우 회귀 없는지 확인
- npx tsc --noEmit / npm run build 통과
