## 작업 지시서 B — 실시간 활동피드: PATIENT 항목 강조 + "확인" 체크 기능

### 배경
- 우측 고정 활동피드 레일에서 PATIENT(환자 행동) 로그는 빨간 글씨로 구분되고 있음, 
  원장님 반응 좋음 → 이 항목을 더 눈에 띄게 강조하고, "누군가 이걸 봤다"는 것을 
  확인할 수 있는 체크 기능 추가 요청

### 명칭/디자인 변경
- PATIENT 항목: 글자 크기 확대 + 앞에 이모티콘(예: 🔴 또는 📣) 추가 — 기존 빨간색은 유지
- 각 PATIENT 항목 옆에 "확인" 버튼/체크박스 추가
- 체크 시: 
  - 해당 항목이 "OOO 확인함" 형태로 시각적 표시(체크완료 아이콘 + 흐림처리 등), 
    이후 다른 사람이 다시 체크 못 하도록 버튼 비활성화
  - 피드에 새 항목 자동 추가: "{체크한 사람}님이 확인했습니다" (원본 항목 내용을 
    짧게 참조 — 예: "최승희님이 '김시은님 [엠타이트] 프로그램문의하기' 확인했습니다")

### 스코프
- 체크 기능은 **PATIENT(빨간색) 항목에만** 적용 (STAFF/SYSTEM 항목은 대상 아님)
- "확인했습니다" 자체가 새로 생성된 로그 항목이므로, 그 항목엔 체크박스가 다시 
  붙지 않아야 함 (무한 체인 방지)

### 대상 파일 (Claude Code가 정확한 경로 재확인)
- Prisma 스키마 — `ActivityLog`에 필드 추가: `isChecked`(Boolean, default false), 
  `checkedByStaffId`(nullable FK → StaffUser), `checkedAt`(nullable DateTime)
- 체크 처리 API: `POST /api/activity-logs/[id]/check` — 현재 선택된 사용자(StaffUser) 
  기준, 서버단에서 `isChecked=false` 조건으로 원자적 업데이트(이미 체크된 경우 
  409 반환 + 누가 체크했는지 응답) → 성공 시 새 ActivityLog(SYSTEM 또는 STAFF 타입) 
  "{staffName}님이 확인했습니다" 생성
- 활동피드 렌더링 컴포넌트 — PATIENT 항목 글자크기/이모티콘 스타일 변경, 체크박스 
  UI 및 클릭 핸들러 추가, 폴링(7초) 시 체크 상태도 함께 갱신되도록 확인

### 동작 규칙
1. 체크는 **최초 1인만 가능** — 이미 체크된 항목은 버튼이 비활성화되고 
   "{이름}님 확인" 라벨로 표시
2. 동시에 두 사람이 거의 같은 시각에 클릭하는 동시성 문제 방지: 서버단 원자적 
   업데이트로 두 번째 요청은 실패 처리, 프론트는 실패 시 최신 상태로 자동 갱신
3. 체크 액션 자체도 활동피드에 새 항목으로 남아야 함 (체크박스 없는 형태로)
4. 독립창(환자와 함께보기, 상담모드, 공개 티칭지/이벤트 페이지)에는 기존처럼 
   활동피드 자체가 미노출 — 이번 변경과 무관하게 유지

### 검증 체크리스트
- [ ] PATIENT 항목 강조 스타일(글자크기/이모티콘) 정상 적용 확인
- [ ] 체크 버튼 클릭 → 항목 상태 변경 + 새 "확인했습니다" 항목 생성 확인
- [ ] 이미 체크된 항목에서 재클릭 시도 시 비활성화 상태 확인 (버튼 자체가 안 눌리거나, 
      눌러도 서버에서 막히는지)
- [ ] 두 명이 동시 클릭 시도 시 한쪽만 성공하는지 확인 (수동 재현 가능한 선에서)
- [ ] STAFF/SYSTEM 항목에는 체크 버튼 자체가 안 보이는지 확인
- [ ] "확인했습니다" 항목 자체에 체크박스가 없는지 확인
- [ ] npx tsc --noEmit, npm run build 통과
- [ ] 기존 활동피드 폴링/독립창 미노출 동작 회귀 없음 확인
