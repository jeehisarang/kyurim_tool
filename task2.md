# 작업 지시서 — 홈/내원체크 환자목록의 프로그램 배지도 상세페이지로 연결

## 배경
치료처방 목록/환자프로필 화면에서는 프로그램 배지(예: "킬팻캡슐·3일체험") 클릭 시
`/prescriptions/[prescriptionId]` 상세페이지로 이동하도록 이미 수정 완료됨(7/15 세션).
그런데 홈 화면(`/home`)과 내원체크 화면(`/visit-check`)의 "오늘 내원한 환자 목록"에도
동일한 프로그램 배지가 표시되는데, 여기서는 클릭해도 상세페이지로 연결되지 않거나
(또는 예전 버그처럼 엉뚱한 곳으로 이동하거나) 클릭 자체가 안 되는 상태로 추정됨.

## 대상 파일
- 홈 화면(`/home`)의 "오늘 내원한 환자 목록" 컴포넌트
- 내원체크(`/visit-check`) 화면의 환자 목록 컴포넌트 (스크린샷 속 화면)
- 이미 올바르게 동작 중인 치료처방 목록/환자프로필 쪽 배지 클릭 핸들러 코드를 참고해서
  동일한 방식으로 맞출 것 (가능하면 공용 컴포넌트/함수로 추출해서 재사용 — 이미
  CategoryBadge/VisitTypeTag처럼 공용 컴포넌트가 있으니 프로그램 배지도 같은 패턴으로
  통일 검토)

## 동작 규칙
1. 홈 화면과 내원체크 화면의 환자목록에 표시되는 프로그램 배지를 클릭하면, 해당
   환자의 해당 치료처방 상세페이지(`/prescriptions/[prescriptionId]`)로 이동
2. 한 환자가 여러 프로그램을 동시 진행 중이라 배지가 여러 개 표시되는 경우, 각 배지는
   각자 자기 프로그램의 상세페이지로 연결 (치료처방 목록에서의 다중 뱃지 처리 방식과 동일)
3. 배지가 아닌 환자 이름을 클릭했을 때의 기존 동작(환자 상세 프로필 이동 등)은 그대로 유지 —
   배지 클릭과 이름 클릭이 서로 다른 목적지로 가는 기존 구분은 유지할 것
4. 회귀 없어야 함: 치료처방 목록/환자프로필 쪽 기존 배지 클릭 동작은 변경 없이 그대로 유지

## 검증 체크리스트
- [ ] 홈 화면에서 프로그램 진행 중인 환자의 배지 클릭 → 올바른 상세페이지로 이동 확인
- [ ] 내원체크 화면에서 동일하게 확인
- [ ] 한 환자가 여러 프로그램을 동시 진행 중인 케이스로 다중 배지 각각 확인
- [ ] 배지 없는 환자, 배지 있는 환자 둘 다 나머지 화면 동작 정상 확인
- [ ] 기존 치료처방 목록/환자프로필 쪽 배지 클릭 회귀 없는지 확인
- [ ] npx tsc --noEmit, npm run build 통과
